## 摘要
键值（KV）缓存增长是自回归解码中的主要瓶颈，因为内存和带宽随上下文长度线性扩展。现有的KV驱逐方法通常依赖于静态启发式或代理分数，这些方法难以准确追踪未来令牌的效用，并且在相关性变化时会导致脆弱的驱逐。为了解决这个问题，我们引入了KVpop，它通过直接监督保留或丢弃决策来学习固定预算的KV驱逐策略。该评分器针对一种新颖的未来注意力目标进行训练，该目标无需物化密集的注意力图即可高效计算。我们进一步引入了一种基于延迟记忆的评分器，这在学习型驱逐方法中独一无二，它将评分延迟固定数量的步骤以利用近期未来的上下文。在AIME和HMMT数学推理任务上，KVpop在Qwen3-4B模型上达到了75% KV缓存压缩时保留98%的全注意力性能，以及在88%压缩时保留97%，持续优于已有的驱逐基线。Qwen3-8B模型显示出更强的结果，接近完全的教师模型性能。这些结果表明，使用未来注意力信号监督驱逐可以在保持质量的同时降低内存成本。

## Abstract
Key-value (KV) cache growth is a major bottleneck in autoregressive decoding, as memory and bandwidth scale linearly with context length. Existing KV eviction methods often rely on static heuristics or proxy scores, which poorly track future token utility and cause brittle eviction as relevance shifts. To address this, we introduce KVpop, which learns a fixed-budget KV eviction policy by directly supervising the keep-or-drop decision. The scorer is trained against a novel future-attention target, computed efficiently without materializing dense attention maps. We further introduce a delayed memory-based scorer that, uniquely among learned eviction methods, defers scoring for a fixed number of steps to exploit near-future context. On AIME and HMMT mathematical reasoning, KVpop retains 98% of full-attention performance on Qwen3-4B at 75% KV cache compression and 97% at 88% compression, consistently outperforming established eviction baselines. Qwen3-8B shows even stronger results, reaching near-full teacher performance. These results show that supervising eviction with future-attention signals cuts memory costs while maintaining quality.

- **问题**：大规模语言模型（LLM）在自回归解码时依赖键值（KV）缓存来存储之前 token 的表示，但 KV 缓存随上下文长度线性增长，成为长序列推理的内存和带宽瓶颈。现有方法通过驱逐不重要的 token 来压缩缓存，但传统的静态启发式或代理分数（如累积注意力、局部相似性）难以预测 token 未来的效用，导致驱逐决策脆弱，尤其在推理轨迹中早期 token 可能在后期重新变重要。

- **整体含义**：本文提出 **KVpop**，一种有监督的固定预算 KV 驱逐政策，通过直接学习“保留/丢弃”决策，并利用未来注意力信号作为监督目标，使驱逐决策能够预测 token 未来的重要性。KVpop 能够在显著降低内存消耗的同时，保持与全注意力教师模型接近的生成质量，为长上下文 LLM 推理提供更高效的方案。

## 2. 方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 为每个 KV 头维护一个固定预算的缓存，包含三个部分：**sink token**（前 s 个）、**保护窗口**（最近的 w 个 token）、以及 **长期 top-k**（从剩余 token 中选出 k 个）。总预算 B = s + w + k，每个头的缓存大小恒定。
- 训练一个轻量级的每头评分器（scorer），在线预测每个 token 的保留重要性。评分器由**未来注意力目标**（future-attention target）监督，该目标量化 token 在离开保护窗口后从未来查询中获得的注意力质量。
- 使用**边界感知损失**（boundary-aware loss）训练评分器，只关注新被允许驱逐的 token 与教师截止 token 之间的排名比较，计算高效。
- 引入**延迟记忆评分器**（delayed memory-based scorer），利用 mLSTM 状态累积 token 进入保护窗口后的近未来上下文，在 token 即将变为可驱逐时才进行评分，做出更明智的决策。

### 关键技术细节

#### （a）未来注意力目标
- 定义 token `t` 的平均未来注意力质量：
  \[
  m_t^{(h,g)} = \frac{1}{N_t}\sum_{d=t+w}^{S-1} p_{d\to t}^{(h,g)}
  \]
  其中 `p` 是全注意力的 softmax 概率，`w` 是保护窗口大小。
- 聚合多个查询头的 log-masses 得到目标分数 `r_tgt(h,t)`。
- 为避免物化稠密注意力图（S×S），通过**转置注意力**（transposed attention）技巧高效计算：将 key 作为 query、原 query 作为 key，利用注意力 kernel 返回的 log-sum-exp（LSE）代替列求和，仅需一次稀疏注意力前向传播即可得到所有 token 的未来注意力质量。

#### （b）教师有效分数与排序
- 在推理时，教师使用有效分数排序：
  \[
  r_tgt(h,t)(q) = r_tgt(h,t) + \lfloor (q-t)/n \rfloor \log \gamma_h
  \]
  其中 `\gamma_h` 是每头衰减因子，引入时间衰减以允许新 token 逐渐替换旧 token。
- 教师保留 Top-k 个 token，产生新可驱逐 token `t_new` 的保留标签 `y_{q,h} ∈ {+1, -1}`。

#### （c）边界损失（Boundary-Aware Loss）
- 损失函数只关注新可驱逐 token `t_new` 与教师截止 token `t_bnd` 之间的分数差：
  \[
  \mathcal{L}_{\text{score}} = \mathbb{E}_{q,h} \left[ \omega_{q,h} \, \text{softplus}\left( -y_{q,h} \frac{\hat{r}_{h,t_{\text{new}}}(q) - \hat{r}_{h,t_{\text{bnd}}}(q)}{\tau} \right) \right]
  \]
  使用 softplus 作为平滑 hinge 损失，温度 τ 控制硬度。权重 ω 可用于降低模糊决策或平衡 keep/drop 样本。

#### （d）高效在线 Top-k 选择
- 使用 Fenwick 树（树状数组）维护已插入的 token 的静态优先级，在每个查询位置增量插入新 token，通过二分查找找到当前 Top-k 截止排名，然后由 FlexAttention 内核根据此掩码动态生成稀疏注意力掩码，无需存储稠密矩阵。

#### （e）状态评分器设计
- **无状态评分器**：MLP/SiLU 直接从当前 token 的 key-value 拼接预测分数。
- **有状态评分器（延迟 mLSTM）**：每个 KV 头维护一个 mLSTM 记忆（矩阵 C 和归一化向量 z），顺序更新所有 token 的键值信息。当 token `t_new = q-w` 进入可驱逐集合时，读取其时记忆状态：
  \[
  h_{h,t_{\text{new}}|q} = \frac{\tilde{q}_{h,t_{\text{new}}}^\top C_{h,q}}{\tilde{q}_{h,t_{\text{new}}}^\top z_{h,q}}
  \]
  然后用一个小型投影输出重要性分数。这种延迟计算可以利用 token 在保护窗口期间积累的全部上下文。

### 算法流程（文字描述）
训练时（算法1）：
1. 对序列进行嵌入。
2. 对于每一层 ℓ：
   - 计算 Q、K、V 投影。
   - 使用评分器预测每个 token 的原始分数（对延迟评分器，只在 token 离开保护窗口时计算）。
   - 应用衰减得到有效分数。
   - 通过 Fenwick 树计算每查询的 Top-k 掩码。
   - 使用稀疏注意力（FlexAttention）计算前向传播并得到稀疏 LSE。
   - 通过转置注意力计算未来注意力目标。
   - 计算边界损失（仅在新可驱逐 token 上）。
3. 总损失 = KL散度（教师输出 vs 学生输出）+ 边界损失，更新模型和评分器。

推理时，仅需执行步骤 2 中的稀疏注意力，无额外开销。

## 6. 主要结论与发现

- **高压缩保留率**：在 Qwen3-4B 上，75% 压缩（CR=75%）保留 98% 全注意力性能（AIME/HMMT 平均相对 0.98），88% 压缩保留 97%；Qwen3-8B 上保留 95% 和 99%，达到或接近教师水平。
- **显著优于基线**：KVpop（特别是状态版）几乎在所有设置下优于 StreamingLLM、TOVA、StreamingLLM+ 和 DMS，尤其是在高压缩比下训练无关方法崩溃，而 KVpop 仍保持高质量。
- **延迟记忆有效**：图5 显示带延迟的 mLSTM 评分器 token 准确率比不带延迟高约 0.2 个百分点，说明利用近未来上下文有助于做出更好的驱逐决策。
- **高效推理**：KVpop 维持恒定缓存大小，推理显存增长平缓（从 18GB 到 19GB），而稠密注意力从 18GB 涨到 36GB；KVpop 端到端延迟低于稠密注意力和 DMS，尤其在长序列下吞吐更高。
- **跨领域泛化**：在 GPQA-D 和 LCB 上 KVpop 仍接近教师水平，表明学习到的驱逐策略未过拟合数学推理数据。

## 7. 优点

- **新颖的监督目标**：首次使用未来注意力质量作为驱逐监督信号，利用转置注意力高效计算，避免物化稠密图。
- **边界损失设计**：仅关注新可驱逐 token 与教师截止 token 的对比，计算代价 O(1) 每查询，而非所有 token 对，高效且聚焦关键决策。
- **延迟状态评分**：区别于 DMS 的延迟驱逐（仅推迟移除，不更新评分），KVpop 利用记忆状态累积 token 在保护窗口期间的上下文，在真正决策时给出更准确的分数。
- **固定预算结构**：均匀分配长期 top-k 给每个 KV 头，便于高效 GPU 执行（对比 DMS 的稀疏集中导致执行性能差）。
- **训练时可重用稀疏 LSE**：用学生稀疏注意力返回的 LSE 近似稠密正常化项，降低训练成本，实验证明效果足够。
- **推理零开销**：转置注意力计算和评分器更新仅在训练时进行，推理时只执行稀疏注意力，无额外计算负担。
