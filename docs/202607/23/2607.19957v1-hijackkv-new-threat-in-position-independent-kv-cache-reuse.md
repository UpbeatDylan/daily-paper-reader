## 摘要
键值（KV）缓存减少了大型语言模型（LLM）的推理延迟。传统的基于前缀的重用在跨推理请求时缓存命中率较低，因为它要求精确的标记和位置匹配。为了提高效率，最近的系统优化引入了位置无关的KV重用，允许在出现相同文本块时重用KV缓存，无论它们在序列中的位置如何。我们表明这种设计引入了一种新的威胁，即KV缓存劫持。由于KV缓存是通过标记匹配来检索的，但它编码了最初计算时的上下文，因此与看似良性的标记块相关联的KV可能编码了攻击者控制的前缀。当后来在受害者查询中重用时，这种受污染的KV会悄无声息地劫持模型的行为，即使输入中没有出现攻击者控制的文本。我们引入了HIJACKKV，这是第一个系统性地利用此漏洞的攻击框架，展示了其严重性和实用性。HIJACKKV优化了攻击者控制的前缀，使得后续常见的良性文本的KV编码了攻击者的目标，而文本本身保持不变以用于未来的缓存命中。HIJACKKV在一次尝试中平均成功率达到94%，在现实约束下仍然有效，包括低命中率（10%）和频繁重新计算（50%），在多轮交互中持续有效，并在黑盒设置中跨模型迁移。我们进一步提供了构建安全KV重用系统的设计见解。

## Abstract
Key-Value (KV) cache reduces inference latency in large language models (LLMs). Traditional prefix-based reuse has low cache hit rates across inference requests because it requires exact token and position matches. To improve efficiency, recent system optimizations introduce position-independent KV reuse, allowing KV cache to be reused whenever identical text chunks appear, regardless of their position in the sequence.   We show this design introduces a new threat, KV Cache Hijacking. Since KV caches are retrieved by token match but encode the context in which they were originally computed, the KV tied to a benign-looking token chunk may encode an attacker-controlled prefix. When later reused in a victim query, this contaminated KV silently hijacks the model's behavior, even if no attacker-controlled text appears in the input.   We introduce HIJACKKV, the first attack framework that systematically exploits this vulnerability, demonstrating its severity and practicality. HIJACKKV optimizes an attacker-controlled prefix, so that the KV computed for a subsequent common benign text encodes the attacker's goal, while the text remains unchanged for future cache hits. HIJACKKV achieves an average 94% success rate in a single attempt, remains effective under realistic constraints including low hit rates (10%) and frequent recomputation (50%), persists over multi-turn interactions, and transfers across models in black-box settings. We further provide design insights for building secure KV reuse systems.

# HijackKV：位置无关KV缓存重用中的新威胁

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：大型语言模型（LLM）推理中，KV缓存可减少延迟。传统基于前缀的重用要求精确匹配令牌和位置，导致跨请求缓存命中率低。为提高效率，近期系统优化引入了**位置无关的KV缓存重用**机制，允许任何相同文本块被复用，无论其在序列中的位置如何。
- **核心问题**：这种机制引入一种新的**安全威胁——KV缓存劫持**。由于KV缓存按令牌匹配检索，但其编码了原始计算的上下文，攻击者可通过控制前缀来“污染”后续良性文本块的KV状态。当受害者查询重用该污染缓存时，模型行为被静默劫持，即便受害者输入中不包含任何攻击者文本。这与需要恶意文本出现在输入中的提示注入有本质区别。
- **整体含义**：效率驱动的优化带来了隐蔽的安全漏洞，可能导致跨用户的输出篡改。论文首次系统性地揭示并形式化了这一漏洞，展示了其严重性和实用性。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：攻击者构造一个**对抗性前缀** \(p\)，附加到目标良性文本块 \(\tilde{X}\) 上，提交给LLM服务。服务计算前缀+文本块的完整KV，但仅将文本块对应的KV（即 \(p\oplus\tilde{X}\) 的后半部分）存入共享缓存池。当受害者查询包含同样的文本块 \(\tilde{X}\) 时（即使前缀不同），系统匹配并复用该已被污染的KV，导致模型输出攻击者期望的恶意响应 \(\tilde{r}\)。
- **关键技术细节**：
  - **优化目标**：寻找前缀 \(p\)，使得在位置无关缓存命中场景下，LLM基于污染KV生成的输出接近目标恶意答案 \(\tilde{r}\)。形式化为最小化交叉熵损失：
    \[
    p^* = \arg\min_p \mathcal{L}_{\text{CE}}\big(\text{LLM}(X\oplus q \mid T_{p}^{\tilde{X}}), \tilde{r}\big)
    \]
    其中 \(T_{p}^{\tilde{X}}\) 表示通过查询 \(p\oplus\tilde{X}\) 得到的 \(\tilde{X}\) 的KV，\(X\oplus q\) 是受害者的查询。
  - **优化算法**：采用**贪婪坐标梯度（GCG）** 算法进行离散令牌优化。步骤：
    1. 初始化随机前缀。
    2. 循环 \(T\) 步：计算当前前缀下，模拟缓存命中后的损失及其关于前缀one-hot编码的梯度。
    3. 对每个位置选择使负梯度最大的 top-k 候选令牌。
    4. 随机选择 \(B\) 个位置并替换为候选令牌，生成候选前缀集。
    5. 评估候选前缀的损失，选择损失最小的作为新前缀。
  - **模拟流程**（`Compute Loss`函数）：
    1. 将离散前缀映射为连续嵌入。
    2. 将前缀嵌入与目标文本块嵌入拼接，输入LLM得到完整KV，只提取文本块对应位置的KV作为污染缓存 \(T_p^{\tilde{X}}\)。
    3. 模拟受害者推理：将目标文本块与用户查询拼接，复用污染缓存（可选地模拟重算或压缩），运行LLM得到输出。
    4. 计算输出与目标恶意答案的交叉熵损失。
  - **攻击流程**分为两阶段：**离线优化前缀**（在本地代理模型上进行，不接触目标系统缓存）；**在线提交恶意查询**（将前缀+目标文本块发给目标服务，使其缓存被污染）。受害者后续使用相同文本块时触发攻击。

## 3. 实验设计

- **数据集**：四个QA基准——**HotpotQA**（多跳）、**SQuAD v1.1 & v2.0**（阅读理解）、**MedQA**（医学）、**PubMedQA**（医学）。每个数据集随机采样200个样本，每样本包含问题、真实答案、包含答案的上下文。
- **场景与基准**：
  - **白盒设置**（RQ1-RQ3）：攻击者拥有本地相同代理模型。
  - **黑盒设置**（RQ4）：在代理模型（Llama-3.1-8B）上优化前缀，再攻击不同目标模型。
  - **对比方法**：
    - `Full`：不使用KV缓存（全重算）。
    - `Vanilla`：完全位置无关复用，不重算。
    - `Random`, `EPIC`, `CacheBlend`：三种选择性重算策略（重算比例默认10%）。
- **评估指标**：
  - **Accuracy (Acc)**：模型在正常缓存复用下的任务准确率。
  - **Untargeted ASR (U-ASR)**：攻击是否改变了模型输出（\(y \neq x\)）。
  - **Targeted ASR (T-ASR)**：攻击是否成功使模型产生指定恶意答案（\(y = \tilde{r}\) 且 \(y \neq x\)）。
- **默认配置**：缓存比例 \(\delta = 0.3\)，重算比例 \(\rho = 0.1\)，块大小 \(L_{\text{chunk}} = 32\)。

## 6. 主要结论与发现

- **有效性**：HIJACKKV在单次攻击中平均**T-ASR达94%**（白盒），即使在有重算的情况下（如CacheBlend/EPIC）平均仍>85%。
- **鲁棒性**：
  - **低缓存比例**（\(\delta=0.1\)）时平均T-ASR仍有67%（EPIC 53%）。
  - **高重算比例**（\(\rho=0.5\)）时平均T-ASR约38%-49%，攻击概率依然可观。
  - **块大小**从32到512变化不影响攻击成功率。
- **持久性**：即使插入1024个无关上下文令牌，平均T-ASR仍>55%；2048令牌后仍有约47%。
- **迁移性**：在**黑盒设置**下，从Llama-3.1-8B迁移到Llama-3.3-70B、Qwen3-8B/14B等模型，仍能获得约37%-44%的T-ASR（U-ASR约70%-90%）。但对小型模型（1B）目标成功率低（~13%）。
- **防御局限**：
  - 现有防御（RobustKV、CachePrune）即使去除50%的KV，T-ASR仅下降19%。
  - 常见KV压缩方法（H2O, StreamingLLM, SnapKV, PyramidKV）几乎不影响攻击（平均T-ASR 97%）。
  - 重算策略可缓解攻击但成本高昂：\(\rho=0.8\)时可降至11.5%，但计算成本增加4倍。
- **根本原因**：位置无关缓存重用解耦了KV与其因果上下文，使得数值偏差足以改变输出（20%偏差即可，而实际偏差可达50%）。

## 7. 优点

- **问题新颖性**：首次系统性地识别并形式化了位置无关KV重用机制带来的**安全威胁**，区别于以往隐私泄露和系统级缓存修改攻击。
- **攻击实用性**：无需系统级特权，仅通过常规用户查询实现，可应用于现有支持跨会话KV共享的服务（如vLLM、LMCache）。
- **实验全面性**：从有效性、鲁棒性、持久性、迁移性多个角度评估，覆盖多种数据集、模型、系统配置和防御策略，结论可靠。
- **方法设计巧妙**：利用GCG算法在离散令牌空间优化前缀，并通过模拟缓存命中流程计算损失，确保攻击欺骗性。
- **防御分析完整**：不仅展示了攻击威力，还评估了多种现有及潜在防御（重算、压缩、专用防御），并给出了安全设计建议（混合重算、偏差拒绝策略）。
- **开源与负责任披露**：公开源代码，并在发布前联系商业开发者进行负责任的披露。

## 8. 不足与局限

- **缓存命中依赖**：攻击成功的前提是目标文本块 \(\tilde{X}\) 能被受害者上下文包含并按位置无关方式命中。若 \(\tilde{X}\) 不常见或已被其他良性KV覆盖，攻击可能失败。论文提到可通过检索投毒（PoisonedRAG）增加出现概率，但未在实验中系统评估。
- **缓存占用冲突**：攻击者需要确保 \(\tilde{X}\) 的 KV 能被成功插入共享缓存，但现有策略（LRU、LFU）可能使攻击者难以覆盖高频良性缓存。论文仅初步实验了占用探测（单次81.4%准确，5次95.4%），但未深入评估动态竞争环境。
- **模型范围有限**：主要实验集中在8B规模的三个模型，对更大（如70B）仅做了迁移实验且T-ASR较低（37%）。对更小型模型的迁移目标准确率低（13%），实用性受限。
- **黑盒场景下成功率有限**：目标成功率约37%，仍低于白盒；攻击者需要额外努力（如集成多个代理模型）才能提升。
- **未计算资源开销**：优化前缀的GCG算法需要多次迭代（默认250步），每步需多次前向/反向传播和无害搜索，计算成本未量化。对于实时或大规模部署，攻击成本可能较高。
- **未考虑缓存替换策略影响**：实际系统（FIFO、LRU、LFU）下攻击的可实现时间窗口未被深入分析，仅简要讨论。
- **防御讨论尚浅**：虽然提出重算和偏差拒绝，但未能给出完整的安全KV重用设计方案，也未测试其在实际系统中的性能开销。
