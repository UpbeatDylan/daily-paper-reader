## 摘要
长视频问答要求模型在不重复处理同一视频的情况下随时间保留视觉证据。一种实用方法是将视觉语言模型的内部键值（KV）缓存存储每个视频块，并在查询时检索该状态。然而，独立缓存的视频块无法正确组合：每个块从局部旋转位置零开始预填充，因此简单的拼接会导致时间相位冲突，并丢失回答“发生了什么事件最先发生、事件发生频率或视频中哪些变化”等问题所需的全局顺序。本文提出ChronoStitch，一种无需训练的方法，用于组合独立存储的视觉KV记忆。该方法首先将存储的旋转后键重新定位到全局三轴多模态RoPE坐标系统，该坐标系统保留时间、高度和宽度结构。我们证明了一维标量重新索引在视觉token上几何不一致，因为它将帧内的空间顺序转化为虚假的时间位移。然后我们解决了位置修复后残留的内容差距：后续块最初编码时未关注前面块。因此ChronoStitch选择性地重新计算后续块中一小部分高偏差视觉token，同时允许它们关注组合后的缓存。在Qwen2.5-VL-3B和TempCompass的时间拆分上，ChronoStitch优于简单组合和仅位置变体，在事件排序准确性上有所提升，同时比完全联合重新预填充快3.3倍。

## Abstract
Long-video question answering requires a model to preserve visual evidence over time without repeatedly reprocessing the same video. A practical approach is to store the vision-language model's internal key-value (KV) cache for each video chunk and retrieve that state at query time. However, independently cached video chunks do not compose correctly: every chunk is prefilled from local rotary position zero, so naive concatenation collides temporal phases and removes the global order required for questions about what happened first, how often events occurred, or what changed across the video. This paper presents ChronoStitch, a training-free method for composing independently stored visual KV memories. The method first re-bases stored post-rotary keys onto a global three-axis multimodal RoPE coordinate system that preserves time, height, and width structure. We show why a one-dimensional scalar re-indexing is geometrically inconsistent for visual tokens because it turns spatial order within a frame into false temporal displacement. We then address the residual content gap left by positional repair: later chunks were originally encoded without attending to earlier chunks. ChronoStitch therefore selectively recomputes a small fraction of high-deviation later-chunk visual tokens while allowing them to attend over the composed cache. On Qwen2.5-VL-3B and the temporal split of TempCompass, ChronoStitch outperforms naive composition and position-only variants, improving event-ordering accuracy while running 3.3x faster than full joint re-prefilling.

# ChronoStitch 论文总结

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：长视频问答（long-video QA）要求模型在无需重复处理整个视频的情况下，保留随时间变化的视觉证据。直接使用文本字幕会丢失视觉细节，因此希望存储视觉语言模型的内部键值缓存（KV cache）来保全面向注意力的状态。
- **核心问题**：独立缓存的不同视频块（chunk）在组合时存在两个根本性障碍：
  1. **位置冲突**：每个块独立编码，旋转位置编码（RoPE）从局部零位置开始，直接拼接会导致时间相位错乱，丢失全局顺序（例如，“哪个事件先发生？”这类问题无法正确回答）。
  2. **内容缺失**：后续块编码时并未“注意”到前面的块，因此缓存缺少跨块因果注意力信息。
- **整体含义**：提出无训练的 ChronoStitch 方法，通过位置修复加选择性重计算来组合独立存储的视觉 KV 记忆，在不重新运行视觉塔的情况下实现长时域时间推理，提升效率与准确性。

## 2. 论文提出方法论

**核心思想**：双层修复——先通过全局三轴位置重基准修正位置误差，再通过少量 token 的重计算修正内容误差。

**关键技术细节**：

- **三轴多模态 RoPE（mRoPE）**：Qwen2.5-VL 等模型中，视觉 token 的位置坐标是三维的 (时间, 高度, 宽度)。每帧内的 token 共享时间坐标。
- **位置修复（Position Repair）**：
  - 将每个块存储的后旋转键（post-rotary keys）从局部坐标（从零开始）通过旋转差旋转到全局坐标，即应用增量旋转：
    \[
    \Delta_i = (p'_{\sigma(i)} - p_{\sigma(i)}) \theta_i, \quad R(\Delta) f(W_k x, p) = f(W_k x, p')
    \]
  - 需要是三轴操作，避免使用一维标量（会混淆空间与时间）。
- **内容修复（Content Repair）**：
  - 由于后块编码时未看到前块，位置修复不能恢复深层内容。ChronoStitch 选择性地重计算一小部分受影响最大的后块视觉 token。
  - 评分标准：浅层重计算时，层-1 键的变化幅度最大的 token 被认为缺失交叉注意最严重。
  - 选择 top ρ 比例的视觉 token（实验中 ρ ≈ 0.35，约整个组合缓存的 25%），让它们重新通过所有 Transformer 层，并允许它们注意整个组合缓存；其他 token 保持不变。
- **算法流程**：检索块 → 三轴坐标重基准 → 对存储键应用增量旋转 → 拼接缓存 → 计算关键偏差代理 → 选择高偏差 token → 重新前向（带完整注意力）→ 替换新键值 → 回答查询。

## 6. 主要结论与发现

- **位置修复是必要但不充分的**：三轴重基准在 layer-0 可完美重建联合预填充的键（误差 3.6e-8），但深层误差仍大；仅靠位置修复，顺序推理几乎没提升（甚至比 naive 差）。
- **内容修复是关键**：选择性重计算约 25% 的视觉 token（ρ=0.35）后，顺序探针达到 100% 准确率，TempCompass 总体得分提升至 54.1%，较 naive 提升 4.8 分，事件顺序子集提升 7 分。
- **一维标量重基准是错误的**：其几何不一致，导致比 naive 更差的结果。
- **效率显著**：ChronoStitch 比全联合预填充快 3.3×（平均 748ms vs 2411ms）。
- **有效性**：长时域时间推理的提升主要来自修复跨块注意力缺失，而非仅仅位置对齐。

## 7. 优点

- **无训练（Training-Free）**：无需额外训练或微调模型，即插即用。
- **理论清晰**：明确区分了位置误差和内容误差两个独立问题，并分别提出了解决方法。
- **高效实用**：仅重计算少量 token，注意力复杂度从 O(T²) 降至约 O(ρT²) + O(|C|T)，且无需重跑视觉塔。
- **评估全面**：包含诊断性探针、标准基准、效率测量，且对三个政策进行了严格消融。
- **数学严谨**：深入分析了多模态 RoPE 的三轴性质，并用公式证明了一维标量重基准的几何错误。

## 8. 不足与局限
