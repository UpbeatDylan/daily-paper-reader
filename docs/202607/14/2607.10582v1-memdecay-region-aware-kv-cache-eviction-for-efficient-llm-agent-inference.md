## 摘要
大语言模型（LLM）智能体积累异构上下文，包括系统指令、计划、用户轮次、检索文档、工具输出和中间推理，其键值（KV）缓存可能成为主要的内存瓶颈。现有的驱逐策略通常对每个令牌应用相同的基于注意力或近期性的规则，忽略了智能体编排器已经可用的语义结构。我们提出MemDecay，一种免训练、区域感知的KV缓存驱逐策略。MemDecay为令牌分配区域特定的基础优先级和衰减率，当令牌受到关注时刷新保留分数，并在固定缓存预算下驱逐得分最低的页面，同时允许关键区域被固定。我们还提供了一种从测量的注意力生命周期中校准衰减率的方法。我们在约450和1700令牌上下文中使用Qwen2.5-1.5B和3B评估了MemDecay。在所有设置中，各区域的注意力生命周期相差一个数量级：系统令牌的半衰期范围为148到189个解码步骤，而草稿令牌为14到16个。固定系统区域事实在每种设置下都能达到完整缓存的准确率，而没有任何基线能保留24个中的13个以上。区域感知保留在上下文增长时仍然有效，而基于近期性的保留则崩溃。然而，累积注意力保留在未固定内容上表现更好，消融实验将注意力分数归一化确定为当前公式的主要局限性。这些结果确立了语义提示结构作为KV缓存管理的稳健信号，同时澄清了它应如何与基于注意力的重要性相结合。

## Abstract
Large language model (LLM) agents accumulate heterogeneous context, including system instructions, plans, user turns, retrieved documents, tool outputs, and intermediate reasoning, whose key-value (KV) cache can become a major memory bottleneck. Existing eviction policies generally apply the same attention- or recency-based rule to every token, ignoring semantic structure already available to the agent orchestrator.   We introduce MemDecay, a training-free, region-aware KV-cache eviction policy. MemDecay assigns tokens region-specific base priorities and decay rates, refreshes retention scores when tokens receive attention, and evicts the lowest-scoring pages under a fixed cache budget while allowing critical regions to be pinned. We also provide a procedure for calibrating decay rates from measured attention lifetimes.   We evaluate MemDecay at approximately 450 and 1,700 token contexts using Qwen2.5-1.5B and 3B. Across all settings, attention lifetimes differ by an order of magnitude across regions: system-token half-lives range from 148 to 189 decoding steps, compared with 14 to 16 for scratchpad tokens. Pinning preserves system-region facts at full-cache accuracy in every setting, while no baseline preserves more than 13 of 24. Region-aware retention remains effective as context grows, whereas recency-based retention collapses. Accumulated-attention retention performs better on unpinned content, however, and ablations identify attention-score normalization as the main limitation of the current formulation. These results establish semantic prompt structure as a robust signal for KV-cache management while clarifying how it should be combined with attention-based importance.

- **现有方法不足**：现有KV缓存驱逐策略通常对所有令牌应用统一的基于注意力或近期性的规则，忽略了智能体上下文中不同语义区域的固有差异。例如，系统指令需要长期保留，而临时推理步骤可以快速丢弃。
- **论文核心问题**：如何利用上下文中已经存在的语义区域结构（由编排器提供）来设计更高效的、无训练的KV缓存驱逐策略，同时与注意力重要性信号结合。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：提出MemDecay，一种无训练、区域感知的KV缓存驱逐策略。它为每个语义区域（如system, plan, user, tool_in, tool_out, retrieval, scratchpad）分配基础优先级（\(b_r\)）和衰减率（\(\lambda_r\)），并根据注意力刷新来重置衰减时钟。在固定缓存预算下，优先驱逐得分最低的非固定页面，允许关键区域（如系统指令）被固定永久保留。
- **关键技术细节**：
  - **区域标签**：假设编排器为每个令牌提供区域标签，对应连续的提示片段（prompt spans），因此可以与现有服务栈中的页面粒度缓存布局对齐。
  - **保留分数公式**：每个令牌\(i\)在时间\(t\)的保留分数\(s_i(t) = b_{r(i)} \exp(-\lambda_{r(i)} \Delta t_i(t)) + \alpha a_i(t)\)，其中\(\Delta t_i(t)\)是自上次被注意力刷新以来的时间（或插入时间），\(a_i(t)\)是通过指数加权移动平均（EWMA）更新的注意力重要性，\(\alpha\)是注意力权重。
  - **刷新机制**：当令牌的注意力高于阈值\(\tau\)时，其衰减时钟重置为0，即“刷新”。
  - **页面级驱逐**：生产服务使用固定大小的页面（如16个令牌）。页面得分取成员令牌平均分。当缓存超过预算时，从最小堆中逐出得分最低的非固定页面，直到满足预算。
  - **衰减率校准**：论文提供一种测量协议，从智能体轨迹中估计区域条件注意力寿命，再用对数线性模型拟合衰减率，从而从数据中校准默认配置。

## 6. 论文的主要结论与发现

- **区域寿命层次稳定**：在所有设置中，系统区域半衰期148–189步，草稿区域14–16步，相差一个数量级。检索区域是寿命最长的可移动区域（半衰期48–76步），推翻了默认快速衰减的假设。
- **固定系统区域保证指令存活**：MemDecay在固定系统区域后，在短层级和长层级半预算下分别达到24/24和21/24系统探针召回率（等于全缓存上限），而流式方法只能保留0–2/24。
- **区域感知保留在长上下文中有效**：流式方法（基于近期性）在长上下文中崩溃（召回率接近0），而MemDecay保持可工作水平（如25%预算下0.38总召回，1.5B长层级）。
- **累积注意力（H2O）在未固定内容上表现更好**：在长层级半预算下，H2O的非系统召回率达到0.53（1.5B）和0.60（3B），而MemDecay仅为0.28和0.22。主要失败原因是用户区域（最早的非固定事实）被结构先验排名驱逐，而注意力信号恰好识别出这些令牌。
- **注意力项需归一化**：消融实验显示，当前注意力EWMA项（\(\alpha a_i\)）的幅度远低于结构先验（因页面平均和衰减导致），无法拯救旧令牌。结论是需要对注意力项进行幅度归一化（例如相对于均匀注意力水平），而不是调整权重。

- **创新性**：首次提出利用编排器提供的区域结构信息进行KV缓存驱逐，将语义角色与时间衰减结合，设计简洁、免训练、可解释。
- **实用性**：策略与现有页面粒度缓存布局（如PagedAttention）兼容，可通过简单校准适应不同工作负载，无需修改模型权重。

- **诊断清晰**：通过消融实验精确定位了注意力项幅度归一化问题，而非简单报告失败，为后续改进提供了明确方向。

## 8. 不足与局限

- **实验规模有限**：模型仅为1.5B和3B，上下文长度仅为450和1,700令牌，远小于生产级场景（如4k、32k）。未在更大模型（8B+）和更长上下文上验证。
- **区域标签依赖编排器**：策略假设编排器提供区域标签，若需在无编排器的系统中使用，需要额外的分类器，而标签噪声的鲁棒性未评估。
- **不可逆性**：基础策略一旦驱逐令牌就无法恢复，当后续查询使旧令牌再次相关时会丢失信息（用户区域失败即为此类实例）。作者提出了分级降级扩展（量化→卸载→丢弃），但未实现。
- **跨区域信息流**：智能体会跨区域复制内容（如在推理中引用工具输出），但两个副本都快速衰减，信息可能丢失。
- **位置混淆**：系统、计划和草稿区域固定位置，因此其寿命估计与位置效应无法分离。
- **单任务评估**：仅使用检索驱动的智能体场景，未涵盖编码、函数调用、多轮浏览等真实工作负载（如SWE-bench, BFCL）。仅评估问答召回，未评估生成质量或端到端任务成功率。
- **注意力计算开销**：当前所有方法（包括H2O）都需读取注意力权重，导致解码步骤减速（在M3 Pro上从26.0ms增至69.8ms）。生产环境中需采用稀疏采样（k>1）或利用现有分数，但论文未在此方向实验。
- **半精度影响检查**：论文仅对数值精度做了0.001%以下影响的审计，但未检查3B在T4上float16模式下的潜在质量退化。
