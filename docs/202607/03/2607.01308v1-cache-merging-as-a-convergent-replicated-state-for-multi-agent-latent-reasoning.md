## 摘要
多智能体潜在推理将各智能体的KV缓存合并为一个上下文供最终智能体使用。先前的工作（Agent Primitives）通过沿序列轴拼接缓存并应用RoPE重编码来实现此操作，我们称之为BagMerge。BagMerge是非交换的，最佳输入顺序不可预测，会随条件、潜在步骤预算和模型规模而变化。我们将这种交换转变为收敛复制状态。首先，CanonicalMerge通过内容固定布局：按中间层的平均K-范数对缓存排序，使得合并后的缓存在任何输入排列下字节相同，通过算法验证（arity N<=5）并在实际Qwen3-1.7B和4B状态上逐位验证。其次，我们将复制状态与解码时布局分离：状态是一组内容可寻址的潜在片段，其合并为集合合并，是一种基于状态的CvRDT（可交换、可结合、幂等、吸收），而CanonicalMerge是其确定性渲染。由于渲染字节等价，每个N=2的准确率数字不变，重复交付的重复项被吸收而非重新拼接。在分区推理基准上，CanonicalMerge在每个预算-排序单元的每个条件下匹配最佳BagMerge排序，无需知道哪个顺序最佳，以微小且统计上不显著的准确率差距换取无条件的结构保证。该行为迁移到真实的多文档问答（HotpotQA），而最接近的无训练输出融合基线（PackLLM）在相同预算下损失45个点，将缓存级合并置于与输出级融合不同的范式中。最后，当k>2时，该方法传输并放置潜在痕迹，但本身不组合它们，我们对此进行表征以推动未来工作。

## Abstract
Multi-agent latent reasoning composes agents' KV-caches into one context for a final agent. Prior work (Agent Primitives) does this by concatenating caches along the sequence axis with RoPE re-encoding, which we call BagMerge. BagMerge is non-commutative, and the best input ordering is unpredictable, shifting with the regime, the latent-step budget, and the model scale. We make this exchange a convergent replicated state. First, CanonicalMerge fixes the layout by content: ordering caches by mean K-norm at a middle layer renders the merged cache byte-identical under any input permutation, verified algorithmically (arity N<=5) and bit-for-bit on real Qwen3-1.7B and 4B state. Second, we separate the replicated state from decode-time layout: the state is a set of content-addressed latent fragments whose merge is set union, a state-based CvRDT (commutative, associative, idempotent, absorbing), and CanonicalMerge is its deterministic render. Because the render is byte-equivalent, every N=2 accuracy number carries over unchanged and re-delivered duplicates are absorbed rather than re-concatenated. On a partitioned-reasoning benchmark, CanonicalMerge matches the best BagMerge ordering in every regime-by-budget-by-ordering cell without knowing which order is best, trading a small, statistically insignificant accuracy margin for an unconditional structural guarantee. The behaviour transfers to real multi-document QA (HotpotQA), while the closest training-free output-fusion baseline (PackLLM) loses by 45 points at matched budget, placing cache-level merging in a regime distinct from output-level fusion. Finally, at k>2 the approach transports and colocates latent traces but does not by itself compose them, which we characterize to motivate future work.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **问题**：多智能体潜在推理中，各智能体将KV缓存合并供最终智能体使用。当前方法（BagMerge，即按序列轴拼接并RoPE重编码）是非交换的，即输入顺序影响解码结果；且最佳顺序不可预测，随部署模式、潜在步骤预算和模型规模变化，导致需要针对每场景手动验证并选择最佳顺序。
- **背景**：在联邦推理、查询盲推理等场景中，独立方可能以任意顺序、多次交付缓存，合并必须收敛到相同结果。BagMerge无法满足这些要求（对顺序敏感、重复项会重新拼接）。
- **核心含义**：将缓存交换建模为一种收敛复制状态（CvRDT），提供无交换的确定性合并和幂等吸收，同时保持实用准确率。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：
  - 分离复制状态与解码时的布局：状态是一组内容可寻址的潜在片段，合并为集合合并（CvRDT）；渲染时使用CanonicalMerge产生字节相同的确定性布局。
- **CanonicalMerge**：
  - 输入：N个智能体的KV缓存。
  - 计算每个缓存在中间层（如第15层）最后T*个位置的K向量的平均L2范数。
  - 按范数降序排序（并列时使用全内容哈希作为确定性断平器），得到规范顺序π<sup>*</sup>。
  - 按该顺序应用BagMerge（即重新拼接并重新旋转RoPE位置）。
  - 结果：输入顺序的任意排列产生的合并缓存字节相同（算法验证N≤5，实际模型验证）。
- **LatentFragmentSet（CvRDT）**：
  - 状态S是每个片段（未移位原始KV缓存）与其内容哈希的无序集合。
  - 合并操作为集合合并 ∪（可交换、可结合、幂等、吸收）。
  - 渲染函数是CanonicalMerge，作用于集合中的片段。
  - 重复交付的字节相同片段被吸收（不增加缓存长度），准确率不变。
- **关键技术细节**：
  - 路由层ℓ<sup>*</sup>是唯一超参数（Qwen3-1.7B选15，Qwen3-4B选24，通过粗扫选择）。
  - RoPE布局仅在渲染时引入，保证状态级幂等性。

## 6. 论文的主要结论与发现

- **结构保证**：CanonicalMerge产生的合并缓存在任意输入顺序下字节相同（N=2在1.7B和4B上验证）；LatentFragmentSet状态级重复吸收幂等，重复交付时准确率不变而朴素方法性能下降。
- **经验表现**：在12个单元的主矩阵中，CanonicalMerge在每个单元匹配最佳BagMerge顺序（平均差距+0.005，绝对差距0.025），无需预先知道哪个顺序最佳。统计检验表明与最佳顺序无显著差异，击败较差顺序。
- **与输出融合的区别**：PackLLM在相同预算下落后45个点（分类级融合无法恢复联合信息），证明缓存级合并是不同范式。
- **边界**：k>2时仅靠缓存合并无法实现组合，需要模型重处理（混合或顺序链）。该边界清晰。
- **跨基准迁移**：在HotpotQA上无退化（F1 0.605 vs 全文本0.610）。
- **预算鲁棒性**：CanonicalMerge在20步和40步间准确率基本不变（±0.01），而BagMerge变化大。

## 7. 优点

- **强结构保证**：输入顺序字节相同、重复吸收幂等，超越训练方法的经验稳定性。
- **无训练且轻量**：仅需一次前向传播并计算K-范数，无额外参数训练，易于集成到现有系统（PagedAttention兼容）。
- **实验设计严谨**：分区基准隔离了合并算子的贡献，多种子统计检验和Holm校正保证了结论可靠。

- **区分了结构性鲁棒性与经验性鲁棒性**，避免过度声明。

## 8. 不足与局限

- **模型系列单一**：仅验证Qwen3（1.7B和4B），未在更大模型（如8B/70B）或其他家族上测试，Agent Primitives报告在8B/70B上RoPE效应更大，结论可能不完整。
- **基准主要为合成**：虽然HotpotQA验证了迁移能力，但合成基准可能缺乏真实复杂性；未进行更大规模多文档问答（如MuSiQue、完整HotpotQA）评估。
- **字节相同性依赖确定性硬件**：交叉平台异构硬件下低阶位差异可能破坏内容哈希吸收；仅声明了相同硬件上的字节相同性。
- **重复吸收仅为语法级**：语义近似的不同片段不会合并；真正的语义合并留待未来。
- **k>2组合问题未解决**：缓存合并仅运输不组合，需要模型重处理才能恢复性能，这在效率上可能不可取。
- **文本级基线未对比**：未与直接拼接文本的RAG或FiD比较，尽管这类方法不提供结构保证。
- **渲染延迟**：每次新片段吸收需要O(m)重渲染，批推理可接受但可能影响流式场景。
- **路由层选择通过粗扫**：虽有一定鲁棒性，但仍需要对不同模型架构重新选择。
