## 摘要
设备端大语言模型推理面临响应延迟、硬件资源有限和用户隐私的三难困境。全云端推理提供强大计算能力，但暴露用户提示和对话数据，而纯设备端推理对大多数消费级和嵌入式边缘设备不可行。本文提出一种基于端点认证KV缓存的以隐私为中心的边云协同LLM推理框架。本地端点处理输入预处理、嵌入计算、自适应特征优化、KV缓存认证、推测解码和低维度模型头计算，而云端执行认证解码器推理、KV缓存管理、令牌验证和高维度词汇投影。端点融合部分输出，应用语言自适应掩码并采样目标令牌。所有传输数据和截断的logits均经量化并使用AES-GCM加密以保护隐私，核心轻量模块、草稿参数和缓存访问策略保留在本地以避免泄露。该框架通过优化流式处理、批处理和量化ONNX部署支持异构设备，包括仅CPU、配备GPU和嵌入式设备。评估表明，与基准拆分推理相比，该框架将每令牌延迟降低高达46.1%，下行负载降低高达67.4%，同时保持与全云端推理相当的性能。

## Abstract
On-device LLM inference faces a trilemma of response latency, limited hardware resources and user privacy. Full cloud inference delivers strong computing power but exposes user prompts and dialogue data, while standalone on-device inference is unfeasible for most consumer and embedded edge devices. This paper presents a privacy-centric edge-cloud collaborative LLM inference framework built on endpoint-authenticated KV cache. Local endpoints handle input preprocessing, embedding computation, adaptive feature optimization, KV cache authentication, speculative decoding and low-dimensional model head calculation, while the cloud conducts authenticated decoder inference, KV cache management, token verification and high-dimensional vocabulary projection. Endpoints fuse partial outputs, apply language-adaptive masking and sample target tokens. All transmitted data and truncated logits are quantized and AES-GCM encrypted for privacy, with core lightweight modules, draft parameters and cache access policies kept local to avoid leakage. The framework supports heterogeneous devices including CPU-only, GPU-equipped and embedded devices via optimized streaming, batching and quantized ONNX deployment. Evaluations demonstrate that the framework reduces per-token latency by up to 46.1\% and downlink payloads by up to 67.4\% over baseline split inference, retaining comparable performance to full cloud inference.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **三难困境**：大语言模型（LLM）在用户直面设备（如手机、PC、嵌入式）上部署时，面临 **交互延迟**、**本地计算资源有限** 和 **用户隐私暴露** 三者之间的矛盾。
    - 纯云端推理：计算能力强，但会暴露原始提示、对话历史等敏感信息，且产生网络延迟。
    - 纯设备端推理：对消费级CPU、移动GPU及嵌入式设备不可行（算力、内存不足）。
- **现有工作局限**：已有的拆分推理（split inference）缺乏对KV缓存的细粒度控制、设备自适应能力和联合保护策略。
- **论文目标**：设计一种 **以隐私为第一级设计目标** 的边云协同推理框架，在降低延迟、减少通信量的同时，将隐私暴露最小化，并支持CPU、GPU、嵌入式等异构端点。

## 2. 论文提出的方法论

### 核心思想
将推理流程分割为**隐私敏感且计算轻量的部分**（本地端点）和**计算密集的解码部分**（云端），并通过 **端点控制的KV缓存授权**、**分布式词汇投影**、**推测解码** 和 **设备自适应传输** 实现高效协同。

### 关键技术细节
- **整体工作流**：
    1. 端点完成：输入预处理（token化）、嵌入计算、生成控制元组（缓存授权标志、语言标识等）、设备自适应特征准备、本地轻量模块推理。
    2. 端点发送量化+加密的特征张量和控制元组到云端。
    3. 云端执行：授权解码器推理（根据端点指令使用KV缓存）、KV缓存维护、推测令牌验证、计算高维部分logits。
    4. 云端返回量化+加密的隐藏状态或logits子集。
    5. 端点：融合部分logits（低维本地+高维云），应用语言自适应掩码，采样目标令牌，执行本地推测解码。
- **设备自适应特征准备**：
    - **纯CPU端点**：按列分块传输隐藏状态（`Nb` block），块流式传输与解密/计算重叠。
    - **GPU端点**：支持模型切片、张量并行、批处理。
    - **边缘设备**：导出为ONNX格式，通过ONNX Runtime执行，支持8-bit量化。
- **授权云解码推理**：
    - 端点通过控制元组 `(flag_cache, L_cache, dtype, lang)` 指定是否允许使用历史KV缓存以及最多使用多少条目。
    - 云端据此执行自注意力计算（公式4-8），并返回隐藏状态。
- **分布式词汇投影**：
    - 将LM-head的隐藏维度分割为低维端点部分（`Dend`）和高维云端部分（`Dcloud`），`Dend + Dcloud = D`。
    - 端点计算 `zend = hend * Wend`，云端计算 `zcloud = hcloud * Wcloud + b`（bias只加一次）。
    - 端点通过元素级加法融合得到完整logits：`z = zend + zcloud`。
    - **语言自适应logits压缩**：若指定单语言生成（如中文、英文），云端只返回该语言子集 `Vℓ` 的logits，压缩比可达30%~40%。
- **端点推测解码与云端验证**：
    - 端点使用轻量草稿模块 `gϕ`（MLP或小Transformer）预测未来N个令牌。
    - 云端使用目标解码器验证，接受则提交并更新KV缓存，拒绝则回退重算。
    - 草稿长度N设备相关（CPU/边缘短，GPU长）。
- **传输保护与隐藏状态保护**：
    - 所有张量先量化到配置比特宽度（默认8-bit），再使用 **AES-GCM** 加密传输。
    - 端点本地**LoRA模块**的权重、草稿模块参数、本地投影权重**不传输到云端**，减少云端重建风险。
    - KV缓存授权由端点控制，可动态缩减暴露的历史上下文。

## 3. 实验设计

### 使用场景与数据
- **模型**：7B类解码器Transformer，隐藏大小 `D=4096`，词汇量 `V=150,000`。
- **端点配置**：
    - CPU端点：12核x86 CPU，32GB RAM，无GPU，`Nb=4` 块流，LoRA rank=4。
    - GPU端点：消费者8GB VRAM GPU，32GB RAM，`Dend=96`，LoRA rank=8，草稿长度 `N=5`。
    - 边缘端点：ARM嵌入式设备，8GB统一内存，8-bit ONNX Runtime，草稿长度 `N=2`。
- **网络**：模拟20ms RTT，2ms抖动，100Mbps带宽。
- **提示集**：1000个提示（400中文，400英文，200混合），每个截断/填充至512输入令牌，生成128个响应令牌。
- **质量评估**：独立的下一令牌一致率、困惑度（PPL）、MMLU（5-shot）、GSM8K（贪婪链式思维）、HumanEval pass@1。

### 基准方法 (Baseline)
1. **Cloud-only**：端点发送原始提示与会话ID，云端完成所有推理，返回文本令牌。
2. **Cloud-only encrypted transport**：同上但传输加密，仍暴露云端可见内容。
3. **Naive split inference**：端点只计算嵌入，云端返回隐藏状态或完整logits，无缓存控制、无语言压缩、无推测解码。
4. **Endpoint-only lightweight**：仅端点使用一个更小的本地模型独立生成，仅用于GPU/边缘场景的质量对比。

### 对比指标
- 平均每令牌解码延迟 (ms)、端到端128令牌响应延迟 (s)、吞吐量 (令牌/秒)、下行每令牌负载 (KB)、推测接受率、生成质量分数、隐私暴露审计。

## 6. 论文的主要结论与发现

- **延迟降低**：与Naive split inference相比，每令牌延迟**减少26.8%~46.1%**（GPU端点最明显），128令牌响应时间分别缩短29.4%（CPU）、46.1%（GPU）、26.8%（边缘）。
- **通信压缩**：语言自适应logits压缩减少下行负载**56.2%~67.4%**（单语言模式），混合模式时无压缩。
- **生成质量接近**：下一令牌一致率98.6%，PPL、MMLU、GSM8K、HumanEval得分与全云模型相差在0.5分以内；本地轻量模型则显著下降（一致率84.2%）。
- **组件有效性**：消融表明推测解码贡献最大延迟降低，语言压缩贡献最大带宽节省，分布式投影和CPU块流均有明显收益。
- **隐私暴露减小**：端点本地保留LoRA权重、草稿参数，传输过程中无原始文本、无完整logits，缓存由端点控制——比纯云和朴素拆分更安全（但非正式差分隐私）。

## 7. 优点

1. **隐私设计作为一等公民**：将隐私控制权赋予端点，而非仅依赖传输加密；通过本地轻量模块和缓存授权降低隐藏状态的可利用性。
2. **分布式词汇投影**：创新性分割LM-head，结合语言自适应压缩，显著减少通信量并保持端点部分贡献。
3. **设备自适应策略**：针对CPU、GPU、边缘分别设计执行路径（块流、切片、ONNX），使框架具有广泛实用性。
4. **集成推测解码**：在协同环境中利用端点草稿减少交互轮次，加速生成。
5. **全面的实验**：包括延迟、吞吐、通信、质量、消融、敏感性、隐私审计，对比多种基线，结论可信。
6. **文档详尽**：提供了清晰的算法描述、威胁模型、设计目标和部署策略，易于复现与扩展。

## 8. 不足与局限

1. **无正式隐私保证**：框架未提供差分隐私或安全多方计算（SMC）等可证明的隐私保护，隐藏状态在云端可见，理论上可被强敌手进行重建攻击（文中承认此点）。
2. **实验环境简化和规模有限**：

    - 只使用7B模型和128令牌生成，未验证更大模型（如13B/70B）或更长上下文的影响。
    - 端点类型覆盖不够广泛（如未测试智能手机SoC、不同品牌GPU）。

5. **静态缓存控制**：缓存授权仅由端点决定，但未讨论动态调整策略（如根据对话敏感度自适应）。

7. **未讨论多用户并发场景**：实验仅运行单客户端，未评估云端多会话压力下的可扩展性。
