## 摘要
基于注意力的KV缓存驱逐策略（H2O及其后续方法）通过按累积注意力质量（此处视为信号能量）对标记进行排序并保留能量最高的标记，来压缩长上下文模型的内存受限状态。在模式密集的输入流（如嵌套JSON）上，该得分充当非平稳滤波器，会不成比例地保留噪声：非内容角色（分隔符或空白）的能量比任何内容角色高出一个数量级，而结构性的KEY标记被过度保留的比率大约是承载答案的VALUE标记的1.8倍，导致在5%的预算下精确匹配准确率从88%降至0%，因为保留状态的信噪比恶化。一项反事实实验表明，抑制KEY标记是最佳的可部署过滤器。我们无需重新训练，基于角色条件分配的方法作用于SnapKV的窗口得分，由单个调整的超参数控制，可在低于20%的预算下缩小H2O差距的63-98%，在更高预算下适度达到或超过完整缓存的准确率——这是一种小规模、对种子敏感的降噪效果（在B=0.50时边界显著；在B=0.30时在四个种子上与零不可区分）。一个15MB的线性角色探针以可忽略的推理成本提供这些标签，但匹配解析器级的下游准确率仍然是开放问题。

## Abstract
Attention-based KV cache eviction (H2O and its descendants) compresses the memory-constrained state of a long-context model by ranking tokens on accumulated attention mass, treated here as signal energy, and keeping the heaviest. On schema-dense input streams such as nested JSON, this score acts as a non-stationary filter that disproportionately retains noise: a non-content sink role (delimiters or whitespace) carries an order of magnitude more energy than any content role, and structural KEY tokens are over-retained at roughly 1.8x the rate of the answer-carrying VALUE tokens, collapsing exact-match accuracy from 88% to 0% at a 5% budget as the signal-to-noise ratio of the retained state degrades. A counterfactual experiment establishes that suppressing KEY tokens is the best deployable filter. Our retraining-free, role-conditional allocation over SnapKV's windowed score, governed by a single tuned hyperparameter, closes 63-98% of the H2O gap at sub-20% budgets and, at higher budgets, modestly matches or exceeds full-cache accuracy -- a small, seed-sensitive denoising effect (borderline significant at B=0.50; not distinguishable from zero at B=0.30 over four seeds). A 15 MB linear role probe supplies these labels at negligible inference cost, though matching parser-level downstream accuracy remains open.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **问题背景**：长上下文 LLM 推理中，KV 缓存驱逐（如 H2O）通过累积注意力质量（信号能量）保留“重头” token，压缩内存。但在模式密集的输入（如嵌套 JSON、XML）上，这种评分机制形成非平稳滤波器，错误地将结构噪声（分隔符、空白、KEY token）视为高能量信号，而承载答案的 VALUE token 因能量低被过早驱逐。
- **核心问题**：注意力质量与答案相关性解耦，导致在 5% 预算下精确匹配准确率从 88% 骤降至 0%，信噪比严重恶化。论文将这种现象诊断为“结构角色偏差”（structural-role bias），即缓存驱逐对高能量结构 token 的过度保留。
- **整体含义**：现有训练无关的驱逐策略在结构化数据工作负载上存在根本性失效，需要一种基于 token 语义角色而非纯能量排序的自适应过滤方法。

## 2. 论文提出的方法论

**核心思想**：在现有基于窗口质量评分（SnapKV）的基础上，引入角色条件分配（role-conditional allocation），根据 token 的语义角色（KEY、VALUE、DELIM、PROSE、WS）差异化分配缓存预算，抑制 KEY token 的过度保留，同时保护 VALUE 和 DELIM 等关键角色。

**关键技术细节**：
- 沿用 SnapKV 的窗口质量评分：\( s(t) = \sum_{q \in \text{last }W} p_t(q) \)，其中 \( W=64 \)，再经过 1D 最大池化（核大小 k=7）平滑。
- 角色标签由结构解析器（实验中）或线性角色探针（部署时）提供。
- 分配算法（Algorithm 1）：
  1. 为每个 (layer, head) 的每个角色桶 \( r \) 保留前 \( \lceil \alpha_r B n_{kv} \rceil \) 个 token（按平滑后的分数排序）。
  2. 角色权重 \( \alpha \) 固定为：VALUE=0.70, DELIM=0.20, PROSE=0.05, WS=0.05，仅 KEY 的权重 \( \alpha_{\text{KEY}} \in [0, 0.2] \) 为可调超参数（推荐默认 0.02）。
  3. 饱和桶的未用预算重新分配。
- 推理时通过 keep-mask 解码，不压缩张量以隔离精度影响。

**公式/算法文字说明**：无需公式，核心是“窗口评分 + 角色比例分配 + 单超参数调优”。

## 6. 论文的主要结论与发现

1. **结构角色偏差普遍存在**：H2O 在密集结构化输入中过度保留 DELIM 和 KEY（KEY 保留率为 VALUE 的 1.8 倍），导致准确率崩溃。
2. **反事实证实抑制 KEY 最优**：强行驱逐所有 KEY token 可以恢复大部分性能，而驱逐 DELIM 或 VALUE 则完全失效。
3. **SnapKV 的增益主要来自隐含的 sink 抑制**：其窗口池化机制（max-pool）优于窗口质量本身，可消除部分噪声。
4. **角色条件分配与 SnapKV 超加性组合**：在 5% 预算下组合方法缩小 H2O 差距 63%，优于单独使用任一方法（21% 和 0%）。
5. **跨模型通用性**：KEY-over-VALUE 偏差在所有四个模型上均成立，但超加性组合仅对 Llama 和 Mistral 显著（Phi-3 和 Qwen 已被 SnapKV 池化自动清洁）。
6. **高预算下的微弱去噪效果**：seed 0 时组合方法在 B=30%/50% 超过全缓存 10/8 EM，但多种子下均值降至 +1% 和 +3.5%（B=50% 边界显著）。
7. **线性探针标签不足**：15MB 探针的孤立准确率 0.9952，但下游 EM 下降 18-32 点，因缺乏块级上下文导致问题路径的参考 token 被错误标记为 KEY 并被驱逐。
