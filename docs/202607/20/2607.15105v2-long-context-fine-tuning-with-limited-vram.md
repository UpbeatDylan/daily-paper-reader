## 摘要
参数高效微调减少了模型和优化器的内存占用，但密集注意力机制仍然使得长训练序列成本高昂。我们结合了层次化全局注意力（HGA）、分段反向传播和分层KV存储。只有当前活跃段在显存中保持可微；较早的KV被卸载到RAM或NVMe中，HGA为每个查询块加载一组有界的历史精确标记。在使用4位QLoRA和PG19的Qwen3-8B模型上，16 GB Quadro RTX 5000的密集训练只能容纳2,048个标记，在4,096个标记时失败，而HGA在15.28 GB峰值显存下达到了16,384个标记。在评估中，相同的适配器在该显卡上处理了131,072个标记；显存消耗并非恒定，而是随驻留的分块摘要缓慢增长，因此RAM和NVMe容量设定了超过这些长度的实际限制。在共享的2K训练长度下，使用相同密集注意力读取方式，HGA训练和密集训练的适配器分别获得2.7405和2.7383 nat，而原始模型获得2.9541 nat。在这个边界上，HGA训练已经略微更快（217.75 vs. 207.02 tokens/s），且HGA相对于密集的吞吐量比从1K到2K有所提升；由于HGA使每个标记的注意力历史集大致恒定，而密集注意力每个标记的计算量随上下文增长而增加，我们预计这一领先优势将随着上下文的扩大而扩大。主要质量和检索对比中使用了密集注意力，以便衡量学习到的权重，并与标准生成框架保持兼容。HGA也可用于检索和生成；一个优化后的生产级服务实现正在开发中。

## Abstract
Parameter-efficient fine-tuning reduces model and optimizer memory, but dense attention still makes long training sequences expensive. We combine Hierarchical Global Attention (HGA) with segment-wise backpropagation and tiered KV storage. Only the active segment remains differentiable in VRAM; older KV is detached into RAM or NVMe, and HGA loads a bounded set of exact historical tokens for each query block. On Qwen3-8B with 4-bit QLoRA and PG19, dense training on a 16 GB Quadro RTX 5000 fits 2,048 tokens but fails at 4,096, whereas HGA reaches 16,384 tokens with 15.28 GB peak VRAM. Under evaluation the same adapter runs through 131,072 tokens on this card; VRAM is not constant but grows gently with the resident chunk summaries, so RAM and NVMe capacity set the practical limit beyond these lengths. At the shared 2K training length, HGA-trained and dense-trained adapters obtain 2.7405 and 2.7383 nat under the same dense-attention readout, while the stock model obtains 2.9541. At this boundary HGA training is already marginally faster (217.75 vs. 207.02 tokens/s), and the HGA-to-dense throughput ratio improves from 1K to 2K; because HGA keeps the attended historical set per token approximately constant while dense work per token grows, we expect this lead to widen as context grows. Dense attention is used for the main quality and retrieval comparisons so that they measure the learned weights and remain compatible with standard generation frameworks. HGA can also be used for retrieval and generation; an optimized production-grade serving implementation is under development.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：参数高效微调（如 QLoRA）虽然大幅降低了模型权重和优化器的显存占用，但模型中的**密集注意力（dense attention）**依然需要处理完整的上下文历史，导致长序列训练时显存（VRAM）不足。
- **研究动机**：在有限 VRAM（如 16 GB）的消费级 GPU 上，长上下文微调受到注意力计算和反向传播状态而非可训练参数数量的限制。
- **整体含义**：提出一种**分层全局注意力（Hierarchical Global Attention, HGA）**，结合分段反向传播（TBPTT）和分层 KV 存储，将 GPU 工作集与完整训练历史解耦，从而在**单卡 16 GB VRAM** 上将训练长度从 2K tokens 扩展到 16K tokens，推理可达 131K tokens，且质量与密集训练几乎无异。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- **分段训练**：将长序列切分为固定长度（如 2K）的段，每段独立前向-反向，历史 KV 在段边界后脱离梯度追踪。
- **层级路由**：不是将全部历史 KV 送入注意力，而是通过两层摘要（chunk 摘要、group 摘要）选择与当前查询块最相关的少量精确 token KV，从而将注意力计算量维持在常数级别。
- **分层存储**：
  - **VRAM**：存放量化模型+适配器、当前活跃可微分段、所有 chunk 摘要、部分 group 摘要缓存。
  - **RAM / NVMe**：存放历史段的完整 KV（第一层）；第二层为分页缓存；第三层为磁盘冷存储。

### 关键技术细节
1. **精确 token 分层路由**（图1）：
   - 上下文分为 64-token chunks，每个 chunk 再分为 8-token groups。
   - 第一层：用紧凑的 chunk 摘要（始终在 VRAM）与当前查询打分，选出最相关的 top-k chunks（加上 sink 和 local 窗口）。
   - 第二层：在选中的 chunk 内，用 group 摘要（VRAM 缓存，未命中时从 RAM/NVMe 取）选择最佳 groups。
   - 第三层：仅将选中 groups 的精确 token K/V 加载到路由工作集，进行全注意力计算。
   - 路由选择过程**不追踪梯度**，但 K/V 收集和精确 token 注意力保持可微。
2. **分段反向传播与分层 KV 存储**（图2）：
   - 序列长度 L 分成若干长度 S 的段。每段完成前向和反向后，该段的历史 K/V 被分离（detach）。
   - VRAM 占用近似：  
     M_GPU ≈ M_model + M_adapter/optimizer + O(S) + O(B_route)  
     其中 B_route 为路由预算，外部历史记录增长为 O(L)。
3. **与密集训练的对称性**：
   - 适配器权重（Q/K/V/O 投影）在密集和 HGA 训练中完全一致，仅注意力计算方式不同。
   - 评估时使用密集注意力读取，以纯净地衡量学习到的权重。

## 6. 论文的主要结论与发现

1. **扩展性**：HGA 在 16 GB GPU 上将可训练上下文从 2K 扩展到 16K（峰值 15.28 GB），验证至 32K；推理可达 131K（VRAM 随 chunk 摘要缓慢增长）。
2. **质量损失极小**：在 2K 共同长度上，HGA 训练与密集训练的适配器在密集读取下仅差 0.0022 nat（PPL 差异 < 0.04），远小于微调带来的改进（相对原始模型 PPL 降低 19.4%）。
3. **效率优势**：在 2K 长度上，HGA 已略微快于密集训练（217.75 vs 207.02 tok/s），且 HGA 的注意力每 token 工作量近似常数，而密集注意力线性增长，因此随上下文延长优势会扩大。
4. **路由稀疏性**：默认 top-8 chunks 已接近饱和，降低到 top-2 可节省 81.2% KV 交互，但 PPL 仅上升 0.54。
5. **检索不变**：HGA 训练不损害密集注意力下的精确检索性能（passkey/multikey 100%，multivalue 93.8-100%）。
6. **与现有框架兼容**：微调后的适配器可直接使用标准密集注意力框架进行推理，HGA 自身也可用于检索/生成（生产级服务仍在开发中）。

## 7. 优点

- **方法创新性强**：将层级路由与外部 KV 存储有机结合，解决了传统稀疏注意力和长上下文微调的内存瓶颈。
- **实验设计干净**：采用相同超参数、数据顺序、种子，并通过密集读取评估隔离路由影响，使质量对比可信。
- **分析详尽**：给出了显存占用公式、路由稀疏度消融、缓存命中率等细节，帮助理解方法特性。
- **可落地性**：适配器与标准推理框架兼容，HGA 本身也具备独立部署潜力，面向生产环境持续开发。

## 8. 不足与局限

- **因果泄漏**：当训练 token 数超过约 1-2 亿时，由于 chunk 内路由决策共享，早期 token 可能获取后期 token 信息，导致明显的因果泄漏。当前实现仅适合微调，不适合从零预训练。
- **实验覆盖有限**：
  - 仅测试了 Qwen3-8B 与 Quadro RTX 5000，未在其他模型/GPU 上验证泛化性。
  - 训练步数仅 100 步，未测试更长训练下的质量与稳定性。
  - 长上下文（8K 以上）缺乏密集训练基线，只能与自身稀疏评估比较。
- **梯度截断损失**：分段反向传播（TBPTT）导致段间的梯度无法流动（虽然注意力可看到历史精确 K/V），可能影响极长距离依赖的建模。
