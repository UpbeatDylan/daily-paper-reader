## 摘要
设备端大语言模型推理面临响应延迟、硬件资源受限和用户隐私的三难困境。纯云推理提供强大算力但暴露用户提示和对话数据，而独立设备端推理对大多数消费级和嵌入式边缘设备不可行。本文提出一种基于端点认证KV缓存的隐私中心型边缘云协同LLM推理框架。本地端点处理输入预处理、嵌入计算、自适应特征优化、KV缓存认证、推测解码和低维模型头计算，而云端执行认证解码器推理、KV缓存管理、令牌验证和高维词汇投影。端点融合部分输出、应用语言自适应掩码并采样目标令牌。所有传输数据和截断logits均经量化及AES-GCM加密以保护隐私，核心轻量模块、草稿参数和缓存访问策略保留在本地以避免泄露。该框架通过优化流式传输、批处理和量化ONNX部署，支持包括纯CPU、GPU设备和嵌入式设备在内的异构硬件。评估表明，与基准分割推理相比，该框架将每令牌延迟降低高达46.1%，下行负载降低高达67.4%，同时保持与纯云推理相当的性能。

## Abstract
On-device LLM inference faces a trilemma of response latency, limited hardware resources and user privacy. Full cloud inference delivers strong computing power but exposes user prompts and dialogue data, while standalone on-device inference is unfeasible for most consumer and embedded edge devices. This paper presents a privacy-centric edge-cloud collaborative LLM inference framework built on endpoint-authenticated KV cache. Local endpoints handle input preprocessing, embedding computation, adaptive feature optimization, KV cache authentication, speculative decoding and low-dimensional model head calculation, while the cloud conducts authenticated decoder inference, KV cache management, token verification and high-dimensional vocabulary projection. Endpoints fuse partial outputs, apply language-adaptive masking and sample target tokens. All transmitted data and truncated logits are quantized and AES-GCM encrypted for privacy, with core lightweight modules, draft parameters and cache access policies kept local to avoid leakage. The framework supports heterogeneous devices including CPU-only, GPU-equipped and embedded devices via optimized streaming, batching and quantized ONNX deployment. Evaluations demonstrate that the framework reduces per-token latency by up to 46.1\% and downlink payloads by up to 67.4\% over baseline split inference, retaining comparable performance to full cloud inference.

- **整体含义**：本文旨在设计一种 **隐私优先** 的边端协同推理框架，在 **不牺牲延迟和性能** 的前提下，保护用户输入的敏感信息，同时支持 CPU、GPU 和嵌入式等异构端点设备。

## 2. 论文提出的方法论：核心思想、关键技术细节（公式/算法以文字说明）

### 核心思想
- **端点控制隐私**：本地端点保留隐私敏感的控制逻辑和轻量计算，云端仅执行经授权的解码器推理和高维投影，所有通信数据经过量化与加密。
- **KV 缓存认证**：端点明确授权云端是否以及如何使用历史 KV 缓存条目，避免不必要的历史信息暴露。
- **分布式词汇投影**：将语言模型头按隐藏维度拆分，端点计算低维部分，云端计算高维部分，并仅返回语言子集对应的 logits，减少通信量。
- **推测解码**：端点本地运行轻量草稿模块生成候选令牌，云端验证，接受后减少交互轮次。
- **设备自适应传输**：针对 CPU/GPU/边缘设备分别采用块式流式、模型切片、ONNX Runtime 等策略。

### 关键技术细节
- **特征准备**：端点根据硬件能力进行特征张量分块（CPU 分块流式）、模型切片（GPU）或量化 ONNX（边缘设备）。
- **授权云端解码**：端点发送控制元组 `(flag_Cache, L_Cache, d_type, ℓ)`，控制云端 KV 缓存使用。
- **分布式投影**：将隐藏状态 `h'` 按维度拆分为 `h_end` 和 `h_cloud`，权重矩阵 `W_lm` 相应拆分，端点计算 `z_end = h_end * W_end`，云端计算 `z_cloud = h_cloud * W_cloud + b`，融合后得完整 logits。语言限制生成时，云端仅返回子集 `V_ℓ` 对应的 z_cloud。
- **推测解码**：端点草稿模块 `g_ϕ` 提出 N 个未来令牌，云端以目标模型验证并更新 KV 缓存。
- **传输保护**：8-bit 量化 + AES-GCM 加密，保护负载不被网络窃听/篡改。
- **端点本地 LoRA 模块**：参数保留在端点，增加推理攻击难度。

### 算法流程（文字说明）
1. 端点上：预处理 → 计算嵌入 → 检测语言 → 生成控制元组 → 设备自适应特征准备 → 加密发送给云端。
2. 云端接收后：执行授权解码器推理（使用/不使用缓存）→ 计算云端部分隐藏状态 → 计算云端部分 logits（仅返回语言子集）→ 加密回传。
3. 端点：计算本地部分 logits → 融合 → 应用语言掩码 → 采样下一令牌 → 运行草稿模块生成候选序列 → 发送验证请求。
4. 云端：验证候选令牌 → 提交接受令牌 → 如有拒绝则从最后接受位置重算 → 更新 KV 缓存。
5. 循环直至结束符。

### 基准方法（Baselines）
1. **Cloud-only**：端点发送原始提示和历史标识符，云端完成全部推理并返回文本令牌。

1. **延迟降低显著**：相对 naive split inference，每令牌延迟在 CPU、GPU、边缘端点上分别降低 29.4%、46.1%、26.8%。

## 7. 优点

- **隐私设计为先导**：将隐私控制权放在端点，通过 KV 缓存授权、分布式投影、本地 LoRA 模块等多层次措施降低信息暴露。
- **设备自适应强**：支持 CPU（块流式）、GPU（批处理+切片）、边缘（量化 ONNX）三类异构端点，实际部署灵活。
- **端到端系统优化**：融合推测解码、语言自适应压缩、加密传输，缓解了延迟-带宽-隐私折中。
- **实验系统完整**：涵盖延迟、吞吐、通信、消融、敏感度、质量、隐私等维度，结论有数据支撑。
- **清晰的权衡分析**：专门讨论 split dimension、speculative length、cache length 等参数的影响，便于实际调参。

## 8. 不足与局限

- **隐私假设较弱**：仅对抗诚实但好奇的云端（honest-but-curious），不提供形式化差分隐私或安全多方计算保证。云端仍可观察解密后的特征张量，存在表示重建攻击风险。
- **实验网络环境单一**：仅模拟 20ms RTT、100Mbps 网络，未测试更高延迟（如跨洋）、丢包、带宽受限等极端情况。
- **任务级评估有限**：MMLU、GSM8K、HumanEval 覆盖了推理和代码，但缺少对话、长文本、多轮对话等典型场景。
