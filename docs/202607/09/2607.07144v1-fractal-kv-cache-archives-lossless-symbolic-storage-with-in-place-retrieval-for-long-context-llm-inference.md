## 摘要
键值（KV）缓存主导了长上下文自回归推理的内存成本，越来越多的研究通过量化、驱逐或卸载来压缩它。我们研究一个互补问题：一旦某个位置的KV状态被量化为码本索引，生成的符号流应如何存储，并且存储层能否做得更多？本文重新审视了一族压缩迭代映射码，它将符号序列序列化为低维实向量序列，并表明它们形成了量化KV缓存的自然档案格式，具有以下特点：该方法精确提供了增长缓存所需的访问模式。它无损、线性时间运行，并支持O(1)随机访问和O(1)均摊追加。我们在GPT-2上进行了控制实验，使用1024个token的上下文，对该档案的量化器进行馈送。保留一个小型精确窗口（4个注意力汇点+32个最近token）并将其余部分存档，每头残差向量量化将存档缓存相对于fp16缓存压缩了36-54倍，困惑度代价为11-15%。我们量化了键/值的明显不对称性——量化键的损害大约是量化值的4倍，这与先前的低位KV工作一致——并用它在混合方案中分配比特。最后，我们展示了该档案同时是一个搜索索引：近似子串查询直接在存储的向量上执行，匹配的上下文从匹配的向量中解码，而无需物化周围的文本。我们发布所有代码；每个数字都可以在笔记本电脑CPU上通过单个命令重现。

## Abstract
The key-value (KV) cache dominates the memory cost of long-context autoregressive inference, and a growing body of work compresses it through quantization, eviction, or offloading. We study a complementary question: once a position's KV state has been quantized to codebook indices, how should the resulting symbol stream be stored, and can the storage layer do more than store? A family of contractive iterated-map codes that serialize a symbol sequence into a sequence of low-dimensional real vectors is revisited, and it is shown that they form a natural archive format for a quantized KV cache with the following features. The method provides exactly the access pattern a growing cache requires. It is lossless, it runs in linear time, and supports O(1) random access and O(1) amortized append. A controlled study of the quantizer feeding this archive is conducted on GPT-2 with 1024-token contexts. Keeping a small exact window (4 attention sinks + 32 recent tokens) and archiving the rest, per-head residual vector quantization reduces the archived cache by 36-54x relative to an fp16 cache at a perplexity cost of 11-15%, and we quantify a sharp key/value asymmetry -- quantizing keys is roughly 4x more damaging than quantizing values, consistent with prior low-bit KV work -- and use it to allocate bits in a hybrid scheme. Finally, we show the archive is simultaneously a search index: approximate substring queries execute directly on the stored vectors, and matched context is decoded from the matched vector without ever materializing the surrounding text. We release all code; every number reproduces from a single command on a laptop CPU.

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：长上下文自回归推理中，KV（键值）缓存的内存成本成为主要瓶颈。GPT-2模型中每token需要18,432个标量，生产级模型更达到每token数百KB，使得缓存远大于模型权重，限制了批大小和上下文长度。
- **现有方案**：通过低比特量化、驱逐“不重要”位置、卸载冷缓存到慢速内存来解决，但这些方法集中在下游有损压缩，忽视了存储层的设计潜力。
- **核心问题**：一旦KV状态被量化为码本索引（符号流），如何高效存储这些符号流？存储层能否同时提供随机访问、追加和检索功能，而不仅仅是存储？
- **整体含义**：作者提出一种无损符号存储格式（基于收缩迭代映射码），将量化后的KV缓存符号流序列化为低维实向量，既能线性时间编码、O(1)随机访问和O(1)均摊追加，又能作为原位搜索索引，从而统一存储与检索。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：使用一族**收缩迭代映射码（contractive iterated-map codes）** 将符号序列（如量化后的KV索引）映射为平面上的点轨迹。具体地，固定一个正则N边形，每个顶点对应一个符号；当前点通过映射 \( p_k = V(c_k) + r (p_{k-1} - V(c_k)) \) 更新，其中 \( r \) 为收缩比率，\( V(c_k) \) 为符号对应的顶点。该映射形成迭代函数系统，生成的吸引子是自相似分形（如谢尔宾斯基地毯）。解码时反转映射：给定点，找到包含该点的细胞对应符号，并恢复前一点。
- **关键技术细节**：
  - 唯一解码要求细胞不相交，因此收缩比率 \( r \) 必须满足包装常数 \( r \le r_N \)（对于 \( N \le 4\) 可使用 \( r=1/2 \)）。
  - 解码时浮点误差按 \( 1/r \) 放大，双精度（64位）可靠携带约 \( 44 / \log_2(1/r) \) 个符号，因此实际编解码使用固定长度跨度存储一个点，并通过向后搜索重建每个跨度。
  - 该码在序列分析领域称为混沌游戏表示（chaos game representation）或通用序列映射（universal sequence maps），本文是其作为KV档案的新应用。
- **归档存储与检索**：存储点轨迹作为符号流的无损表示。由于收缩效应，存储点的主要信息由最近符号决定，早期符号几何衰减。因此，两点之间的欧氏距离近似为 \( 2 r^s \)（s为共同后缀长度），基于此可实现近似子串检索：对查询编码后，在存储点集合中阈值距离邻近搜索即可找到匹配后缀。

## 6. 主要结论与发现

1. **收缩迭代映射码作为KV档案是可行的**：无损、线性时间、O(1)随机访问和O(1)均摊追加（表1基准）。
2. **每头码本优于池化码本**：在同等比特预算下，每头码本的困惑度降低明显（如单阶段：+23.7% vs. +51.0%）。
3. **键/值量化不对称性**：量化键对困惑度的影响约为量化值的4倍（+14.5% vs. +4.0%），二者近似独立复合。该方向与KIVI一致。
4. **比特非对称混合方案最佳**：分配4阶段RVQ给键、2阶段给值，达到36倍压缩（相对于fp16）同时困惑度仅增加11.2%。
5. **档案也是检索索引**：利用几何衰减性质，近似子串匹配直接在存储点上执行，召回率始终为1.0，精度取决于数值精度，且解码可验证。查询可在0.9ms内搜索100K位置（2D搜索）。

## 7. 优点

- **创新性**：将存储层从被动容器提升为主动助手，实现统一的无损存储、随机访问和原位检索。这一思路在KV压缩领域新颖。
- **简洁高效**：使用经典分形编码，无需复杂索引结构；编码和访问操作直观。
