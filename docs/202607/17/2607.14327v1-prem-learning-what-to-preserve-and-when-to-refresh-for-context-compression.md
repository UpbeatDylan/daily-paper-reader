## 摘要
高效的长上下文推理不仅关乎减少内存开销，还关乎在生成过程中保持有用的上下文证据可访问。然而，现有的压缩导向方法，如键值缓存压缩和上下文压缩，通常要么在早期决定保留哪些上下文信息，要么依赖外部压缩器。这种设计使得压缩后的上下文难以适应后续推理步骤所需的证据。本文介绍了PReM（保留与刷新记忆），一种上下文压缩框架，它将长上下文维护为模型的内部逐层键值记忆，并学习保留什么以及何时刷新它。具体而言，PReM使用一个专用记忆层来做记忆选择决策，以及一个特殊记忆标记< m >来在生成过程中触发刷新。为了训练这种行为，PReM引入了分阶段分离的刷新训练，将记忆选择与记忆条件生成对齐，同时保持刷新间的连续性。在32K令牌上下文上的实验表明，PReM在16倍和32倍压缩下均优于强基线，同时在答案质量和推理效率之间保持了良好的平衡。

## Abstract
Efficient long-context inference is not only about reducing memory cost, but also about keeping useful contextual evidence accessible as generation proceeds. However, existing compression-oriented approaches, such as key-value (KV) cache compression and context compression, often either make an early decision about which contextual information to keep or rely on an external compressor. Such designs make it difficult to adapt the compressed context to the evidence needed by later reasoning steps. This paper introduces PReM (Preserve and Refresh Memory), a context-compression framework that maintains the long context as the model's internal layer-wise KV memory and learns what to preserve and when to refresh it. Specifically, PReM uses a dedicated memory layer to make memory-selection decisions, and a special memory token <m> to trigger refreshes during generation. To train this behavior, PReM introduces Phase-Separated Refresh Training, aligning memory selection with memory-conditioned generation while preserving continuity across refreshes. Experiments with 32K-token contexts show that PReM outperforms strong baselines under both 16x and 32x compression, while maintaining a favorable balance between answer quality and inference efficiency.

### 2. 论文提出的方法论

#### 核心思想
- 将长上下文以 **逐层 KV 状态（layer-wise KV memory）** 的形式保留在 LLM 内部，不依赖外部压缩器。
- 通过 **专用记忆层（memory layer）** 进行分块（chunk）级的选择决策，决定哪些 chunk 保留全 token 级 KV、哪些 chunk 被平均池化（压缩）。
- 引入 **特殊记忆 token ⟨m⟩**，模型在生成过程中自主发出该 token 以触发记忆刷新，每次刷新时记忆层根据当前隐藏状态重新选块，并传播到所有层。

#### 关键技术细节
1. **分块与评分**：将上下文 token 序列分为 chunk（大小 c），对第 j 个 chunk 计算其平均 key；再利用当前步骤的 selection query（来自问题或 ⟨m⟩ 的隐藏状态）通过类注意力得分（公式 1）进行打分。
2. **选择与压缩**：在记忆层 ℓₘ 进行 Top‑k 选择（k 为 chunk 预算），选中 chunk 保留全 token KV，未选中 chunk 被平均池化为一个代表 KV pair。已保留的原始 KV 仍保留作为后续刷新的来源。
3. **记忆跨层传播**：记忆层的选择结果 (Sₛ) 被复用给其他层——低于记忆层的层在 refresh forward pass 中临时应用压缩，得到 Sₛ 后刷新；高于记忆层的层直接使用 Sₛ 压缩自身 KV。
4. **阶段分离刷新训练**：每一步分为两个阶段，分别前向传播：

### 6. 论文的主要结论与发现

1. **PReM 显著优于所有对比方法**：在 16× 压缩下，3B 模型平均 EM/F1 比最强基线高出 5.30/5.15 点；7B 下高出 10.23/12.55 点。32× 下优势继续扩大（+3.07/+3.80 和 +8.20/+9.89）。
2. **学习型压缩可超越全上下文提示**：即使在 32× 压缩下，PReM 仍比 32K 无压缩提示高出约 11 个 EM/F1 点，说明选择性记忆更新比暴露全部上下文更有效。
3. **对多跳任务特别有效**：2WikiMQA、MuSiQue 等数据集上优势最大，验证了步骤级刷新带来的证据适配能力。
4. **小模型超越大基线**：PReM‑3B 平均超越 7B 的 Activation Beacon（16× 提升 7.93/7.49，32× 提升 8.67/9.44），证明内部动态压缩可弥补骨干规模劣势。
5. **记忆层选择**：Middle‑to‑late 层比早期层更适合作为记忆层，且该发现泛化到 Llama3、Qwen2.5‑14B。
6. **效率与效果的平衡**：PReM 解码峰值内存比全记忆减少约 35%，端到端延迟适中，且可通过 ⟨m⟩ 数量灵活调控计算开销。

### 7. 优点

- **问题定义创新**：将上下文压缩重新定义为“保留与刷新”问题，不再是单次静态压缩。
- **方法设计精巧**：
  - 使用 LLM 自身 KV 状态作为记忆，无需额外压缩器。
  - 专用记忆层 + ⟨m⟩ token 实现轻量级自主刷新，刷新成本低。
  - 阶段分离刷新训练保证了训练‑推理一致性（refresh 后的 token 都在新记忆下生成）。
- **全面实验**：涵盖多个数据集、两种压缩率、多种基线，并配有消融、效率、证据召回分析，支撑充分。
- **实用性**：在 3B 模型上即可达到甚至超过 7B 压缩方法的性能，且内存和延迟可控。
