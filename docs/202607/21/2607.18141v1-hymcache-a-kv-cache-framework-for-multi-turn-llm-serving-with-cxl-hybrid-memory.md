## 摘要
长上下文、多轮和代理型LLM工作负载越来越多地重用先前处理过的上下文，使得KV缓存重用对于减少冗余计算至关重要。然而，这种重用将瓶颈转移到存储和提供可重用KV状态的存储层。GPU HBM和主机DRAM成本过高，无法扩展到TB级的共享上下文容量，这促使了基于更低成本、更高容量介质的远程存储层的出现。本文介绍了HyMCache，一种集成CXL混合内存（CXL-HM）用于多轮LLM服务的KV缓存框架。CXL-HM将少量设备内DRAM与大量SSD后端容量结合在CXL接口之后。通过利用多轮KV缓存访问的读主导、可预测和仅追加特性，HyMCache重新思考了CXL-HM中的DRAM管理，以有效支持TB级SSD后端的KV重用。它使用请求级前缀预取和机会性写缓冲，将延迟关键的读取阶段置于设备DRAM中，从而以SSD级别的成本实现DRAM级别的KV缓存效率。我们在真实的CXL-HM原型上，在单聚合器和PD分离服务配置下评估了HyMCache。在相同的DRAM预算下，HyMCache在单节点服务中性能优于本地LMCache 3.0倍，在PD分离服务中性能优于1.45倍。与1 TB分布式DRAM Mooncake相比，HyMCache性能降低约30%，但DRAM使用量减少16倍。

## Abstract
Long-context, multi-turn, and agentic LLM workloads increasingly reuse previously processed context, making KV-cache reuse essential for reducing redundant computation. However, this reuse shifts the bottleneck to the memory tier that stores and serves reusable KV states at cluster scale. GPU HBM and host DRAM are too costly to scale to TB-scale shared context capacity, motivating remote tiers built from lower-cost, higher-capacity media. This paper presents HyMCache, a KV-cache framework that integrates CXL-hybrid memory (CXL-HM) for multi-turn LLM serving. CXL-HM combines a small amount of in-device DRAM with large SSD-backed capacity behind a CXL interface. By exploiting the read-dominant, predictable, and append-only nature of multi-turn KV-cache access, HyMCache rethinks DRAM management within CXL-HM to efficiently support TB-scale SSD-backed KV reuse. It uses request-level prefix prefetching and opportunistic write buffering to stage latency-critical reads in device DRAM, enabling DRAM-scale KV-cache efficiency at SSD-level cost. We evaluate HyMCache on a real CXL-HM prototype under both single-aggregator and PD-disaggregated serving configurations. Under the same DRAM budget, HyMCache outperforms local LMCache by 3.0x in single-node serving and 1.45x in PD-disaggregated serving. Compared with 1 TB distributed-DRAM Mooncake, HyMCache incurs about 30% lower performance but uses 16x less DRAM.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **背景**：大语言模型（LLM）服务正从单轮提示向长上下文、多轮对话和代理型应用演变。KV缓存重用可大幅减少冗余计算，但可重用KV状态的存储和传输成为新瓶颈。GPU HBM和主机DRAM成本过高，无法经济地扩展到TB级共享上下文容量。
- **问题**：业界探索存储介质（如SSD）和CXL内存扩展来构建低成本远程KV缓存层。然而，纯存储方案（如NVMe-oF）引入软件栈开销，而纯DRAM的CXL内存池成本依然高昂。CXL混合内存（CXL-HM）结合小DRAM和大SSD，但通用LRU缓存管理策略不匹配多轮KV缓存的访问模式（扫描式、弱局部性、读主导）。
- **整体含义**：本文旨在设计专门适配多轮LLM服务的CXL-HM管理方案，以SSD级成本提供接近DRAM的KV缓存效率，解决成本与性能的权衡问题。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：利用多轮KV缓存访问的**读主导、可预测、仅追加**特性，重新设计CXL-HM内部DRAM的管理方式：将设备DRAM从透明LRU缓存改为**显式管理的暂存空间**，通过请求级前缀预取和机会性写缓冲实现延迟关键读取在DRAM中的命中。
- **关键技术细节**：
  - **KV对象预取（Prefetching）**：多轮前缀缓存查找可获知请求将消费的KV对象顺序列表。CXL-HM提供显式预取API（`chm_prefetch_object`、`chm_prefetch_wait`、`chm_prefetch_release`），允许服务栈在远程读取到达前将对象从SSD阶段到设备DRAM。
  - **延迟隐藏暂存窗口**：固定深度的暂存窗口，KV对象流经窗口后被释放，避免DRAM容量按总前缀KV规模增长；依靠SSD带宽保持窗口填充。
  - **读优先写隔离刷新**：写操作（新生成KV）与延迟关键读路径隔离：写缓冲有限空间，异步批量刷新到SSD，避免内部读写干扰。
  - **用户级预取协调**：基于`issue-wait-release`协议，服务栈按消费顺序发送预取命令。
  - **系统集成**：HyMCache在Dynamo/vLLM栈上新增四个模块：
    - **HyMCache Master**：维护全局KV对象到CXL-HM节点地址的映射。
    - **HyMCache Lookup**：提前远程命中匹配、批量元数据查询、协调预取。
    - **HyMCache KV Connector**：嵌入vLLM的KV传输回调，负责RDMA读写和超时回退。
    - **HyMCache KV Manager**：在CXL-HM侧管理预取资源，动态分配预取窗口（默认最多128MB/请求）。
  - **统一地址空间**：每个CXL-HM设备作为CPU-less NUMA节点，以请求粒度分配NUMA节点，保持有序预取。
