## 摘要
随着大语言模型（LLMs）处理越来越长的提示，计算和KV缓存内存成本已成为推理系统的主要瓶颈。现有的输入级提示压缩方法通过一个标量相关性分数对每个句子进行排序，将文档视为无结构的单词和句子池，试图解决这一问题。但在预算紧张的情况下，这会导致主题坍缩：文档的主导主题消耗了预算，丢弃了不那么频繁但与任务相关的主题。为了保持主题覆盖，需要跨重复出现的主题分配预算，而不是孤立地对句子评分。为此，我们提出SALT，一个与模型无关的抽取式框架，它将每个句子的关键词组织成一个按句子频率（SF）排序的特里树，句子频率是文档主题结构的轻量级、可复用代理。这种基于特里树的结构平滑了内存分配，防止主导主题垄断预算。多锚点检索可以激活由查询关键词标记的任何深度的特里树节点，且该特里树跨对话轮次持久存在，支持无需重新编码文档的多轮使用。通过保留文档主题，SALT减少了长上下文提示的预填充计算和内存成本，同时可与针对解码时延和内存的KV缓存方法组合使用。

## Abstract
As large language models (LLMs) process increasingly longer prompts, computation and KV-cache memory costs have emerged as major bottlenecks in inference systems. Existing input-level prompt compression methods address this, but rank each sentence by a scalar relevance score, treating the document as an unstructured pool of words and sentences. Under tight budgets, this causes theme collapse, where the dominant theme(s) of a document consumes the budget, discarding less-frequent yet task-relevant themes. Preserving thematic coverage instead requires allocating the budget across recurring themes rather than scoring sentences in isolation. To this end, we propose SALT, a model-agnostic extractive framework that organizes per-sentence keywords into a trie ordered by sentence frequency (SF), a lightweight, reusable proxy for document thematic structure. This trie-based organization smooths memory allocation and prevents dominant themes from monopolizing the budget. Multi-anchor retrieval activates trie nodes labeled by query keywords at any depth, and the trie persists across dialogue turns, supporting multi-turn use without re-encoding the document. By preserving document themes, SALT reduces the prefill computation and memory cost of long-context prompts while remaining composable with KV-cache methods that target decoding-time latency and memory.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：大语言模型（LLMs）处理长上下文提示（几十万token）时，预填充计算和KV缓存内存成为主要瓶颈。现有输入级提示压缩方法（如EXIT、RECOMP）将文档视为无结构单元，根据标量相关性分数对每个句子独立排序，在预算紧张时会导致**主题坍缩（theme collapse）**：文档的主导主题（高频主题）独占预算，低频但任务相关的次要主题被丢弃。
- **核心问题**：如何在固定预算下保持文档的主题覆盖，而不是单纯按全局分数“砍句子”。
- **整体含义**：SALT是一种模型无关的抽取式压缩框架，通过构建文档的词汇主题结构并据此分配预算，避免了主题坍缩，同时减少了预填充计算和内存成本，且可兼容KV缓存方法（如SnapKV、FastKV）来优化解码阶段。

## 2. 论文提出的方法论

### 核心思想
- 将压缩问题重构为**在预算内保留文档主题覆盖**：先构建轻量级的文档词汇主题结构（基于句子频率的词汇特里树），再基于该结构分配预算到不同主题分支，最后在分支内选择句子。
- 使用**句子频率（Sentence Frequency, SF）** 作为主题结构代理：统计每个关键词在多少句中被提取为关键词，高频词代表文档的重复主题。

### 关键技术细节
1. **索引阶段（Indexing）**：
   - 使用轻量级编码器（BGE-small-en-v1.5）对每个句子提取关键词：采用[CLS]注意力权重排序内容词，并通过膝盖检测（Kneedle）算法选择前k个关键词（占比不超过40%）。
   - 构建**句子频率（SF）**：统计每个关键词跨句子的出现次数。
   - 构建**显著性关键词集S**：保留SF高于p分位数（默认0.9）的关键词。
   - 构建**关键词特里树（Trie）**：每个句子按关键词排序（降频）插入特里树，叶子节点存储句子ID，共享前缀表示共同主题。

2. **选择阶段（Selection）**：
   - **摘要模式（Summary mode）**：从根节点开始，首先在深度1分支上分配预算（固定基数 + 基于未覆盖质量的残差份额），然后在各分支内根据边际增益选择句子，最后用全局填充（GlobalFill）使用剩余预算。
   - **查询模式（Query mode）**：提取查询关键词，激活匹配的特里树节点，执行锚点阶段（按词汇重叠度和嵌入相似度选句），剩余预算同摘要模式。
   - **多锚点检索**：激活所有匹配标签的节点，联合其子孙区域进行选择，支持跨对话轮次复用索引。

### 公式或算法流程（文字说明）
- **分支未覆盖质量**：`Uv(R) = Σ_{w∈Γ(v)\Cv(R)} cSF(w)`，其中`Γ(v)`是子树v的关键词签名，`Cv(R)`是已选句子覆盖的关键词。
- **句子得分**：`score_b(si | R) = Δb(si | R) * L(ni) * I(si)`，其中`Δb`是句子使分支未覆盖质量的减少量，`L(ni)`是句子长度惩罚，`I(si)`是位置/查询相关先验。
- **预算分配**：分支配额 = 固定基数 + 残余份额 × (M_b + ε)^α，M_b是分支未覆盖质量，α∈(0,1)子线性指数。

- **准确性**：在20%预算下，SALT在LongBench平均准确率（43.51%）显著优于其他预处理方法（EXIT 31.68%、RECOMP 40.22%、CPC 38.05%、Sentinel 39.47%），接近KV缓存方法（SnapKV 49.04%）。在QuALITY多轮对话中，SALT（72.6%）接近基线（74.1%），远好于RECOMP（61.5%）。

- **硬件兼容性**：可在V100上处理256k上下文，而许多KV缓存方法依赖现代架构。
- **主题覆盖**：诊断实验（表3）证明，基于特里树的主题分配优于全局分数或其它多样性启发式。

## 7. 优点

- **新颖性**：首次提出“主题坍缩”问题，并引入基于词汇特里树的显式预算分配，将覆盖目标提升到句子选择之前。
- **模型无关**：输出纯文本，可与任何下游模型和KV缓存方法组合。
- **轻量高效**：使用小型编码器（BGE-small）和简单统计，索引和选择过程极快。
- **多轮对话**：索引一次构建，多次查询复用，显著降低累积成本。
- **硬件可移植性**：无需特定CUDA核或FlashAttention，可在旧GPU上运行。

## 8. 不足与局限

- **代码任务弱**：在LongBench的Code子任务上显著落后于KV缓存方法和基线，原因：句子级粒度不适合代码语义；词汇关键词对代码标识符提取效果差。
- **句子级粒度的下限**：无法像token级方法那样超强压缩（低于20%），因为任务相关跨度通常在5-20%。
- **主题结构依赖词汇共现**：对语义高度重叠但词汇不同的文档（如同义词过多的文本）可能不够鲁棒。
- **实验覆盖**：仅评估了8B模型，未测试更大模型（如70B）或更极端预算（<10%）。消融实验较少（仅APPENDIX A和B中的少量分析）。
- **偏差风险**：仅使用英语数据集，未检验多语言泛化能力；索引构建依赖BGE模型，可能偏向其训练分布。
