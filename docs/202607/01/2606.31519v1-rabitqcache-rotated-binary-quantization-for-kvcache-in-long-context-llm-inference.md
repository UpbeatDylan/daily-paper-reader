## 摘要
长上下文大型语言模型推理因大量的键值缓存而严重受限，然而现有的稀疏注意力方法往往采用静态固定预算（Top-k）检索，或者依赖计算成本高且有偏的代理得分。为克服这些限制，我们提出RaBitQCache，一种新颖的稀疏注意力框架，利用随机旋转二元量化和高吞吐量的二元-INT4算术来高效估计注意力权重。我们的代理得分作为无偏估计量，具有可证明的误差界，支持自适应的Top-p检索，能根据实际注意力稀疏性动态调整token预算。我们进一步实现了一个具有硬件意识的系统，采用异步流水线和延迟更新来掩盖开销。评估表明，与最先进的基线相比，RaBitQCache在保持生成质量的同时显著加速推理并减少内存I/O。代码可在 https://github.com/Sakuraaa0/RaBitQCache.git 获取。

## Abstract
Long-context Large Language Model inference is severely bottlenecked by the massive Key-Value (KV) cache, yet existing sparse attention methods often suffer from static fixed-budget (Top-k) retrieval or rely on proxy scores that are computationally expensive and biased. To address these limitations, we propose RaBitQCache, a novel sparse attention framework that utilizes randomized rotated binary quantization and high-throughput binary-INT4 arithmetic to efficiently estimate attention weights. Our proxy score serves as an unbiased estimator with a proven error bound, enabling adaptive Top-p retrieval that dynamically adjusts the token budget based on actual attention sparsity. We further implement a hardware-aware system with asynchronous pipelining and lazy updates to mask overhead. Evaluations demonstrate that RaBitQCache significantly accelerates inference and reduces memory I/O while preserving generation quality compared to state-of-the-art baselines. Code is available at https://github.com/Sakuraaa0/RaBitQCache.git.

## 2. 论文提出的方法论

### 核心思想
基于Johnson-Lindenstrauss引理，利用随机旋转将高维键向量映射到单位超立方体顶点，得到1-bit二进制索引；对查询向量进行INT4均匀量化；通过二进制-INT4内积快速计算代理得分，该得分为原始注意力的无偏估计量，具有严格的误差界 \( O(1/\sqrt{D}) \)。利用该得分实现自适应Top-p检索（累积概率阈值），动态调整每个解码步骤的token预算。

### 关键技术细节
1. **预处理与归一化**：对query和key减去预填充阶段的质心 \( c_q, c_k \) 并单位化。
2. **随机旋转**：使用固定随机正交矩阵 \( P \in \mathbb{R}^{D \times D} \) 旋转归一化向量。
3. **二进制索引构建**：对旋转后的key取符号，得到二进制向量 \( \bar{C}_b \in \{0,1\}^D \)，并计算校正因子 \( \alpha = \langle \bar{c}, P^T k_c \rangle \)。
4. **查询INT4量化**：将旋转后的query \( q' \) 均匀量化到4-bit整数向量 \( \bar{q}_u \)。
5. **高效得分估计**：利用自定义CUDA内核计算二进制-INT4点积，重建近似内积，除以 \( \alpha \) 后结合式(2)还原原始注意力得分。
6. **自适应Top-p选择**：基于线性时间三分搜索实现Top-p掩码，避免排序的 \( O(L \log L) \) 开销。
7. **系统级优化**：
   - **异步预填充**：将索引构建与主注意力计算流水线化，隐藏复杂度为 \( O(L) \) 的量化开销（主注意力为 \( O(L^2) \)）。
   - **延迟更新**：解码阶段新token暂存于全精度局部窗口缓冲区，累积到一定量后再批量量化，减少内核启动开销。

### 算法流程
- **预填充阶段（算法1）**：计算质心、归一化、旋转、取符号得到二进制索引、计算校正因子。
- **解码阶段（算法2）**：对query进行归一化、旋转、INT4量化；计算二进制-INT4点积并转换为得分；Top-p选择索引；获取全精度KV对进行稀疏注意力计算（结合局部滑动窗口）。
