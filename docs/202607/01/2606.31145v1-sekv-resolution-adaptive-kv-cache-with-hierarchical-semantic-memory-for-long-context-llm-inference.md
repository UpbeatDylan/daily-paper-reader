## 摘要
大语言模型越来越多地处理长上下文，此时KV缓存成为主要的内存瓶颈：其大小随序列长度线性增长，且必须在解码过程中全程保留，若不进行压缩，在GPU上完整缓存代价过高。现有的KV缓存压缩方法难以在效率与忠实保留上下文之间取得平衡。令牌驱逐会丢弃信息，而语义分组在预填充阶段固定压缩决策；一旦生成过程中某个压缩片段变得相关，两者都无法从中恢复令牌级别的细节。作为解决方案，我们提出SeKV，一种分辨率自适应的语义KV缓存，将上下文组织为熵引导的语义片段，并跨GPU-CPU内存层次结构存储，不丢弃信息。每个片段在GPU上保留轻量级摘要向量用于粗粒度路由，在CPU上保留低秩SVD基用于按需令牌级重建。训练得到的缩放机制在解码过程中选择性地展开与查询相关的片段，从而在不将完整KV缓存加载到GPU的情况下实现精确检索。SeKV在完全冻结基础LLM且仅增加不到0.05%可训练参数的情况下，实现了自适应令牌级重建。在四个基准测试中，与最强语义压缩基线相比，SeKV平均提升5.9%，同时在128K上下文下，与完整KV缓存相比，GPU内存减少53.3%。代码可在https://github.com/AmirAbaskohi/SeKV获取。

## Abstract
Large language models increasingly operate over long contexts, where the KV cache becomes a dominant memory bottleneck: its size grows linearly with sequence length and must be retained throughout decoding, making full GPU caching prohibitively expensive without compression. Existing KV cache compression methods struggle to balance efficiency with faithful context preservation. Token eviction discards information, while semantic grouping fixes compression decisions at prefill time; neither can recover token-level detail from a compressed span once it becomes relevant during generation. As a solution, we propose SeKV, a resolution-adaptive semantic KV cache that organizes context into entropy-guided semantic spans and stores them across a GPU-CPU memory hierarchy without discarding information. Each span keeps a lightweight summary vector on GPU for coarse routing and a low-rank SVD basis on CPU for on-demand token-level reconstruction. A trained zoom-in mechanism selectively expands query-relevant spans during decoding, enabling precise retrieval without materializing the full KV cache on GPU. SeKV enables adaptive token-level reconstruction while keeping the base LLM fully frozen and adding fewer than 0.05% trainable parameters. Across four benchmarks, SeKV improves over the strongest semantic compression baseline by 5.9% on average while reducing GPU memory by 53.3% versus full KV caching at 128K context. Code is available on https://github.com/AmirAbaskohi/SeKV.

## 1. 论文的核心问题与整体含义

长上下文LLM推理面临严峻的内存瓶颈：KV缓存大小随序列长度线性增长，且解码全程必须驻留GPU，导致完整缓存（Full KV）代价极高。现有压缩方法（令牌驱逐、语义分组）要么永久丢弃信息，要么在预填充阶段固定压缩策略，无法在解码过程中按需恢复已被压缩片段的令牌级细节。SeKV提出一种分辨率自适应的语义KV缓存，不丢弃任何信息，而是通过GPU/CPU异构存储和动态缩放机制，在保持基LLM冻结的条件下实现高效长上下文推理。

## 2. 论文提出的方法论

### 核心思想
将上下文组织为层次化语义记忆：熵引导的语义分段，每段用双分辨率表示（GPU摘要+CPU低秩基），通过训练得到的缩放机制动态展开查询相关片段。

### 关键技术细节
- **熵引导分段**：利用预填充阶段令牌的 surprisal（−log p(xt|x<t)），将超过阈值 μ+ασ 的位置作为边界，保留高 surprisal 的“锚点令牌”在GPU上全精度，其余令牌组成语义片段。
- **双分辨率表示**：
  - **GPU摘要**：每个片段通过每头每层的投影矩阵 W^(l,h) 将键向量投影到低维空间（d′=32），用 surprisal 加权平均得到紧凑摘要向量，用于粗粒度路由。
  - **CPU低秩SVD基**：对片段的键/值矩阵进行截断SVD（最大秩R=32），并训练一个轻量级门预测器 gϕ 自适应分配每个奇异分量的保留门 m，实现按需令牌级重建。
- **内存层次**：锚点令牌和片段摘要驻留GPU，SVD基存储在CPU；解码时先路由摘要，对超过阈值 τ^(l,h) 的片段触发异步CPU→GPU传输和重建。
- **训练缩放机制**：仅训练路由投影、门预测器和阈值（共约430万参数，<0.05%），损失函数包含蒸馏（对齐全KV教师）、缩放监督（基于教师注意力质量）、重建误差（SVD门控损失）、预算惩罚。

## 8. 不足与局限

- **依赖分段质量**：熵引导分段在碎片化文档（如表格、代码、非标准格式）中可能产生噪声边界，导致子最优分配。
- **额外超参数**：引入多个手动设定（α、锚点数量、最大秩、局部窗口大小、预算系数等），跨模型或长度域可能需要重新调参。
- **CPU-GPU带宽瓶颈**：在慢速设备或对抗性查询（反复触发大量远端片段）时，延迟优势可能减弱。
- **冻结基LLM**：虽利于部署，但可能限制了模型完全适应压缩记忆结构的潜力；未来可探索联合训练。
- **稀疏SVD存储成本**：CPU上存储的SVD因子仍与片段长度线性相关（左奇异向量），虽较全KV省但非完全轻量。
