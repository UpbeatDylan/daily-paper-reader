## 摘要
我们研究了在严格键值（KV）缓存内存预算下，批量大语言模型（LLM）推理的非先验调度问题。每个请求具有已知的提示长度但未知的响应长度，其内存占用由固定的提示部分和随每个解码令牌增长的响应部分组成。在每个解码轮次中，调度器选择一个可行的活跃请求批次；驱逐一个请求会丢弃其累积的缓存状态，浪费之前的计算。目标是相对于知道所有响应长度的最优先验调度，最小化总完成时间。我们提出了第一个对于任意提示长度和任意响应长度且无额外假设的常数竞争比算法。我们的算法并非依赖于单一的通用调度策略，而是基于一种新颖的态势感知路由框架。专门的子调度器处理不同的内存增长形态，而元调度器在它们之间时分共享内存预算，并在每个作业的执行逐步揭示其行为时动态路由该作业。该框架还为在线到达情况下的制造期和总完成时间提供了常数竞争比保证。

## Abstract
We study non-clairvoyant scheduling for batched Large Language Model (LLM) inference under a hard Key-Value (KV) cache memory budget. Each request has a known prompt length but an unknown response length, and its memory footprint comprises a fixed prompt component together with a response component that grows with each decoded token. At each decoding round, the scheduler chooses a feasible batch of active requests; evicting a request discards its accumulated cache states, wasting prior computation. The goal is to minimize total completion time against the optimal clairvoyant schedule that knows all response lengths.   We present the first constant-competitive algorithm for arbitrary prompt lengths and arbitrary response lengths with no additional assumptions. Rather than relying on a single universal scheduling policy, our algorithm is built on a novel regime-aware routing framework. Specialized sub-schedulers handle different memory-growth geometries, while a meta-scheduler time-shares the memory budget across them and dynamically routes each job as its execution progressively reveals its behavior. This framework also yields constant-competitive guarantees for makespan and for total completion time under online arrivals.

#### 1. 核心问题与整体含义（研究动机和背景）

- **背景**：大语言模型（LLM）推理正成为现代计算基础设施的基石。Transformer 架构的自注意力机制在每个请求的解码过程中生成键值（KV）缓存，其大小随生成的 token 数量线性增长。现代推理系统通过**连续批处理**和**基于块的缓存管理**来管理有限的内存预算。当内存耗尽时，必须“杀死”活跃请求，丢弃已积累的 KV 缓存，浪费先前计算。因此，调度器需要在**高并发**（提升吞吐）与**避免 kill 浪费**之间权衡。
- **核心问题**：调度器是**非先验的（non-clairvoyant）**——响应长度在完成前未知，且由于自回归和随机性，即使借助专用预测器也无法精确预测。未知的响应长度直接转化为未知的未来内存需求，因为缓存随每个解码 token 增长。这属于调度理论中的非先验设置，但额外增加了“资源需求在执行过程中不断增长”的特点。
- **研究目标**：在一个拥有**硬内存预算 M** 的单一副本（replica）上，调度一批已知提示长度但未知响应长度的请求，最小化**总完成时间**（所有请求完成时间之和）。衡量标准是与知道所有响应长度的**最优先验最优调度**相比的竞争比（competitive ratio）。已有工作仅在受限场景（如先验、相同提示长度、大内存）下给出保证。**本文首次回答：能否设计一个非先验调度器，与完全知情的最优调度在总完成时间上只差常数因子？**

#### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：**态势感知路由框架（Regime-Aware Routing）**。将作业按几何特征分为三个**态势（regime）**，每个态势使用专门的子调度器，再由一个**元调度器**时间共享内存并动态路由作业（因为非先验下无法预先知道作业属于哪个态势）。具体地：
    - **提示长度阈值**：取 \(\Lambda = M/4\)。
    - **三大态势**：
        1. **大型作业（Large jobs）**：\(s_i > \Lambda\)。每个作业几乎单独填满内存，使用固定宽度 \(w_i = M\) 的矩形调度。
        2. **小型提示主导作业（Small prompt-heavy jobs）**：\(s_i \le \Lambda\) 且 \(o_i \le s_i\)。内存增长不超过提示长度两倍，使用固定宽度 \(w_i = 2s_i\) 的矩形调度。
        3. **小型响应主导作业（Small response-heavy jobs）**：\(s_i \le \Lambda\) 且 \(o_i > s_i\)。内存增长远超提示，面积由响应长度二次主导，使用几何切片调度。
- **关键技术细节**：
    - **矩形条调度器 Rect（算法1）**：用于大型和提示主导态势。每个作业的尝试（attempt）具有几何递增的长度上限 \(2^r\) 和预算 \(B_{i,r} = w_i 2^r\)。维护按预算排序的待处理队列，每次事件（开始、完成、到期）后贪心地启动最小预算的尝试（若能放入剩余内存）。关键性质是**队列单调性**：待处理尝试的最小预算随事件非减，且活跃尝试的预算总不大于待处理尝试的预算。这保证了近似面积序的完成顺序和常数内存利用率。
    - **响应主导调度器 Response（算法2）**：用于响应主导态势。采用几何递增的响应上限 \(\tau_r = 2^r\)，每阶段使用代理提示长度 \(\bar{s}_r = \min\{\tau_r, M-\tau_r\}\)，利用 SPS（Staggered Pipeline Scheduling）子程序调度所有满足 \(s_i \le \tau_r\) 的作业。SPS 在共同代理提示下实现高内存利用率，且由于几何上限，完成顺序与面积序接近常数因子。
    - **路由元调度器 Route（算法4）**：在非先验下运行。初始将小提示作业放入提示主导分支。若作业在该分支运行了 \(s_i\) 轮仍未完成，则被认证为响应主导，转移到响应主导分支。大型作业直接进入大型分支。使用与黑盒元调度器相同的倍增阶段（\(2^r\) 轮），每阶段按顺序运行提示主导、响应主导、大型分支。关键引理（Lemma 11）：认证和完成的开销被常数因子控制。
