## 摘要
多示例上下文学习作为一种有前景的范式，通过利用大量示例来释放大型语言模型的推理潜力。然而，现有方法通常依赖预定的固定示例数量。这种静态方法往往无法适应不同查询的难度变化，导致上下文不足或噪声干扰。此外，长上下文带来的高昂计算和内存成本严重限制了多示例学习的可行性。为解决上述局限性，我们提出AdapShot，它动态优化示例数量并利用KV缓存重用来实现高效推理。具体地，我们设计了一种基于探测的评估机制，利用输出熵确定最佳示例数量。为了避免在探测和推理阶段进行冗余的预填充计算，我们引入了语义感知的KV缓存重用策略。在该重用策略中，为解决位置编码不兼容问题，我们提出了一种解耦与重编码方法，使得缓存的键值对可以灵活重排序。大量实验表明，与最先进的DBSA相比，AdapShot平均性能提升约10%，速度提升4.64倍。

## Abstract
Many-Shot In-Context Learning (ICL) has emerged as a promising paradigm, leveraging extensive examples to unlock the reasoning potential of Large Language Models (LLMs). However, existing methods typically rely on a predetermined, fixed number of shots. This static approach often fails to adapt to the varying difficulty of different queries, leading to either insufficient context or interference from noise. Furthermore, the prohibitive computational and memory costs of long contexts severely limit Many-Shot's feasibility. To address the above limitations, we propose AdapShot, which dynamically optimizes shot counts and leverages KV cache reuse for efficient inference. Specifically, we design a probe-based evaluation mechanism that utilizes output entropy to determine the optimal number of shots. To bypass the redundant prefilling computation during both the probing and inference phases, we incorporate a semantics-aware KV cache reuse strategy. Within this reuse strategy, to address positional encoding incompatibilities, we introduce a decoupling and re-encoding method that enables the flexible reordering of cached key-value pairs. Extensive experiments demonstrate that AdapShot achieves an average performance gain of around 10% and a 4.64x speedup compared to state-of-the-art DBSA.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：Many-Shot In-Context Learning (Many-Shot ICL) 通过向大型语言模型（LLM）提供大量示例（数百甚至上千个）来解锁其推理潜力，性能可媲美微调方法。然而，现有方法普遍采用**固定的示例数量**，导致两个核心问题：
  1. **适应性不足**：不同查询的难度各异，固定示例数量会导致简单查询下冗余计算（甚至引入噪声），而困难查询下上下文不足。
  2. **效率瓶颈**：长上下文带来 $O(n^2)$ 的自注意力计算开销和 GPU 内存压力，使得 Many-Shot ICL 的实际部署变得高昂甚至不可行。
- **整体含义**：本文旨在解决“如何**自适应**地为每个查询分配最优示例数量，同时**高效**地利用大上下文”这一关键问题，推动 Many-Shot ICL 从静态、昂贵的范式转向动态、轻量的实用方案。

## 2. 论文提出的方法论

### 核心思想
AdapShot 包含三个关键技术模块：
1. **基于探测的动态评估机制**（Probe-based Dynamic Evaluation）：利用输出熵衡量模型置信度，以确定每个查询所需的最小示例数。
2. **语义感知的 KV 缓存重用**（Semantically-Aware KV Cache Reuse）：离线预计算所有示例的 Key/Value 表示，在线推理时通过注意力分数进行语义排序与选择，避免重复预填充。
3. **位置解耦与重编码**（Position Decoupling and Re-encoding）：利用 RoPE（旋转位置编码）的旋转合成性质，对缓存的 KV 对进行低成本向量旋转，以适配动态顺序，无需重新计算 Key。

### 关键技术细节

#### ① 探测机制（Probe-based Dynamic Evaluation）
- **步骤**：对查询 $q$，先对全局示例池 $S$ 进行语义排序（使用第②步的注意力分数）。然后以步长 $n$ 逐步增加示例数量，构造探测上下文 $C_{\text{probe}}^k$，并在末尾添加探测提示（如“Based on the above information, are you confident enough to answer?”）。
- **置信度判据**：模型仅生成一个 token（“Yes”或“No”），计算其输出熵 $H_k = -\sum_{c\in\{\text{Yes},\text{No}\}} p(c|C_{\text{probe}}^k)\log p(c|C_{\text{probe}}^k)$。当 $H_k \le \tau$（预设阈值）时停止，否则继续增加示例。
- **并行优化**：利用树结构注意力（Tree-structured Attention）实现单次前向传播中并行评估多个候选示例数量，降低延迟。

#### ② 语义感知的 KV 缓存重用
- **离线阶段**：对每个示例 $\text{shot}_i$，独立执行预填充，保存其在所有层、所有注意力头的 Key 和 Value 张量。
- **在线阶段**：对查询 $q$ 编码得到 Query 向量，计算与每个示例 Key 的注意力分数并取平均，得到相关性得分 $s^{(i)}$；按得分降序选取前 $k$ 个示例作为活动集，拼接其缓存 KV。

#### ③ 位置解耦与重编码
- **问题**：独立预填充时每个示例的位置编码从 0 开始；在线拼接后逻辑位置偏移，导致位置冲突。
- **解法**：利用 RoPE 的旋转可加性：对偏移 $\Delta$，将缓存 Key 向量 $K_{\text{old}}$ 通过如下公式变换：
  $$K_{\text{new}} = K_{\text{old}} \odot \cos(\Delta \theta) + \operatorname{rotate\_half}(K_{\text{old}}) \odot \sin(\Delta \theta)$$
  其中 $\theta_i = 10000^{-2i/d}$。该操作仅需向量旋转开销，无需重新计算 Key。

## 6. 论文的主要结论与发现

1. **自适应优于固定**：AdapShot 在所有 7 个任务上超越固定多示例基线，平均性能比最先进的 DBSA 提升约 10%。
2. **效率大幅提升**：相比 512-shot 基线，最高加速 9.12×；相比 DBSA，平均加速 4.64×。
3. **位置重编码至关重要**：移除 PDR 模块后性能显著下降（如 GSM8K 从 0.649 降至 0.456），证明动态重排序必须配合正确的位置编码。
4. **语义 KV 缓存是效率关键**：移除 SAKVC 后推理时延增加，验证了重复预填充的冗余。
5. **更大模型对冗余更敏感**：在 32B 模型上，AdapShot 仅用约 60 示例达到 86.41% EM，而固定 256-shot 仅为 67.91%，说明大模型更易受噪声干扰。
