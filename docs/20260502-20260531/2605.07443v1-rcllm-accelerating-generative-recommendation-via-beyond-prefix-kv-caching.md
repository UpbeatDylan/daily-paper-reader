## 摘要
大型语言模型（LLM）正将推荐从排序任务转变为生成式任务，但工业部署仍受限于处理长个性化提示的高延迟。标准前缀缓存效果有限，因为推荐工作负载中的重用通常跨用户历史和项目上下文非连续。我们提出RcLLM，一种支持超越前缀KV缓存的生成式推荐分布式推理系统。RcLLM将提示分解为可重用块，并通过分层分布式存储设计支持大型项目目录：紧凑的用户历史缓存被复制以实现零延迟检索，而大量项目缓存使用基于相似性的放置进行分片。为减少冗余的二次注意力计算，RcLLM结合了基于亲和性的全局调度器（改善数据局部性）和选择性注意力机制（纠正近似误差）。在真实数据集上的实验表明，与最先进的前缀缓存系统相比，RcLLM将首令牌时间（TTFT）降低了1.31倍至9.51倍，实现了实时服务，而对推荐准确性的影响可忽略不计。

## Abstract
Large Language Models (LLMs) are transforming recommendation from ranking into a generative task, but industrial deployment remains limited by the high latency of processing long, personalized prompts. Standard prefix caching provides limited benefit because reuse in recommendation workloads is often non-contiguous across user histories and item contexts. We present RcLLM, a distributed inference system for generative recommendation with Beyond-Prefix KV Caching. RcLLM decomposes prompts into reusable blocks and supports large item catalogs with a stratified distributed storage design: compact user-history caches are replicated for zero-latency retrieval, while massive item caches are sharded using similarity-aware placement. To reduce redundant quadratic attention computation, RcLLM combines an affinity-based global scheduler that improves data locality with a selective attention mechanism that corrects approximation errors. Experiments on real-world datasets show that RcLLM reduces Time-To-First-Token (TTFT) by 1.31x-9.51x compared with state-of-the-art prefix caching systems, enabling real-time serving with negligible impact on recommendation accuracy.

- **研究背景**：大型语言模型（LLM）正在推动推荐系统从传统检索+排序范式向生成式范式转变，如OneRec、HLLM等框架。生成式推荐通过读取用户历史、项目描述等丰富上下文，利用LLM推理能力生成排序结果，具备更强的可解释性和个性化能力。
- **核心问题**：生成式推荐面临严重的推理延迟瓶颈。推荐提示（prompt）通常包含数千词元（tokens）的用户历史、候选项目描述等，其预填充（prefill）阶段因自注意力二次复杂度（O(n²)）计算开销巨大。且推荐场景是“预填充重、解码轻”，首令牌时间（TTFT）成为主要延迟瓶颈，难以满足工业界<100ms的严格延迟SLO（服务等级目标）。
- **标准前缀缓存的局限**：现有前缀缓存（如vLLM、SGLang）仅重用连续前缀（如系统指令），但推荐提示中相同内容（如候选项目描述、语义相似的用户评价）因位置不固定（非前缀）而无法被命中，导致近零的缓存命中率，被迫重复计算。
- **研究目标**：设计一种超越前缀的KV缓存系统，实现非连续内容的高效重用，大幅降低TTFT，使生成式推荐符合工业延迟约束。

## 2. 论文方法论：核心思想、关键技术细节

### 核心思想
- 将推荐提示分解为两个可重用的KV组成部分：
  1. **用户历史**：语义上重复（如评价中高频出现的短语），可压缩为语义原型缓存。
  2. **候选项目**：静态不变，可离线预计算并分片存储。
- 采用**分层分布式KV存储**：紧凑的用户历史缓存全复制到每个节点（零延迟访问），大规模项目缓存使用**相似性感知放置**（分片存储，减少远程获取）。
- 在线阶段通过**亲和性调度**最大化数据局部性，并使用**选择性注意力机制**纠正近似误差。

### 关键技术细节

#### （1）离线阶段：分布式KV构建与放置
- **语义历史KV缓存池**：
  - 对用户历史中的词元进行位置感知嵌入（结合位置编码）。
  - 使用局部敏感哈希（LSH）聚类为约10⁵个语义原型，每个原型预计算KV状态并复制存储在所有节点的CPU内存中。
- **候选项目KV缓存池**：
  - 离线预计算所有项目的KV块（万亿字节级别，见表II）。
  - **放置算法（Algorithm 1）**：
    1. 识别最热门的0.1%项目，全局复制到所有实例。
    2. 其余长尾项目：构建项目相似性图（基于历史共现），使用METIS图划分工具将图划分为k个子图（k为实例数），最小化跨分区边切割，将强相关项目放在同一节点。
    3. 定期重新执行以应对目录变化和流行度漂移。

#### （2）在线阶段：缓存感知分布式推理
- **亲和性全局调度器**：
  - 计算请求与每个实例的**亲和性分数**：`Affinity(R, p) = α·cHit(R,p) + β·(1−Load(p))`，其中cHit为本地缓存命中率，Load为节点负载（如GPU利用率或队列深度），α、β为超参数。
  - 在缓存局部性和负载均衡间动态权衡。
- **零拷贝检索与选择性重计算**：
  - 指令词元：完全重计算。
  - 评价词元：通过LSH匹配最近语义原型，从复制库中获取；实例特定字段（时间戳、ID等）隔离并重计算。
  - 候选项目词元：从本地CPU缓存中通过ID检索，命中则通过零拷贝路径直接传输到GPU，未命中则在线计算。
  - **选择性重计算**：在第一层解码器中计算完整注意力，识别“重头词元”（heavy hitters），其重要性分数定义为：
    \[
    S_i = (1-\lambda)\|A_i\|_1 + \lambda \sum_{M\in\{K,V\}}\|M_i^{new} - M_i^{cached}\|_1
    \]
    其中第二项表示当前上下文与缓存状态的差异。后续层仅重计算这些重头词元及局部滑动窗口内词元的精确注意力，其余复用缓存。
- **端到端预填充流水线**：KV块组装 → 位置调整（如RoPE旋转）→ 选择性重计算；通过将第一层注意力计算与PCI-e传输重叠隐藏延迟。

## 6. 主要结论与发现

- **延迟大幅降低**：在40实例集群上，RcLLM将TTFT中位数（P50）降低1.5×–2.2×（Qwen3-8B），尾部（P99）降低1.4×–2.4×；Qwen-72B上更大模型获益更明显（P99最高9.51×）。相比前缀缓存，RcLLM有效消除冗余二次计算。

- **调度策略关键**：RcLLM的亲和调度在负载均衡与局部性间取得最优平衡，避免纯命中调度导致的队列热点。
- **选择性重计算有效**：仅重计算少量重头词元就能恢复精度，避免全套重计算的代价。

## 7. 优点

### 方法亮点
- **超越前缀的重用设计**：首次针对生成式推荐的非连续复用特性，提出分层KV缓存（历史复制+项目分片），解决了前缀缓存失效问题。
- **离线-在线解耦架构**：大部分计算移至离线预计算，在线仅需检索与少量重计算，降低关键路径延迟。
- **相似性感知放置算法**：利用图谱划分（METIS）将强相关项目分组到同一节点，最大化局部命中率，同时复制热门项目避免热点。
- **亲和性调度公式**：显式量化局部性与负载的权衡，动态适应流量波动，避免单一目标策略的缺陷。
- **选择性注意力机制**：基于第一层注意力分布和语义漂移识重要词元，仅重计算这些锚点，有效抑制近似误差且开销小。
- **零拷贝传输与PCI-e重叠**：优化了跨存储层的数据移动延迟。

### 实验亮点
- 同时覆盖延迟和精度两个维度，分别用两组独立的实现（基于Vidur模拟器 + HuggingFace原型），确保测量的准确性和可行性。
- 丰富的消融实验，验证每个组件的必要性。
- 使用真实工业级的Qwen3-8B和Qwen-72B模型，而非小模型。

## 8. 不足与局限

### 实验覆盖与偏差风险
- **模型局限**：仅使用Qwen3-8B/72B和Llama3-8B部分分析，未涵盖更大参数模型（如Llama3-70B、Qwen-72B已有），但指出更大模型基线延迟已经远超SLO，所以未重点实验。
- **数据集局限**：三个数据集虽然多样，但均为英文通用推荐场景（Amazon、Yelp、Goodreads），未覆盖短视频、新闻、社交等高度动态的内容推荐。
- **硬件假设**：假设100Gbps网络和A100 GPU，实际工业部署可能使用不同配置（如H100、不同网络带宽），结果可能变化。
- **模拟器使用**：延迟实验基于Vidur模拟器而非真实集群，虽经高度校准，但与真实物理部署仍可能存在偏差。

### 应用限制
- **用户历史的语义聚类精度**：LSH聚类依赖历史数据质量，对于罕见短语或新领域可能存在原型匹配不准。
- **项目目录的动态性**：离线放置需定期重执行，在快速变化目录中可能造成短暂性能下降。
- **推荐任务约束**：论文假设输出仅为排序列表（decode轻），若生成式推荐需要输出多轮解释或对话，解码阶段权重增加，本文方法的收益可能减弱。

- **安全与公平性**：未讨论KV重用的公平性影响（如热门项目被复制可能加剧流行度偏差），以及重计算策略对长尾项目公平性的潜在副作用。
