## 摘要
上下文密集型智能体会对键值（KV）缓存造成异常压力：长前缀在多个短轮次中被重复使用，而并发性决定了服务系统能否保持GPU利用率。我们针对这一场景研究了4位KV缓存压缩，采用TurboQuant风格的旋转和码本量化作为质量锚点，以vLLM FP8 KV缓存作为部署锚点。我们报告了三个贡献。首先，我们围绕多轮智能体工作负载构建了4位KV缓存框架，其中任务质量、缓存驻留时间和服务吞吐量必须联合衡量。其次，我们描述了使4位路径稳健所需的实际设计选择，包括非对称K/V处理、Walsh-Hadamard旋转、QJL移除以及块尺度变体。第三，我们提出了在AMD GPU上的服务优化，包括优化的解码注意力核和UltraQuant——一种使用FP8查询、FP4 KV张量、UE8M0组尺度和CDNA4原生缩放MFMA支持的FP4近似路径。在长上下文、多轮智能体工作负载上，UltraQuant在缓存压力大的后期轮次中，将P50首令牌生成时间减少了3.47倍（所有轮次平均减少2.3倍），并将输出吞吐量相比FP8 KV基线提升了1.63倍。

## Abstract
Context-heavy agents place unusual pressure on the key-value (KV) cache: long prefixes are reused across many short turns, while concurrency determines whether the serving system can keep GPUs utilized. We study 4-bit KV-cache compression for this setting, using TurboQuant-style rotation and codebook quantization as a quality anchor and vLLM FP8 KV caching as the deployment anchor. We report three contributions. First, we frame 4-bit KV caching around multi-round agent workloads where task quality, cache residency, and serving throughput must be measured jointly. Second, we describe the practical design choices needed to make the 4-bit path robust, including asymmetric K/V treatment, Walsh-Hadamard rotation, QJL removal, and block-scale variants. Third, we present serving optimizations on AMD GPUs, including optimized decode-attention kernels and UltraQuant, an FP4 approximation path that uses FP8 queries, FP4 KV tensors, UE8M0 group scales, and native scaled-MFMA support on CDNA4. On a long-context, multi-turn agentic workload, UltraQuant cuts P50 time-to-first-token by 3.47x in the cache-pressured late rounds (2.3x across all rounds) and raises output throughput by 1.63x over the FP8 KV baseline.

### 2. 论文提出的方法论：核心思想、关键技术细节

- **整体思路**：采用TurboQuant风格的旋转+码本量化作为算法质量锚点，但针对服务端部署进行了实用化改进，最终提出两套4位缓存方案：
  1. **Ultra-TurboQuant (Ultra-TQ)**：保留TurboQuant的码本表示，通过优化布局、查找表（LUT）和MFMA调度提升性能，并引入校准质心提升精度。
  2. **UltraQuant**：用FP4微张量近似替换码本，使反量化过程融入CDNA4原生缩放MFMA指令，实现高效服务。
- **关键技术细节**：
  - **非对称K/V处理**：键值错误对softmax分布的影响不同，分别采用不同量化策略。
  - **Walsh-Hadamard旋转**：固定正交变换将离群能量分散到整个通道，使旋转后的分布接近高斯分布，便于4位标量量化。
  - **校准质心（Ultra-TQ）**：基于模型激活的统计特性，通过Lloyd–Max算法拟合16个质心，仅对10%的层进行校准（每MSE较高的层），降低量化误差约10.3%。
  - **块尺度设计（UltraQuant）**：将头维度分为32通道组（group-32），每组使用一个UE8M0字节作为缩放因子，缩放规则：`value = code × 2^scale`，scale为有符号整数（偏移127）。这使得反量化直接通过MFMA指令的指数移位完成，无需软件查找表。
  - **全局常数优化**：对于所有模型和头，使用一个全局常数c = 0.156将组内最大绝对值映射到FP4网格，该常数通过最小化重建误差离线求解（公式略），效果优于固定FP4网格。
  - **消除逐令牌L2归一化**：因为块尺度已吸收所有幅度变化，旋转后的分布只依赖于头维度d，无需额外存储每令牌范数。
- **编码/解码算法**（以UltraQuant为例）：
  - 编码：旋转K → 按32通道分组 → 每个组计算最大绝对值m → 计算UE8M0指数E = round(log2(c·m)) → 每个坐标舍入到最近FP4码点 → 存储2个FP4码/字节 + 1字节UE8M0。
  - 解码：Q旋转并转为FP8 → 加载码字和UE8M0字节 → 通过缩放MFMA指令直接执行QK和PV乘积，不显式反量化。
