## 摘要
上下文密集型智能体对键值（KV）缓存施加了不寻常的压力：长前缀在多个短轮次中被重用，而并发性决定了服务系统能否保持GPU利用。我们研究了针对这一场景的4位KV缓存压缩，使用TurboQuant风格的旋转和码本量化作为质量锚点，以vLLM FP8 KV缓存作为部署锚点。我们报告了三项贡献。首先，我们将4位KV缓存框定为多轮智能体工作负载，其中任务质量、缓存驻留和服务吞吐量必须联合衡量。其次，我们描述了使4位路径稳健所需的实际设计选择，包括非对称K/V处理、Walsh-Hadamard旋转、QJL移除以及块缩放变体。第三，我们提出了在AMD GPU上的服务优化，包括优化的解码注意力内核和UltraQuant，这是一个使用FP8查询、FP4 KV张量、UE8M0组缩放以及CDNA4上原生缩放MFMA支持的FP4近似路径。在长上下文、多轮智能体工作负载上，UltraQuant在缓存压力大的后期轮次中将P50首次令牌延迟降低了3.47倍（所有轮次平均降低2.3倍），并将输出吞吐量相比FP8 KV基线提高了1.63倍。

## Abstract
Context-heavy agents place unusual pressure on the key-value (KV) cache: long prefixes are reused across many short turns, while concurrency determines whether the serving system can keep GPUs utilized. We study 4-bit KV-cache compression for this setting, using TurboQuant-style rotation and codebook quantization as a quality anchor and vLLM FP8 KV caching as the deployment anchor. We report three contributions. First, we frame 4-bit KV caching around multi-round agent workloads where task quality, cache residency, and serving throughput must be measured jointly. Second, we describe the practical design choices needed to make the 4-bit path robust, including asymmetric K/V treatment, Walsh-Hadamard rotation, QJL removal, and block-scale variants. Third, we present serving optimizations on AMD GPUs, including optimized decode-attention kernels and UltraQuant, an FP4 approximation path that uses FP8 queries, FP4 KV tensors, UE8M0 group scales, and native scaled-MFMA support on CDNA4. On a long-context, multi-turn agentic workload, UltraQuant cuts P50 time-to-first-token by 3.47x in the cache-pressured late rounds (2.3x across all rounds) and raises output throughput by 1.63x over the FP8 KV baseline.

# 论文详细总结：UltraQuant: 面向上下文密集型智能体的4位KV缓存

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：上下文密集型智能体（如代码助手、网页浏览Agent）需要长上下文支持，频繁重用长前缀，导致KV缓存成为HBM瓶颈。高并发场景下，FP8 KV缓存（2×压缩）仍不足以在有限显存中容纳足够多的会话前缀，导致频繁重新预填充，增加延迟。
- **整体含义**：将KV缓存压缩至4-bit可以在相同显存下容纳更多上下文，提高缓存驻留率，从而改善服务吞吐和首令牌延迟。然而，4-bit方法往往需额外反量化开销，可能抵消压缩收益。论文旨在设计一种在任务质量、缓存驻留和服务吞吐三者间取得平衡的可部署4-bit方案。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：采用TurboQuant风格的旋转（Walsh-Hadamard）使数据分布近似高斯，然后使用硬件原生FP4 (E2M1) 码本 + UE8M0组缩放，使反量化折叠进CDNA4矩阵核的scaled-MFMA指令，避免软件反量化。
- **关键技术细节**：
  - **非对称K/V处理**：Keys先旋转，再量化；Values不旋转，直接量化。
  - **旋转**：使用Walsh-Hadamard变换分散离群值能量。
  - **移除QJL**：TurboQuant原使用QJL的码本查找，UltraQuant改为固定FP4网格。
  - **块缩放**：每32通道一组，用UE8M0字节存储缩放指数（基于组内absmax和全局常数c计算）。
  - **全局常数优化**：单常数c=0.156，离线校准，跨模型通用，使FP4网格匹配旋转后的分布。
  - **Ultra-TQ**：保留TurboQuant码本，但通过优化内核（GQA感知布局、向量化LUT、MFMA调度）提高效率。
- **算法流程**（文字说明）：
  - 编码：Keys通过旋转 → 分组（32通道）→ 每组计算absmax m → 缩放指数E = round(log2(c·m)) → 将每个值量化到FP4网格。Values类似但不旋转。
  - 解码：查询先旋转并量化为FP8；加载FP4码和UE8M0缩放；使用scaled-MFMA指令同时执行QK和PV，反量化由硬件完成。

## 6. 主要结论与发现

- **服务性能**：UltraQuant在长期上下文（后期轮次）中P50 TTFT相比FP8 KV降低3.47倍（所有轮次平均2.3倍），输出吞吐量提高1.63倍。
- **硬件效率**：FP4路径性能与硬件FP8 KV几乎持平（1.38× vs 1.37× BF16），但KV显存占用减半；Ultra-TQ稍慢但仍优于开源TurboQuant。
- **任务精度**：在GPQA、LCB-128K、MATH500上接近BF16，但在AIME25上退化明显（-10~-13 pp）。精度表现依赖基准，非均匀近无损。
- **关键发现**：端到端服务优化中，硬件原生码本（FP4+缩放）比算法最优码本更高效；缓存驻留比单纯压缩更重要。

## 7. 优点

- **方法创新**：首次将FP4网格与UE8M0缩放结合到KV缓存中，利用硬件scaled-MFMA消除软件反量化，实现了接近硬件FP8的速度同时获得4倍压缩。
- **系统视角**：从端到端服务出发，联合衡量任务质量、缓存驻留和吞吐，而非仅关注压缩率。
- **设计简化**：使用单个全局常数，跨模型泛化；无需逐层或逐模型校准码本。
- **消融实验充分**：通过块缩放、全局常数、校准码本等消融，验证了各设计选择的必要性。
- **性能提升显著**：在AMD GPU上有效压低了长上下文下的重新预填充开销。

## 8. 不足与局限

- **精度退化**：在AIME25等复杂推理基准上，UltraQuant比BF16退化超过10个百分点，并非近无损。作者如实报告为局限性。
- **缩放常数固定**：全局常数c=0.156可能不是逐层最优，论文未探索逐层或逐头精细优化。
- **速度优势条件**：UltraQuant相比FP8的速度优势仅在上下文长度超过FP8缓存容量时才显著；短上下文场景下FP8可能更快。
- **GPU厂商局限**：仅在AMD MI355X上验证，未在NVIDIA GPU上测试，泛化性未知。
- **校准范围有限**：仅对10%的层应用校准码本，效果有限。
- **代码未开源**：尽管提供算法伪代码，但具体内核实现未公开，可复现性受限。
- **并发度单一**：仅测试并发度64和32会话，未系统扫描不同并发压力下的行为。
