## 摘要
自注意力是Transformer性能的核心，并且在长上下文长度下通常是Transformer中最昂贵的部分，因为其成对令牌交互的计算量与序列长度呈二次方关系。标准密集注意力还对每个令牌应用相同的注意力头集合，无论令牌的难度或信息内容如何。这种均匀激活会浪费计算资源，尤其在序列变长且注意力成本迅速增加时。我们提出分组查询专家（GQE），一种在分组查询注意力（GQA）之上的混合专家层。在每个GQA组内，一个路由为每个令牌选择k个查询头专家，而所有键值（KV）头保持密集且不变。因此，GQE保留了GQA的KV缓存优势，并且仅减少了活跃的查询头计算。在固定30B令牌预算、250M参数规模下，GQE在下游准确率上与全活跃的GQA基线相当，同时每个令牌仅激活一半的查询头。

## Abstract
Self-attention is central to Transformer performance and is often the most expensive part of the Transformer at long context lengths because its pairwise token interactions scale quadratically with sequence length. Standard dense attention also applies the same set of attention heads to every token regardless of token difficulty or information content. This uniform activation can waste compute, especially as sequences grow longer and attention cost increases rapidly. We propose Grouped Query Experts (GQE), a mixture-of-experts layer on top of grouped-query attention (GQA). Within each GQA group, a router selects k query-head experts per token while all key-value (KV) heads remain dense and unchanged. Thus, GQE keeps the KV cache benefits of GQA and reduces only the active query-head computation. On a fixed 30B token budget at the 250M parameter scale, GQE matches the all-active GQA baseline in downstream accuracy while activating half the query heads per token.

### 1. 论文的核心问题与整体含义（研究动机和背景）
- **核心问题**：Transformer 的自注意力机制对每个 token 都激活所有注意力头，导致计算量随序列长度二次增长，尤其在长上下文场景中成本过高。且这种均匀激活忽略了不同 token 对注意力头的差异化需求（如停用词与关键内容）。
- **整体含义**：本文旨在将混合专家（MoE）的稀疏计算思想从 MLP 模块迁移到注意力模块，在保持 GQA 的 KV 缓存优势的同时，仅激活部分查询头，从而在不损失下游精度的条件下减少计算量。

### 2. 论文提出的方法论：核心思想、关键技术细节
- **核心思想**：在 GQA 的每个组内，将多个查询头视为专家，由路由器为每个 token 选择 top-k 个专家进行计算，而所有 KV 头保持密集且不变。
- **关键技术细节**：
  - **分组专家池**：假设有 G 个 KV 组，每个组内有 M 个查询头专家，总专家数 N = M × G。每个组内路由器对 token 输出概率，再取 top-k 个专家。
  - **输出构造**：选中的专家输出直接硬连接（hard concatenation）；额外增加一个**重新归一化的加权求和槽**（renormalized weighted-sum slot），利用路由器概率加权组合选中输出，提供可微梯度路径；此外包含一个**始终激活的共享注意力头**（shared head）作为稳定锚点。
  - **路由辅助损失**：使用负载均衡损失防止路由器崩溃到固定专家。
  - **活跃计算量**：k=1 时，每 token 激活 kG = 8 个路由专家 + 1 个共享头 = 9/16 ≈ 56% 的查询头计算。
  - 输出投影层 WO 的输入槽数量由 16 变为 10（8个硬连接 + 1个加权和 + 1个共享头），因此参数数略有变化。

### 6. 论文的主要结论与发现
- **准确率**：GQE 在激活 9/16 查询头的情况下，平均下游准确率（56.04）与 GQA 基线（55.86）持平，甚至略高 0.18 个百分点。
- **吞吐量**：长序列（4k 以上）可达到约 1.7–1.8× 的 prefill 加速。
- **关键发现**：路由成功依赖于**重新归一化的加权和槽**（提供梯度信号）和**共享头**（稳定训练）。移除任一者，稀疏路由都会导致准确率下降。

### 7. 优点
- **方法简洁高效**：直接嫁接 MoE 到 GQA 的查询头部分，不改动 KV 路径，保留 GQA 的 KV 缓存优势。
- **消融实验清晰**：逐步揭示为什么简单的硬连接或加权拼接无法工作，而加入归一化槽和共享头后恢复性能，因果解释强。
- **长上下文受益**：由于减少的是查询侧计算，随序列增长节约比例增大，与理论预期一致。
- **训练共存**：路由器与专家从零开始联合训练，无需后期转换或剪枝。
