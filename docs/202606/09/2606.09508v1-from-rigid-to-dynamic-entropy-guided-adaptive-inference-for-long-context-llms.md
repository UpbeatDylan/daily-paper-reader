## 摘要
现有的针对长上下文LLM推理的稀疏注意力和KV缓存压缩方法通常在所有注意力头上应用固定的稀疏模式或均匀预算，忽视了头和上下文之间注意力行为的显著差异。我们观察到注意力头中存在两种不同的熵模式：刚性头，其熵在输入片段中保持在零附近；动态头，其熵波动显著。关键在于，这些类型的分布依赖于上下文，无法离线预判。因此，我们提出EntropyInfer，一种无需训练框架，在预填充阶段利用注意力熵在单个头和片段粒度上自适应分配计算。对于解码阶段，我们引入了一种潜在KV缓存压缩方案，利用生成的输出令牌（而非仅预填充令牌）来识别并保留最关键的缓存条目。在Llama、Qwen和openPangu模型系列上的大量实验表明，EntropyInfer在包括SnapKV、AdaKV和CritiPrefill在内的基线上持续取得更好表现，在10万令牌以上实现了高达2.39倍的端到端加速，同时与全注意力相比质量下降最小。代码已发布在https://github.com/SHA-4096/EntropyInfer。

## Abstract
Existing sparse attention and KV cache compression methods for long-context LLM inference typically apply fixed sparsity patterns or uniform budgets across all attention heads, overlooking the substantial variation in attention behavior among heads and contexts. We observe two distinct entropy patterns among attention heads: Rigid Heads, whose entropy stays near zero across input segments, and Dynamic Heads, whose entropy fluctuates significantly. Crucially, the distribution of these types is context-dependent and cannot be predetermined offline. We therefore propose EntropyInfer, a training-free framework that uses attention entropy to adaptively allocate compute at the granularity of individual heads and segments during prefilling. For decoding, we introduce a latent KV cache compression scheme that leverages generated output tokens, rather than prefill tokens alone, to identify and retain the most critical cache entries. Extensive experiments on Llama, Qwen and openPangu model series show that EntropyInfer consistently outperforms baselines including SnapKV, AdaKV, and CritiPrefill, achieving up to 2.39$\times$ end-to-end speedup beyond 100k tokens with minimal quality degradation compared to full attention. The code is released in https://github.com/SHA-4096/EntropyInfer.

## 2. 论文提出的方法论

### 核心思想
利用注意力熵作为在线信号，在预填充阶段按头、按片段动态分配稀疏注意力预算；在解码阶段，利用生成后的输出令牌重新排序KV缓存条目，进行潜在压缩。

### 关键技术细节

- **预填充阶段：熵引导的稀疏预填充**
  1. **观察注意力矩阵构建**：将Q、K分段（segment size \(L_S\)），每段取最大值和最小值组成表示向量，计算四种注意力得分的最大值作为观察矩阵 \(A_{obs}\)（算法1、2）。
  2. **逐行熵计算**：对 \(A_{obs}\) 的每行计算 Shannon 熵 \(H(a_{i:}) = -\sum_j a_{ij}\log a_{ij}\)。
  3. **头部分类**：根据整头最大熵是否超过阈值 \(e_t=10^{-5}\)，分为刚性头（固定基础预算 \(B_h\)）和动态头。
  4. **动态头预算分配**（算法3）：动态头中，相邻片段熵波动 \(|\Delta e|/e_{i-1}\) 越大，预算增加越多，受限于 \([ \min(B_0, 3B_0) ]\)；刚性头所有片段预算相同。
  5. **稀疏注意力**：每个片段内选取 Top-K 块（block）进行计算，块大小为 \(L_B\)。

- **解码阶段：潜在KV缓存压缩**（算法4）
  1. 生成 \(N_d\) 个输出令牌后，构建观察窗口（包含输出令牌）。
  2. 使用观察窗口查询矩阵与原始 KV 计算重要性分数 \(w\)（聚合行和）。
  3. 选取 Top-\(B_d\) 个索引，保留对应 KV 缓存，其余丢弃。
  4. 延迟压缩时机，利用生成令牌的注意力模式更准确选择重要缓存。

- **复杂度**：总计算复杂度为 \(O((\frac{1}{L_S L_B} + \frac{3}{2L_S L_B}\log\frac{B_h}{L_B})N^2 + 3B_h N)\)，当 \(N\) 很大时接近线性。

## 6. 论文的主要结论与发现

- **主要发现**：注意力头可分为“刚性头”和“动态头”，且分类依赖于输入上下文，无法离线先验。
- **方法有效性**：EntropyInfer 在 LongBench 和 InfiniteBench 上平均得分优于 SnapKV、AdaKV、CritiPrefill，且与全注意力相比质量下降极小（甚至部分任务超过全注意力）。
- **效率提升**：在 100k+ token 下实现最高 **2.39倍端到端加速**，且加速比随上下文增长而增大。
- **泛化能力**：在 openPangu 系列（1B/7B）上同样获得显著加速和质量保持，验证了方法在不同架构上的适用性。
