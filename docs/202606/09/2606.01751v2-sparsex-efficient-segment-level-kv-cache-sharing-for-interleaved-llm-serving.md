## 摘要
在长上下文LLM服务中，预填充阶段通常主导首字延迟和计算成本。尽管vLLM/PagedAttention中的前缀缓存已被广泛用于重用相同的提示前缀，但实际应用中的重复内容经常以非前缀、跨请求、跨轮次和跨代理片段的形式出现，这使得传统缓存机制不足。本文提出SparseX，一种面向常见服务场景的段级KV缓存共享方法。SparseX使用连续令牌段作为重用单元，并利用KV缓存重用工作负载中自然出现的Sparse-Q索引来估计需要纠正的关键令牌。基于此估计，SparseX在单次前向传播中执行稀疏KV重计算，从而在复杂的交错重用模式下恢复跨段上下文交互，同时避免额外的模型或用于令牌选择的单独预处理阶段。SparseX进一步基于层特定阈值实现了全+稀疏混合注意力模式：早期层保持全注意力以获得更稳定的令牌重要性信号，后期层切换到稀疏重计算以提高复杂长上下文任务上的重用质量。我们在vLLM之上实现了SparseX-vLLM，将段级缓存查找、PagedAttention管理、RoPE对齐、Sparse-Q令牌选择和FlashAttention后端集成到统一执行路径中。SparseX与模型无关、无需训练、与前缀缓存兼容，并为常见的在线服务场景（包括多轮聊天、检索增强生成（RAG）和代理工作流）提供统一支持。

## Abstract
In long-context LLM serving, the prefill stage often dominates time-to-first-token and computational cost. Although Prefix Cache in vLLM/PagedAttention has been widely used to reuse identical prompt prefixes, repeated content in practical applications frequently appears as non-prefix, cross-request, cross-turn, and cross-agent segments, which makes conventional cache mechanisms insufficient. This paper presents SparseX, a segment-level KV Cache sharing method for common serving scenarios. SparseX uses contiguous token segments as reuse units and exploits Sparse-Q indices that naturally arise in KV Cache reuse workloads to estimate the key tokens that require correction. Based on this estimate, SparseX performs Sparse-KV Recomputation within a single forward pass, thereby restoring cross-segment contextual interactions under complex interleaved reuse patterns while avoiding additional models or separate preprocessing stages for token selection. SparseX further implements a full+sparse hybrid attention mode based on a layer-specific threshold: early layers retain full attention to obtain a more stable token-importance signal, and later layers switch to sparse recomputation to improve reuse quality on complex long-context tasks. We implement SparseX-vLLM on top of vLLM, integrating segment-level cache lookup, PagedAttention management, RoPE alignment, Sparse-Q token selection, and FlashAttention backends into a unified execution path. SparseX is model-agnostic, training-free, and compatible with Prefix Cache, and it provides unified support for common online serving scenarios including multi-round chat, retrieval-augmented generation (RAG), and agent workflows.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：长上下文LLM在线服务中，预填充（Prefill）阶段通常主导首字时延（TTFT）和计算成本。虽然前缀缓存（Prefix Cache）可重用相同的前缀，但实际场景中重复内容经常以**非前缀**形式出现——例如多轮对话中的历史消息、RAG中的知识片段、智能体工作流中的中间结果等，这些内容分布在不同位置，传统前缀缓存无法命中。
- **背景**：现有非前缀KV缓存重用方法存在三个关键不足：
  - 对复杂交错重用模式支持不足（例如多个重用段与非重用段交替出现）；
  - 与推理引擎（如vLLM）深度集成不够，易被工程开销抵消收益；
  - 针对特定场景设计，缺乏通用性。
- **整体含义**：SparseX旨在提供一种**模型无关、无需训练、兼容前缀缓存**的段级KV缓存共享方法，覆盖多轮对话、RAG、智能体工作流等常见在线服务场景，显著降低预填充延迟和计算开销，同时保持输出质量接近完全重计算。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
- 将重复内容组织为**连续令牌段**作为重用单元。
- 利用KV缓存重用过程中自然形成的**Sparse-Q索引**：非重用段必须计算Query向量并参与注意力，而重用段的KV缓存可直接读取。这些Sparse-Q（非重用段的Query）可作为关键令牌估计的天然信号，指导哪些重用令牌需要被稀疏重计算。
- 通过**单次前向传播**内的Sparse-KV重计算恢复跨段上下文交互，避免额外模型或预处理阶段。

### 关键技术细节
1. **RoPE对齐**：将缓存段的Key向量通过增量旋转对齐到新位置，Value直接复制。复杂度O(|S|·dk)，通过融合拷贝-对齐核实现高效执行。
2. **Sparse-Q驱动的令牌选择**：
   - 定义非重用位置集合Inr，计算Sparse-Q（Q[Inr]）到所有序列Key的注意力分数，聚合得到每个Key位置的全局注意力强度sj，选取Top-k作为关键令牌集合Skey。
   - 复杂度O(|Inr|·T·d)，远低于完全注意力的O(T²d)。
   - 当提示尾部全是重用令牌时，回退到使用最后一个重用段末尾64个Query的注意力估计。
3. **Overflow近似**：为补偿局部位置注意力，对每个非重用段区间首尾各扩展一个块，将边界区域纳入重计算范围（Sov）。
4. **全+稀疏混合注意力**：
   - 基于层特定阈值ℓ*，早期层（1~ℓ*）执行完全注意力以稳定令牌重要性信号；在边界层进行一次Sparse-Q估计；后期层（ℓ*+1~L）仅重计算R=Inr∪Skey∪Sov∪Stail中的令牌，其余直接使用对齐缓存。
   - 阈值设置：小模型（7B）约15%~20%层，大模型/MoE约10%~15%层。
5. **算法流程**（以单次预填充为例）：
   - 输入：提示令牌序列、候选重用段、命名空间键、缓存段KV张量、原始/新位置、层边界ℓ*、Top-k预算k、块大小b。
   - 输出：首个生成令牌及更新后的KV缓存。
   - 步骤：
     1. 段查找：匹配虚拟块，确定重用位置Ireuse和非重用位置Inr。
     2. RoPE对齐：对每个匹配段按位移Δ对齐Key。
     3. 阶段1：执行1~ℓ*层完全因果注意力。
     4. 阶段2：在边界层用Sparse-Q估计关键令牌（Top-k），加上Overflow和尾回退得到最终重计算集R。
     5. 阶段3：对ℓ*+1~L层，仅重计算R中的令牌，其余使用对齐缓存，执行稀疏因果注意力。
     6. 生成首个令牌。

## 6. 主要结论与发现

1. **质量接近完全重计算**：在LOCOMO上SparseX得分0.67（完全重计算0.68），在RULER平均0.94~0.98，与完全重计算几乎持平，显著优于朴素重用（0.69~0.79）、CacheBlend（0.79~0.86）和EPIC（0.76~0.82）。
2. **TTFT大幅降低**：在多轮对话中，完全重计算TTFT为7~9秒，SparseX降至0.56~0.87秒，下降约10~15倍。
3. **处理复杂交错模式有效**：在RULER变量追踪（VT）子任务中，SparseX以0.94~1.00准确率远超其他方法（33%~66%），证明其对随机位置关键信息的定位能力。
4. **全+稀疏混合注意力提升稳定性**：启用混合注意力后，LOCOMO得分从0.57/0.64提升至0.61/0.67，RULER平均从0.86~0.91提升至0.94~0.98，对复杂检索任务帮助显著。
5. **多智能体系统通用性**：在AutoGen和MAD两种框架上，SparseX在8个任务上总体与原始（Vanilla）执行质量相近，部分任务甚至略高，证明了系统级优化的可行性。
