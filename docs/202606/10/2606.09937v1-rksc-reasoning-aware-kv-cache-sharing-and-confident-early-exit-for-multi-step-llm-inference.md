## 摘要
我们引入了RKSC（感知推理的KV缓存共享），这是一个无需训练的推理框架，消除了多分支LLM推理流程中的两种结构性冗余。ASKS（注意力相似性KV共享）一次计算前缀KV缓存，并通过隐藏状态余弦相似度将其广播到所有语义相似的分支，严格推广了vLLM和SGLang使用的令牌精确前缀缓存。CGEE（置信门控早退）应用了两种互补的退出机制：（1）当跨分支的生成置信度具有决定性时，完全跳过验证前向传播；（2）当逐层熵稳定时，在中间层终止验证传播，使用Transformer主干上的轻量级钩子。RSBCM（推理选择性块缓存管理器）通过注意力加权深度优先驱逐防止缓存无限增长。在五个模型系列（7B-10B）、四个基准测试和1000个评估问题上，RKSC相对于无KV基线实现了平均3.008倍的加速（峰值3.990倍），相对于vLLM等效前缀缓存平均提升1.66倍，CGEE导致的错误率仅为0.37%（1616次验证调用中6个错误）。无需微调或架构更改。代码可在https://github.com/AnirudhSekar/RKSC获取。

## Abstract
We introduce RKSC (Reasoning-Aware KV Cache Sharing), a training-free inference framework that eliminates two structural redundancies in multi-branch LLM reasoning pipelines. ASKS (Attention-Similarity KV Sharing) computes the prefix KV cache once and broadcasts it to all semantically similar branches via hidden-state cosine similarity, strictly generalising the token-exact prefix caching used by vLLM and SGLang. CGEE (Confidence-Gated Early Exit) applies two complementary exit mechanisms: (1) it skips the verification forward pass entirely when generation confidence is decisive across branches, and (2) it terminates the verification pass at an intermediate layer when per-layer entropy stabilises, using lightweight hooks on the transformer backbone. RSBCM (Reasoning-Selective Block Cache Manager) prevents unbounded cache growth via attention-weighted depth-priority eviction. Across five model families (7B-10B), four benchmarks, and 1,000 evaluated problems, RKSC achieves a mean speedup of 3.008x over the No-KV baseline (peak 3.990x), a 1.66x mean improvement over vLLM-equivalent prefix caching, with a CGEE-induced error rate of only 0.37% (6 errors out of 1,616 verify calls). No fine-tuning or architecture changes are required. Code is available at https://github.com/AnirudhSekar/RKSC.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：多步LLM推理（如Tree of Thoughts、DeepSeek-R1等）通常生成多个推理分支，但现有推理系统存在两种结构性冗余：
  - **跨分支KV缓存冗余**：并行分支共享大部分前缀KV计算，却各自独立重算。例如，在深度3、分支因子4的树搜索中，从深度2开始发散的分支仍然共享了66%的KV工作。vLLM和SGLang仅在令牌精确匹配时复用KV，无法利用跨分支的语义相似性。
  - **验证深度过深**：过程奖励验证（process-reward verification）进行全深度前向传播，即使模型已经足够自信（如熵已稳定）仍然跑完所有层，浪费算力。
- **整体含义**：本文提出RKSC，一个无需训练的推理加速框架，同时消除上述两种冗余，显著加速多分支推理流水线，且对下游任务精度影响极小。

## 2. 论文提出的方法论

### 核心思想
通过三个互补机制协同加速：
1. **ASKS（Attention-Similarity KV Sharing）**：基于隐藏状态余弦相似度，将前缀KV缓存广播给所有语义相似的分支，严格推广令牌精确前缀缓存。
2. **CGEE（Confidence-Gated Early Exit）**：双重退出机制：第一级在生成置信度足够高且分支间差距明显时跳过整个验证前向传播；第二级在验证前向传播中，当逐层logit熵稳定时在中间层提前终止。
3. **RSBCM（Reasoning-Selective Block Cache Manager）**：通过注意力加权深度优先驱逐策略，防止深层树搜索中缓存无限增长。

### 关键技术细节
- **ASKS**：
  - 只执行一次带有前缀的前向传播，存储所有层的KV状态和隐藏表示。
  - 对每个分支计算加权余弦相似度（指数加权，更关注高层表示），仅当相似度超过阈值τ时共享缓存。
  - 共享缓存通过`repeat_interleave`扩展并克隆为连续内存，然后所有分支在一个batch中解码。
  - 包含自校准运行时探针（KV probe），在每64个令牌的桶中测量三种路径耗时，自动决定是否启用共享。
- **CGEE**：
  - **Level 1（验证跳过）**：在解码过程中累积每个分支的top-1 softmax概率的均值作为生成置信度p(b)。若`max p(b) ≥ τ_conf`且相对间隙`(max - 2nd-max)/max ≥ r_gap`，则完全跳过验证，直接按p(b)排序选答案。
  - **Level 2（层级熵退出）**：当Level 1不触发时进入验证前向传播，安装前向钩子，每层提取last-token隐藏状态，投影到词嵌入矩阵计算熵。当层数≥2、熵低于阈值θ且连续层熵变化小于ϵ时，触发提前退出。
- **RSBCM**：
  - 每个缓存块赋予重要性分数ω = branch_score / (depth + 1)。当块数超过容量时，按ω升序驱逐。优先保留浅层高置信度分支的块。

### 算法流程简述
1. 单次前缀前向传播计算根KV缓存和隐藏状态。
2. ASKS门控：对每个分支计算与根的相似度，相似者接收共享缓存，否则独立重算。
3. 所有分支在共享缓存上批量解码，同时记录生成置信度。
4. Level 1：若置信度条件满足，直接输出答案。
5. Level 2：否则进入验证前向传播，并逐层检查熵稳定条件，提前退出。

## 7. 优点

- **无需训练**：所有机制都是推理时后处理，无微调或架构改动，易于集成。
- **严格推广现有方法**：ASKS是完全包含令牌精确前缀缓存的超集。
- **可分解设计**：各组件独立可控，可灵活启用/禁用。
- **低错误率**：双条件验证跳过门和熵稳定条件提供了强安全保证，错误极少。
- **难度自适应**：CGEE自动在简单任务上施加更多加速，在困难任务上保持谨慎。
- **理论支撑**：给出了延迟模型和加速分解公式，预测与实验结果吻合。
- **广泛评估**：覆盖5个不同模型家族、4个数据集、多种解码步长和前缀长度，并做了详细的消融和压力测试。
- **开源代码**：提供完整代码和复现脚本，促进可重复性。

## 8. 不足与局限

- **样本量有限**：每数据集仅50个问题，导致CGEE skip率置信区间较宽（±10-18%）。更大样本（N≥200）可缩小CI。
- **模型规模限制**：仅评估7B-10B模型，未在14B+或多GPU张量并行下测试。理论预测更大模型受益更多，但多GPU通信开销可能改变比例。
- **领域覆盖**：限于科学和数学。CGEE熵阈值在GPQA上校准，可能不直接迁移到其他领域（如文学、法律）。需要每个数据集单独校准（5-10个问题可恢复）。
- **CGEE错误**：虽然仅0.37%，但所有错误都来自Level 1验证跳过门，且集中在ARC-Challenge（多选题格式敏感）。对于要求零错误的场景应禁用Level 1仅用Level 2。
- **ASKS广播内存开销**：`repeat_interleave` + `.clone()`在B≥64、长前缀时可能非微不足道，可用零拷贝视图优化但需解决SDPA连续性要求。
- **钩子依赖架构**：Level 2依赖于`register_forward_hook`和可访问的unembedding矩阵，对MoE或融合层架构可能需要适配。
- **RSBCM并发限制**：当前假设单次求解拥有缓存，多租户推理服务器需扩展为tenant感知的重要性评分。
- **实验平台单一**：仅使用A100 GPU，未验证其他GPU或推理卡（如H100、AMD MI300）。
