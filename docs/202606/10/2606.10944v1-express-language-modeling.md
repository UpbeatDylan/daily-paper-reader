## 摘要
我们引入了一个新工具 Express，用于将非因果注意力近似转换为具有匹配近似保证的因果注意力近似。当与最先进的 Thinformer 近似结合时，Express 改进了已知的最佳因果注意力保证，对于长度为 n 的序列，实现了 log^{3/2}(n)/s 的近似误差，仅需 O(s) 内存和 O(s^2 log^2(n)) 的压缩开销。我们将这些进展与高效的 I/O 感知 Triton 实现相结合，展示了相对于 FlashAttention 2 的显著加速，并利用 Express 克服了语言建模流程中的四个资源瓶颈：长上下文预填充、KV 缓存压缩、长形式内存受限解码和长形式计算受限解码。

## Abstract
We introduce a new tool, Express, for converting a non-causal attention approximation into a causal approximation with matching approximation guarantees. When combined with the state-of-the-art Thinformer approximation, Express improves upon the best known causal attention guarantees, delivering $\log^{3/2}(n)/s$ approximation error with only $O(s)$ memory and $O(s^2 \log^2(n))$ compression overhead for a sequence of length $n$. We pair these developments with an efficient I/O-aware Triton implementation, demonstrate substantial speedups over FlashAttention 2, and use Express to overcome four resource bottlenecks in the language modeling pipeline: long-context prefill, KV cache compression, long-form memory-constrained decoding, and long-form compute-constrained decoding.

# Express Language Modeling 论文总结

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：在语言模型推理中，因果注意力（causal attention）的计算复杂度随序列长度 \(n\) 呈二次增长（\(O(n^2 d)\)），导致长上下文预填充、KV 缓存压缩、长序列生成解码等环节出现内存和计算瓶颈。现有的大多数高效注意力近似方法（如基于子高斯 thinning 的 Thinformer）主要针对非因果（无掩码）注意力，无法直接应用于语言建模所必需的因果掩码场景。
- **整体含义**：若能高效地将非因果注意力近似转化为因果近似，则可以大幅降低大语言模型在处理长序列时的资源消耗，扩大模型在资源受限设备上的部署能力，同时降低能耗。本文提出的 Express 元过程正是为解决这一转化问题而设计。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：Express 是一种流式加权缓存机制，能够将基於 thinning（子选择）的离线非因果注意力近似转化为因果（在线）近似，且保持近似的子高斯误差保证。
- **关键技术细节**：
  - **Express 算法（Algorithm 1）**：维护一个加权缓存，初始目标缓存大小 \(n_{\text{out}}\)、膨胀因子 \(m\) 和一个 halving 算法 HALVE。处理过程分三个阶段：
    1. **精确阶段**：前 \(n_{\text{out}}\) 个点直接存入缓存。
    2. **thin 阶段**：后续点按批次（大小 \(2m n_{\text{out}}\)）先通过 SUBSAMPLE 均匀子采样，再通过 COMPRESS 2（递归分组 halving）压缩至 \(n_{\text{out}}\) 个点，合并到主缓存。
    3. **HALVE 阶段**：当主缓存大小达到 \(4n_{\text{out}}\) 时，连续两次执行 HALVE 将其缩减回 \(n_{\text{out}}\)，并增加膨胀因子 \(m\)。
  - **质量保证**（Theorem 2）：若每个 HALVE 调用是 \((\kappa, \nu_H)\)-子高斯的，则 Express 生成的加权核心集是 \((\kappa, \nu_E)\)-子高斯的，且 \(\nu_E^2 = O\big(\frac{\|\kappa\|_n \log n_{\text{out}}}{n_{\text{out}}^2} + \frac{\|\kappa\|_n}{2^m n_{\text{out}}}\big)\)，仅比单次 HALVE 放大一个常数因子（约 \(2\sqrt{\log n_{\text{out}}}\)）。
  - **Thinformer Express（Algorithm 4）**：将 Express 与注意力核 \(\kappa_{\text{att}}\) 结合，使用 KH-Express（kernel halving 作为 HALVE）并调用 WTD_ATTENTION（Algorithm 5）进行近似注意力计算。
  - **复杂度**：查询时间 \(O(n_{\text{out}} n d)\)，压缩时间 \(O(d n_{\text{out}}^2 \log n_{\text{out}} \log(n/n_{\text{out}}))\)，空间 \(O(n_{\text{out}} d)\)（均与序列长度 \(n\) 为对数或无关关系）。
  - **Triton 实现**：利用 I/O 感知的 tiling、并行化、双引用避免数据拷贝，显著优于原始 torch.compile 实现。

## 3. 实验设计

- **数据集和场景**：
  - **长上下文预填充**：使用 ChatGLM2-6B-32K 模型，在 LongBench-E 任务（TREC-E、TriviaQA-E、HotpotQA-E）上评估掩码注意力的速度与困惑度。
  - **加速 KV 缓存压缩**：在 Llama 3.1 8B Instruct 模型上，结合三种压缩方法（SnapKV、StreamingLLM、PyramidKV）在 LongBench-E 三个子任务上比较准确率和运行时。
  - **长形式解码（内存受限和计算受限）**：使用 DeepSeek-R1-Distill-Llama-8B 模型在 MATH-500 竞赛数学题上评估准确率-缓存大小/运行时 trade-off。

  - 解码对比：StreamingLLM、SnapKV、ExpectedAttention、KeyDiff、Knorm。

## 6. 主要结论与发现

- **理论层面**：Express 将非因果 thinning 近似转化为因果近似，误差仅放大常数倍数（\(\le 2\sqrt{\log n_{\text{out}}}\)），且给出比 HyperAttention 和 BalanceKV 更紧的误差界、更小的内存开销和压缩时间。
- **实践层面**：
  - 长上下文预填充：Thinformer Express 在 512K 词元长度下比 FlashAttention 2 快 82 倍（h=9），同时困惑度优于 HyperAttention。
  - KV 缓存压缩：结合三种压缩方法后，在保持准确率的同时显著减少运行时（例如在 HotpotQA-E 上运行时从约 500ms 降至约 300ms）。
  - 长形式解码：在 MATH-500 上，Thinformer Express 在准确率-缓存大小和准确率-运行时两个 trade-off 曲线中均优于所有基线，仅用 61% 的缓存大小（或 56% 的计算时间）即可匹配精确注意力的准确率。

## 7. 优点
