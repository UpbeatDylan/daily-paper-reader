## 摘要
自注意力机制允许跨序列自由选择信息，但在深度维度上，Transformer仅将每一层的输出添加到残差流中，因此后续层无法有选择地重用早期层的表示。最近的跨层方法改善了这种信息流动，但作用于注意力之外的隐藏状态，在推理时增加了键值缓存之外的状态——随着现代大语言模型通过分组查询和多头潜在注意力压缩缓存，这种成本变得越来越显著。我们提出了深度注意力（Depth-Attention），它在注意力模块内部完成这种选择：在一层对序列进行注意力计算之前，其查询会关注同一token位置处早期层的键，并将它们的值混合到自注意力随后读取的值中。由于深度注意力重用了标准的注意力查询、键和值缓存槽，用深度混合后的值替换原始值，它不引入任何参数，也不在标准键值缓存之外引入额外的持久推理状态——其缓存大小与原始解码器相同，且小于基于隐藏状态的跨层方法。在1.5B和3B参数的Qwen3风格解码器上，深度注意力取得了最低的困惑度和最高的平均下游准确率，相比原始Transformer最高提升2.3个准确率点，并在困惑度和平均准确率上超越了强大的跨层基线方法，同时仅增加不到0.01%的额外算术FLOPs，且无额外持久推理状态。这些增益在360M到3B参数规模上保持一致，并扩展到了循环Transformer上。

## Abstract
Self-attention selects information freely across the sequence, but across depth, Transformers merely add each layer's output to the residual stream, so later layers cannot selectively reuse earlier-layer representations. Recent cross-layer methods improve this flow but operate on hidden states outside attention, adding state beyond the key-value cache at inference--a cost that becomes increasingly salient as modern LLMs compress the cache with grouped-query and multi-head latent attention. We introduce Depth-Attention, which performs this selection inside the attention module itself: before a layer attends over the sequence, its query attends over the keys of earlier layers at the same token position and mixes their values into the value that self-attention then reads. Because Depth-Attention reuses the standard attention queries, keys, and value-cache slots, storing depth-mixed values in place of the original values, it adds no parameters and introduces no persistent inference state beyond the standard key-value cache--the same cache size as a vanilla decoder and less than hidden-state-based cross-layer methods. On Qwen3-style decoders at 1.5B and 3B parameters, Depth-Attention attains the lowest perplexity and the highest average downstream accuracy, improving over the vanilla Transformer by up to 2.3 accuracy points and surpassing strong cross-layer baselines in perplexity and average accuracy, while adding under 0.01% extra arithmetic FLOPs and no additional persistent inference state. The gains hold from 360M to 3B parameters and extend to looped Transformers.

### 1. 核心问题与整体含义（研究动机和背景）
- **问题**：标准 Transformer 中，深层只能通过残差流接收所有先前层的聚合信息，无法**选择性**重用某个特定早期层的表示。这种“压缩”限制了信息流动。
- **背景**：现有跨层方法（如 DenseFormer、Attention Residuals、mHC）改进了信息流动，但操作在**隐藏状态**上，需要在标准 KV-cache 之外额外存储或传递中间表示，导致推理时内存和计算开销显著增长，尤其在 GQA/MLA 等压缩缓存机制下更加突出。
- **本文目标**：提出一种**无额外推理状态**的跨层信息选择机制，在注意力模块内部复用已有的键和值，不增加参数和缓存大小。

### 2. 论文提出的方法论
- **核心思想**：在每个 token 位置，当前层的查询（query）沿深度维度对该位置**早期层的键（key）**进行注意力计算，利用所得权重混合它们对应的**深度混合值（depth-mixed value）**，替换原始值，随后执行标准因果自注意力。
- **关键技术细节**：
  - **公式**：对于 layer ℓ 位置 t 的 token，深度注意力权重 \(\alpha_{t}^{\ell,j} = \text{softmax}_{j \le \ell}(q_t^\ell \cdot k_t^j / \sqrt{d})\)，深度混合值 \(\tilde{v}_t^\ell = \alpha_t^{\ell,\ell} v_t^\ell + \sum_{j<\ell} \alpha_t^{\ell,j} \tilde{v}_t^j\)。
  - **稀疏深度源**：采用步长 \(s\)（默认 \(s = L/2\)）选择小部分早期层，避免全层注意力带来的通信和计算负担。
  - **分组查询注意力（GQA）适配**：将每组查询平均成一个查询，在 KV 头分辨率上执行深度注意力，进一步降低计算。
  - **无额外参数**：复用 query、key、value 已有计算，仅将深度混合后的值存入标准 V-cache 槽，**不增加任何持久推理状态**。

### 6. 论文的主要结论与发现
- **性能**：在 1.5B 和 3B 上，Depth-Attention 获得**最低的困惑度**和**最高的平均下游准确率**（零样本 1.5B: 53.60% vs 基线最高 52.88%；3B: 55.69% vs 55.01%），提升高达 2.3 个点。
- **效率**：理论额外 FLOPs 仅 0.004%（相对 2N 估算），加算仅 2.40×10⁵ FLOPs/token；实际训练时间增加 9%-11%，推理时间增加 1.18%，**无额外持久内存**（对比基线需 1.5~24 GiB 额外内存）。
- **缩放能力**：从 360M 到 3B 持续优于 vanilla，表明收益随规模增长保持稳定。
- **循环 Transformer**：500M 循环次数 3 的模型中，使用 Depth-Attention 使验证损失从 2.208 降至 2.194。
- **消融**：稀疏步长（s=L/2）最佳；自适应 softmax 混合优于均匀平均；仅混合值效果最好。
