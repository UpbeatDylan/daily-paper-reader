## 摘要
大型语言模型日益成为智能体系统的执行引擎，但它们仍通过顺序文本接口消耗上下文。这与现代结构化智能体工作流不匹配，在后者中，独立分支探索子任务、检索证据或生成候选解决方案，然后进行最终合成步骤。现有系统通常通过拼接这些分支的文本输出来合并它们，这丢弃了并行结构并招致冗余的预填充计算。在这项工作中，我们引入了Parallel-Synthesis，一个即插即用框架，使合成器能够直接消耗由并行工作智能体产生的KV缓存。Parallel-Synthesis结合了一个缓存映射器（校准独立生成的分支缓存）和一个微调的合成器适配器（实现从此非顺序缓存接口生成）。我们使用暴露合成器于并行缓存上下文的数据训练Parallel-Synthesis，教授跨缓存分支的聚合，并从基于标准文本拼接的合成中蒸馏推理行为。在涵盖数学、科学问答、代码生成、GAIA和多智能体数据库诊断的九个下游数据集上，Parallel-Synthesis在七个数据集上匹配或超越了基于文本的合成，并在另外两个上保持接近。它还将首个token生成时间减少了2.5倍到11倍，表明基于缓存的直接合成是一种更有前景的接口，用于对并行智能体分支进行更原生和高效的合成。

## Abstract
Large language models increasingly serve as execution engines for agentic systems, yet they still consume context through a sequential text interface. This creates a mismatch with modern structured agent workflows, in which independent branches explore subtasks, retrieve evidence, or generate candidate solutions before a final synthesis step. Existing systems typically merge these branches by concatenating their textual outputs, which discards the parallel structure and incurs redundant prefill computation. In this work, we introduce Parallel-Synthesis, a plug-and-play framework that enables a synthesizer to directly consume the KV caches produced by parallel worker agents. Parallel-Synthesis combines a cache mapper that calibrates independently generated branch caches with a fine-tuned synthesizer adapter that enables generation from this non-sequential cache interface. We train Parallel-Synthesis using data that exposes the synthesizer to parallel cache contexts, teaches aggregation across cached branches, and distills reasoning behavior from standard text-concatenation-based synthesis. Across nine downstream datasets spanning math, science QA, code generation, GAIA, and multi-agent database diagnosis, Parallel-Synthesis matches or outperforms text-based synthesis on seven datasets and remains close on the other two. It also reduces time-to-first-token by 2.5x-11x, suggesting that direct cache-based synthesis is a promising interface for more native and efficient synthesis over parallel agent branches.

# 中文详细总结

## 1. 论文的核心问题与整体含义

- **研究动机**：大型语言模型（LLM）在智能体系统（agent）中作为执行引擎，但处理上下文时仍采用**顺序文本接口**。现代智能体工作流常包含**并行分支**（如独立子任务、候选方案生成、多智能体协作），现有方法通过**拼接各分支的文本输出**来合并，这丢弃了有向无环图（DAG）结构，并导致**冗余的预填充（prefill）计算**（即重复编码已经生成的文本）。
- **核心问题**：能否让下游合成器（synthesizer）**直接复用并行工作智能体产生的KV缓存**（latent states），避免文本序列化与重复计算，从而更高效、更原生地支持并行分支合成？
- **整体意义**：提出一种新的通信接口——**基于KV缓存的并行合成**，为结构化智能体工作流提供更高效的执行范式和理论基础。

## 2. 方法论

### 核心思想
- **Parallel-Synthesis** 框架：使合成器直接消费来自 **m 个并行工作智能体**的 KV 缓存（每个工作智能体在本地上下文 `c_j` 下生成输出 `z_j`），而无需将 `z_j` 重新编码为文本。
- 训练合成器以近似**文本拼接路线**的生成分布，同时跳过冗余预填充。

### 关键技术细节
1. **位置重编码（Positional Re-encoding）**：
   - 原始每个工作智能体输出 `z_j` 的每个 token 带有基于其本地上下文长度的 RoPE 位置索引。通过 RoPE 旋转矩阵，将每个分支的所有 token 映射到**共享的分支后位置区间** `[n, n + M_par - 1]`（n 为分支点后的首个位置），从而让合成器感知它们是从同一分支点并行延伸的，而非串行排列。

2. **缓存映射器（Cache Mapper）**：
   - 学习一个 **MLP**，输入为每工作智能体的元数据 `s_j = (|z_j|, m)`（输出长度和分支总数），输出每层、每工作智能体的**仿射变换系数** `(α_K, β_K, α_V, β_V)`，对位置重编码后的 KV 进行逐元素缩放和平移，以校准独立编码的缓存。

3. **合成器 LoRA 适配**：
   - 在合成器上添加低秩适配（LoRA）模块，与缓存映射器联合训练，帮助模型适应非顺序的缓存输入格式。

### 训练策略（两阶段 + 合并）
- **Track 1：通用并行上下文适应**
  - 使用大规模对话数据（WildChat、UltraChat、LMSYS-Chat）将多轮对话的历史轮次编码为并行 KV 缓存，训练模型从非顺序前缀继续生成（继续预训练风格）。
  - 使用并行上下文任务（工具调用数据 Toucan、DTA-Tool；上下文学习数据 FLAN；多文档 QA 数据 2WikiMultiHopQA）进行监督微调，迫使模型主动综合多个缓存块的信息。
- **Track 2：蒸馏文本合线路由**
  - 使用 BrowseComp 数据生成三路并行轨迹，先运行文本拼接合成以获得“教师”合成轨迹（仅保留高质量样本），再将此轨迹转化为缓存格式进行训练，以学习更复杂的推理和判断能力。
- **检查点合并**：将两个 Track 训练得到的参数（缓存映射器 + LoRA）按加权平均合并（λ=0.5），避免顺序微调的遗忘问题。

## 6. 主要结论与发现

1. **性能匹配或超越文本合成**：在 9 个数据集中，Parallel-Synthesis 在 **7 个数据集上匹配或超过** Text-Serialization（数学、科学 QA、MARBLE 提升明显），在 GAIA Level 1 和 MBPPPlus 上略有差距。
2. **大幅降低首 token 延迟**：TTFT 降低 **2.5x–11x**（得益于跳过工作输出重预填充）。
3. **真正进行推理级合成**：显著优于 Voting 基线（除 MedQA 外），表明能利用轨迹级信号而非仅最终答案；案例研究展示其可组合部分正确推理、引用来源、选择最佳支持答案。
4. **后训练至关重要**：无训练变体或推理时方法（APE、CacheBlend）性能差；通过两阶段合并后训练可实现稳健泛化。
5. **轨迹信息传递的最佳实践**：对于多轮工具任务，传递工作智能体的**最终输出**（而非完整轨迹或每轮输出）在准确率与效率间取得最佳平衡。

## 7. 优点

- **创新性高**：首次系统性地研究**多对一**的并行分支 KV 缓存合成，区别于已有的一对一潜空间通信或 RAG 块缓存复用。
- **实用性强**：即插即用，不改变工作智能体侧执行，仅替换合成器侧接口和适配模块。
- **效率显著**：避免文本重编码，TTFT 降低一个数量级，且性能不降反升（在推理密集型任务上）。
- **训练策略精巧**：两阶段合并优于顺序微调；Track 1 提供通用适应，Track 2 蒸馏复杂推理能力。
- **泛化能力好**：训练数据与评估数据集无重叠，仍能在 9 个域外任务上取得可靠表现。

## 8. 不足与局限

- **性能差距**：在 GAIA Level 1（证据型）和 MBPPPlus（代码）上略逊于 Text-Serialization，表明基于缓存的表述可能丢失文本中的细粒度可读信息。
- **需要后训练**：不适用于完全零样本设置；相比文本拼接需要额外训练成本。
- **额外计算开销**：缓存映射 MLP 和 LoRA 的推理增加少量延迟，但已被 TTFT 节省覆盖。
- **实验局限**：
  - 仅采用 Qwen3-14B 一个骨干模型，未在更大或不同架构模型上验证。
  - 未报告多次运行的统计显著性，可能存在任务内方差。
  - 未与压缩摘要方法（如 Anthropic 的总结路线）做直接对比（论文提及但未纳入基线）。
  - 仅测试 3 或 5 个并行分支，未探索更多分支下的可扩展性。
- **应用限制**：要求合成器接收并行的完整 KV 缓存，对于极长轨迹或超大规模分支，显存/延迟可能仍是瓶颈；当前框架假设所有工作智能体均使用相同骨干模型。
