## 摘要
随着推理模型生成数万token长的思维链，KV缓存日益成为部署瓶颈。现有缓存淘汰方法通过注意力权重对token进行排序，但在长推理轨迹中注意力权重视为有噪的重要性代理，并且通过迫使模型物化注意力矩阵而禁止在生产推理中使用融合内核。在本工作中，我们转而使用一种称为顿悟分数的指标对token评分：该分数直接从前向传播中读取模型内部表示的变化，无需注意力矩阵且额外状态可忽略不计。由此产生的缓存淘汰方法EpiKV无需训练、分类器或自定义内核，可直接在FlashAttention推理堆栈中不变地使用——相比于基于注意力的评分，可将可行上下文长度扩展16倍。与基线不同，我们的方法是因果的，并专门设计用于快速变化的长推理链：我们发现层间顿悟分数差异很大（下层正面，中上层负面），并通过因果滚动z分数消除位置趋势。在4096-token缓存下，EpiKV在MATH-500上达到72%，与最强的基于注意力的基线持平（ThinKV 71%，H2O 67%）；一种滞后归一化的KV变体在8192 token下在AIME-2024上达到37%，超过最佳基线（33%），速度高达2.8倍。

## Abstract
As reasoning models emit chains of thought tens of thousands of tokens long, KV cache increasingly becomes a deployment bottleneck. Existing cache eviction methods rank tokens by attention weight, which is a noisy importance proxy in long reasoning traces, and prohibits the use of fused kernels in production inference by forcing the model to materialize the attention matrix. In this work, we instead score tokens with a metric we term the epiphany score: the change in the model's internal representation, read directly from the forward pass with no attention matrix and negligible extra state. Our resulting cache eviction method, EpiKV, requires no training, classifier, or custom kernel, and can be used directly in FlashAttention inference stacks unchanged -- scaling to a 16x longer feasible context than attention-based scoring. upper-mid layers negatively) and remove a positional trend with a causal rolling z-score. At a 4096-token cache EpiKV reaches 72% on MATH-500, matching the strongest attention-based baseline (ThinKV 71%, H2O 67%); a lag-normalized KV variant reaches 37% on AIME-2024 at 8192 tokens against the best of them (33%), at up to 2.8x the speed.

## 1. 核心问题与整体含义（研究动机与背景）

- **问题**：推理模型（如 DeepSeek-R1）生成数万 token 的长思维链，使得 KV 缓存线性增长，迅速成为部署时的内存瓶颈。现有缓存淘汰方法在解码时依赖注意力权重来排序 token，但存在两个关键缺陷：
  - **信号有噪**：注意力权重在长推理轨迹中是重要性指标的有噪代理（注意力 sink 吸收权重、填充 token 吸引注意力但实际不重要）。
  - **架构不兼容**：读取注意力权重需要物化注意力矩阵（n×n），这将迫使模型使用 Eager 内核，无法利用 FlashAttention 等融合内核，导致 GPU 内存耗尽（80GB A100 在 8192 token 时 OOM）。
- **意义**：提出一种无需物化注意力矩阵的缓存淘汰方法，使长上下文推理的部署变得高效且兼容现有推理栈（vLLM、TGI 等）。

## 2. 方法论：核心思想、技术细节

### 核心思想
- 使用**顿悟分数（Epiphany Score）** 来评价 token 重要性，该分数基于前向传播中模型内部表示的变化（隐藏状态 L2 差异），无需注意力矩阵，仅需读取标准 `output_hidden_states` 接口。
- 顿悟分数捕捉推理轨迹中的转折点（如一个步骤的结论、一个已形成的洞见）——这些 token 对应隐藏状态的剧烈变化，是应保留的重要 token。

### 关键技术细节
1. **两波段层解剖**（Two-Band Layer Anatomy）：
   - 对 DeepSeek-R1-Distill-LLaMA-8B 的 32 层进行反事实遮挡分析，发现：
     - **Band A（层 7–13）**：隐藏状态变化与 token 重要性**正相关**（这些层对应事实检索与组合，来自 ROME/MEMIT 等工作的定位）。
     - **Band B（层 18–25）**：隐藏状态变化与 token 重要性**负相关**（这些层准备输出分布，大幅变化表示流畅性延续而非重要内容）。
   - 将两波段组合为原始分数：\( s(t) = \bar{g}_{10}(t) - \bar{g}_{21}(t) \)（取两个代表性层的滚动均值之差）。

2. **时间趋势校正**（Temporal-Trend Correction）：
   - 原始分数存在位置监控（早期 token 分数偏高，晚期偏低），干扰同一推理序列内的排名。
   - 使用**因果滚动 z 分数**（causal rolling z-score）校正：
     \[
     z_l(t) = \frac{g_l(t) - \mu_l(t)}{\sigma_l(t) + \varepsilon}
     \]
     其中 \( \mu_l(t), \sigma_l(t) \) 是过去 w=64 个 token 的均值和标准差。
   - 最终分数：\( z_{10}(t) - z_{21}(t) \)。该分数因果、位置无关。

3. **淘汰策略**：
   - 保留 prompt（预填充）token 和尾部 recency 窗口（min(128, K/4)），其余位置按分数保留 top-K。
   - 支持多种变体：仅隐藏状态（EpiKV）、仅 KV 向量（key/key variance、Lag-KV）、隐藏状态与注意力混合（需 Eager）。

### FlashAttention 兼容性
- 所有隐藏状态和 KV 向量方法**无需物化注意力矩阵**，可运行在 FlashAttention v2 内核中，保持 O(n) 内存。
- 注意力基方法必须设置 `output_attentions=True`，触发 Eager 内核，内存 O(n²)。

## 6. 主要结论与发现

1. **两波段解剖**：隐藏状态变化在中间层（7–13）与重要性正相关，在上中段（18–25）负相关，且该模式在竞赛数学上一致，但随任务难度移动（简单题正相关层前移）。
2. **EpiKV的性能**：
   - **MATH-500 4096 缓存**：EpiKV 准确率 72%，持平最强注意力基线 ThinKV（71%），高于 H2O（67%），距无淘汰上界（75%）仅 3 个点。
   - **AIME-2024 8192 缓存**：Lag-KV（FA2 兼容）达到 37%，优于所有注意力基线（最佳 33%）。
3. **速度优势**：FA2 兼容方法避免 Eager 内核，在 AIME-2024 8192 缓存下，Lag-KV 比最快注意力基线 ThinKV 快 1.6×，比最慢 RaaS 快 2.8×。
4. **注意力基方法在低预算下崩溃**：H2O 在 1024 缓存下仅 5% 准确率（93/100 问题生成空输出），证明注意力信号在推理轨迹中噪声极大。
5. **内存增益**：预填充阶段物化注意力矩阵导致 80GB A100 在 8192 token 时 OOM，而 FlashAttention 可扩展到 65536 token（16×）。解码阶段峰内存主要由缓存预算决定，方法间差异小。

## 7. 优点（亮点）

1. **无注意力矩阵**：完全兼容 FlashAttention 和其他融合内核，无需 Eager 回退，直接部署在生产推理栈（vLLM、TGI、SGLang）中。
2. **无需训练/分类器/自定义内核**：仅需标准 `output_hidden_states` 接口，零额外训练成本。
3. **因果性**：分数仅在 token 产生时计算一次并冻结，不依赖于未来 token。
4. **时间趋势校正**：发现并解决了隐藏状态信号的位置监控问题，通用性高（其他基于隐藏状态的方法也可能受益）。
5. **层解剖解释性**：与机械可解释性（ROME/MEMIT）一致，提供了可理解的推理。
6. **释放反事实重要性标签**：作为验证资源公开，促进后续研究。
