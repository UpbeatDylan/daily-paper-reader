## 摘要
交错多模态推理通过在多步生成过程中重新访问视觉证据来改善视觉定位，但现有方法通常依赖令牌重放，反复转发选定的视觉令牌。一个自然的捷径是直接重用历史视觉键值缓存。然而，我们发现了这种策略的一个关键失败模式：缓存的视觉键已经绑定到它们原始的位置上下文。这种过时的位置绑定会扭曲后续解码上下文中的注意力，并可能触发严重的自回归解码崩溃。这一失败表明，有效的缓存重用需要在与当前解码状态兼容的位置下重建视觉证据，而不是直接复制绑定位置的缓存条目。为此，我们提出了位置重绑定缓存复用（PRCR），一种用于无重放视觉重新访问的缓存级框架。PRCR将原始视觉KV缓存及其原始空间坐标一起存储，然后重新分配位置兼容的坐标以选择条目，并在将重建的缓存注入到活动解码器缓存之前重新绑定它们的键。这种设计重用了历史视觉证据，同时保留了文本位置连续性和相对视觉结构。在多个多模态推理基准上的实验表明，PRCR达到了与重放相当或更好的性能，平均准确率提高了5%，并将视觉重新访问计算量减少了数万倍。

## Abstract
Interleaved multimodal reasoning improves visual grounding by revisiting visual evidence during multi-step generation, yet existing methods typically rely on token replay, repeatedly forwarding selected visual tokens. A natural shortcut is to reuse the historical visual key-value (KV) cache directly. However, we identify a critical failure mode of this strategy: cached visual keys are already bound to their original positional context. Such stale positional binding distorts attention under later decoding contexts and can trigger severe autoregressive decoding collapse. This failure suggests that effective cache reuse requires reconstructing visual evidence under positions compatible with the current decoding state, rather than directly copying position-bound historical cache entries. To this end, we propose Position Rebinding Cache Reuse (PRCR), a cache-level framework for replay-free visual revisiting. PRCR stores raw visual KV cache together with their original spatial coordinates, then reassigns position-compatible coordinates to select entries and rebinds their keys before injecting the reconstructed cache into the active decoder cache. This design reuses historical visual evidence while preserving textual positional continuity and relative visual structure. Experiments across multiple multimodal reasoning benchmarks show that PRCR achieves replay-level or better performance, improving average accuracy by 5 percent and reducing visual-revisiting computation by up to tens of thousands of times.

# 论文总结：Position Rebinding Cache Reuse (PRCR)

## 1. 核心问题与整体含义（研究动机和背景）

- **问题背景**：交错多模态推理（Interleaved Multimodal Reasoning）要求在生成过程中多次重新访问视觉证据以增强视觉定位。现有方法（如 ICoT）采用 **token 重放**机制：将选定的原始视觉 token 重新插入解码上下文并再次前向传播，这导致大量的冗余计算。
- **自然探索**：一个直观的捷径是直接复用历史视觉键值（KV）缓存，跳过重放。但作者发现 **直接复制历史 KV 缓存会导致灾难性失败**：缓存的视觉 key 已通过 RoPE 绑定到原始空间坐标，当插入新的解码上下文时，过时的位置绑定扭曲注意力分布，引发低熵重复 token 循环（autoregressive decoding collapse）。
- **研究目标**：设计一种无需重放的缓存级视觉重用框架，在保留位置兼容性的前提下复用历史视觉证据，同时保持文本位置连续性和相对视觉结构。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- **关键洞察**：历史视觉 KV 缓存中的 key 已被 RoPE 绑定到原始位置，直接复用会带来位置错位；**必须先存储未经位置编码的原始 KV，然后在当前解码状态下重新分配兼容位置并重新绑定 key**。
- **总体框架**：Position Rebinding Cache Reuse (PRCR)，包含三个核心组件：Raw Visual Evidence Memory (RVEM)、Position Reassignment for Cache Reinsertion、Replay-Free Cache Decoding。

### 2.2 关键技术细节
- **Raw Visual Evidence Memory (RVEM)**：
  - 在多模态预填充阶段，额外存储每一层视觉 token 在 RoPE 之前的原始 key/value 投影（`k_raw`, `v_raw`），以及该 token 的原始二维空间坐标 `p_m = (h_m, w_m)`。
  - 不存储 RoPE 后的位置绑定 key，避免后续干扰。
- **Position Reassignment for Cache Reinsertion**：
  - 当解码触发视觉重访（通过 ADS 选择索引集 I_τ）时，从 RVEM 中取出对应的原始 KV 和坐标。
  - 提出 **Position-Consistent Reinsertion (PCR)** 策略：
    1. 保持下一文本位置 `p_txt_τ + 1` 不变，左向扩展插入区间为 `(p_txt_τ - L, p_txt_τ + 1)`，L 为左扩展长度（默认 L=2）。
    2. 计算选定视觉子集在原始图像中的坐标范围 `p_(min)` 和 `p_(max)`。
    3. 对每个选定 token，沿 h、w 轴分别归一化相对位置：`u = (p_m - p_min) / (p_max - p_min)`。
    4. 重新锚定坐标：`p_hat = (p_txt_τ - L) + (L+1) * (1 + (K_τ-1)u) / (K_τ + 1)`。
    5. 这样所有重插入的视觉 token 位于 `(p_txt_τ - L, p_txt_τ + 1)` 内，且保持原始相对空间布局，不移动后续文本位置。
- **Replay-Free Cache Decoding**：
  - 将选定的原始 key 用新的坐标 `p_hat` 重新做 RoPE 编码，value 直接复用（value 不参与 RoPE）。
  - 将重建的视觉 KV 缓存拼接至当前解码器缓存末尾，继续自回归生成。
  - 整个过程无需 forward 原始视觉 token，仅涉及轻量级 key 重绑定和缓存拼接。

### 2.3 算法流程（文字说明）
1. 从 RVEM 中根据索引集 I_τ 取出选定视觉 token 的原始 KV 及坐标。
2. 计算原始坐标范围，归一化相对位置。
3. 使用 PCR 公式计算新坐标 `p_hat`。
4. 对每个 transformer layer，用新的坐标对原始 key 做 RoPE，得到 `K_vis_hat`，value 保持为 `V_raw`。
5. 拼接活跃解码器缓存：`C_act = Concat(C_ctx, (K_vis_hat, V_raw))`。
6. 继续解码，下一文本位置为 `p_txt_τ + 1`。

## 6. 主要结论与发现

1. **直接复用历史 KV 缓存不可行**：Direct KV Reuse 在 M³CoT 上准确率从 66.4% 跌至 23.5%，81% 的生成陷入重复循环。

3. **PRCR 在多个模型和基准上一致优于或持平 Token-Replay**：平均准确率提升 2~5 个百分点（Table 1）。
4. **计算效率极大幅提升**：相比 Token-Replay，PRCR 减少视觉重访计算量约 3,300 倍（8B 模型）至 61,600 倍（32B 模型）。
5. **额外内存开销很小**：仅 0.3%~0.64% 显存。

## 7. 优点
