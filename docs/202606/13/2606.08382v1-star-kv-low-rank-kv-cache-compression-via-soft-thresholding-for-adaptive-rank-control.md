## 摘要
低秩投影通过利用隐藏维度冗余已成为压缩KV缓存的一种有前景的方法。然而，先前的方法依赖于固定或启发式秩选择，并且难以在最小化精度下降的情况下实现积极压缩。我们提出了STAR-KV，一种具有细粒度秩控制的自适应低秩KV缓存压缩框架。STAR-KV包括：1）可微阈值机制，能够在注意力头和块级别实现最优秩选择；2）混合分解策略，根据键和值投影的敏感性应用不同的低秩因子分解；3）低秩感知混合精度量化，利用数据统计实现近乎无损的低比特量化。在多个LLM和基准测试中评估，STAR-KV实现了高达75%的KV缓存压缩，当与量化结合时，整体KV缓存减少高达20倍。通过基于自定义Triton的GPU内核，STAR-KV实现了注意力模块高达6.9倍的加速和3.1倍的端到端生成吞吐量。我们的代码公开于：https://github.com/PriyanshBhatnagar/STAR-KV。

## Abstract
Low-rank projection has emerged as a promising approach for compressing the KV cache by exploiting hidden-dimension redundancy. However, prior methods rely on fixed or heuristic rank selection and struggle to achieve aggressive compression with minimal accuracy degradation. We propose STAR-KV, an adaptive low-rank KV cache compression framework with fine-grained rank control. STAR-KV encompasses 1) a differentiable thresholding mechanism that enables optimal rank selection at both attention-head and block levels, 2) a hybrid decomposition strategy that applies different low-rank factorizations according to the sensitivity of key and value projections, and 3) a low-rank-aware mixed precision quantization that leverages data statistics for near lossless low-bit quantization. Evaluated across multiple LLMs and benchmarks, STAR-KV achieves up to 75% KV cache compression and up to 20x overall KV cache reduction when combined with quantization. Enabled by custom Triton-based GPU kernels, STAR-KV delivers up to 6.9x speedup for the attention module and 3.1x end-to-end generation throughput. Our code is publicly available at: https://github.com/PriyanshBhatnagar/STAR-KV.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：大语言模型（LLM）推理时，KV（Key-Value）缓存的内存占用随着上下文长度线性增长，成为长序列推理的主要瓶颈。现有低秩KV缓存压缩方法依赖固定或启发式的秩选择（如统一分配所有头/层相同的秩），无法适应不同层和注意力头对压缩敏感度的差异，因此在激进压缩率（如 >50%）下会导致显著的精度下降。
- **研究动机**：需要一种能够**自适应、细粒度地控制每个注意力头和每个解码器块的低秩表示维度**的方法，从而在保持模型质量的同时实现更高压缩率。
- **整体含义**：STAR-KV提出通过**可微软阈值**来学习最优秩分配，结合混合分解策略和低秩感知量化，显著提升了KV缓存压缩的精度-压缩率帕累托前沿，并为实际部署带来了显著加速。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 将投影矩阵（W_K, W_V）进行SVD分解，引入**可学习阈值α**作用于奇异值，通过**软阈值函数**（基于tanh的平滑近似）代替硬截断，使得秩选择可微分，从而通过训练自动决定每个头/块的保留成分数量。
- 采用**混合分解策略**：键投影使用头级分解（HD）以降低重建开销，值投影使用联合分解（JD）以获得更低的近似误差（因为值投影更敏感）。
- 利用**低秩感知混合精度量化**：由于保留的奇异值顺序递减，低秩潜表示中前几个通道具有较大幅度（ outlier ），通过**分块Hadamard变换**重分布能量，并对 outlier 块使用更高比特（如4bit），inlier块使用更低比特（如3bit），实现近乎无损的低比特量化。

### 关键技术细节
1. **自适应秩选择**：
   - 对每个投影矩阵执行SVD：\( W = U \Sigma V^\top, \Sigma = \operatorname{diag}(\sigma_1,...,\sigma_p) \)。
   - 定义软阈值算子：  
     \[
     \tilde{\sigma}_i = \begin{cases}
     \sigma_i \tanh(s(\sigma_i - \alpha)), & \sigma_i \ge \alpha \\
     0, & \sigma_i < \alpha
     \end{cases}
     \]
     其中α是可学习参数，s控制锐利度。
   - 有效秩\( r_{\text{eff}} \)为大于α的奇异值个数。
   - 训练过程中，α通过梯度下降优化，学习使模型在精度和压缩间平衡。

2. **混合分解方案**：
   - **键分解**：头级分解（HD），每个注意力头独立分解，重建开销小（FLOPs比JD低h倍）。
   - **值分解**：联合分解（JD），所有头拼接后一起分解，近似误差更小（根据Eckart-Young-Mirsky定理和Lemma 4.2）。
   - 通过实验验证HD-K/JD-V组合在精度与重建成本之间取得最佳平衡（图7）。

3. **损失函数**：
   - 总损失：\( \mathcal{L}_{\text{tot}} = \mathcal{L}_{\text{KD}} + \gamma \mathcal{L}_{\text{acmp}} \)。
   - 知识蒸馏损失\( \mathcal{L}_{\text{KD}} = \text{KL}(p_t \| p_s) \)，使用原始模型作为教师。
   - 自适应压缩损失\( \mathcal{L}_{\text{acmp}} = \sum_i \sum_h e^{-\alpha_{K,i,h}} + \sum_i e^{-\alpha_{V,i}} \)，鼓励阈值增大（即压缩提升），但通过指数衰减避免过度压缩。

4. **低秩感知混合精度量化**：
   - 低秩潜表示Z的通道按幅度排序，前20%通道作为outlier，后80%为inlier。
   - 分别对outlier和inlier应用独立的分块Hadamard变换（H_out, H_in），然后进行不同比特精度的逐token量化：outlier用4-bit，inlier用3-bit（平均3.2-bit）。
   - Hadamard变换可融合到分解矩阵中，不增加额外运行时开销。

5. **GPU内核优化**：
   - 使用Triton实现融合内核：将键的低秩重建、RoPE应用、去量化、注意力打分融合到单个内核中；将值的去量化、重新排序后的输出计算融合，减少中间数据搬运和内存带宽消耗。

## 6. 主要结论与发现

1. **精度-压缩率帕累托最优**：STAR-KV在75%压缩率下仍能逼近未压缩模型的零样本精度（LongChat-7B：60.49% vs 基线61.53%），而Palu在50%压缩率下已降至58.49%，ReCalKV在70%下更降至53.07%。
2. **长上下文任务**：在LongBench和RULER上，STAR-KV在60%压缩下优于Palu 50%压缩，尤其对LLaMA-3.1-8B-Instruct，RULER平均得分从85.06%提升至89.03%。
3. **量化协同**：结合3.2-bit平均精度量化，在75%低秩压缩基础上，总压缩率达20×，零样本精度59.18%，优于KIVI、KVQuant和Palu量化版本。
4. **系统加速**：自定义Triton内核实现注意力模块在64K上下文下6.9×加速（相对FP16 SDPA基线），端到端生成吞吐量提升3.1×；能一次性容纳128K上下文到RTX 4090。

## 7. 优点

- **方法创新性**：首次将可微软阈值应用于KV缓存低秩压缩的秩选择，实现了端到端、细粒度的自适应压缩，避免手工启发式。
- **混合分解策略**：基于理论和实验分析（键值敏感性差异、重建开销差异）设计，兼顾精度与效率。
- **低秩感知量化**：巧妙利用低秩分解后通道幅度有序性，通过分块Hadamard + 混合精度实现无损低比特压缩，无需额外outlier检测。
- **工程实现实用**：提供开源Triton内核，支持实际部署加速；代码公开，可复现。
- **实验全面**：覆盖多模型、多任务、多压缩率，消融和稳定性分析充分，结果可信。

## 8. 不足与局限

- **需要少量微调**：虽然训练快（6 GPU小时），但仍然需要一小部分校准数据（3000条）进行阈值学习，对于某些无微调权限的场景可能受限（但相比重新训练已很轻量）。
- **部分模型在极高压缩下仍有退化**：在LLaMA-3.1-8B-Instruct上，75%低秩压缩时零样本精度下降约3.7%（67.87→64.15），虽仍优于对比方法，但非完全无损。

- **系统实验仅针对单一GPU**：未展示多GPU、分布式推理场景下的扩展性。
