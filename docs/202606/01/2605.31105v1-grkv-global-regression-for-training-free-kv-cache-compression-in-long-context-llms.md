## 摘要
具有扩展上下文长度的大语言模型（LLM）依赖键值（KV）缓存来支持对先前令牌的注意力。然而，维护KV缓存会带来大量的内存开销，这促使了通过驱逐和合并来强制执行固定预算的KV缓存压缩方法。现代驱逐方法越来越多地采用基于跨度的保留，因为保留连续跨度在经验上有效且能更好地保持语义连贯性。然而，当与驱逐后合并相结合时，基于跨度的保留会将合并集中在少数跨度边界的载体令牌上，产生高度不平衡的合并模式，从而加剧过度合并并增加信息损失。为了解决这种不平衡，我们提出了GRKV（全局回归KV缓存），一种免训练的KV缓存合并方法，直接最小化压缩缓存与完整缓存注意力输出之间的差异。GRKV使用基于岭回归的合并步骤，将驱逐令牌的信息分布到保留令牌上，同时正则化更新以防止过度平滑。在LongBench和RULER长上下文基准测试中，GRKV是唯一一种以最小开销提升整体性能的合并方法。

## Abstract
Large language models (LLMs) with extended context lengths rely on the key-value (KV) cache to support attention over prior tokens. However, maintaining the KV cache incurs substantial memory overhead, motivating KV-cache compression methods that enforce a fixed budget through eviction and merging. Modern eviction methods increasingly adopt span-based retention because preserving contiguous spans is empirically effective and better preserves semantic coherence. Yet, when combined with post-eviction merging, span-based retention concentrates merges onto a small set of span-boundary carrier tokens, producing a highly imbalanced merge pattern that exacerbates over-merging and increases information loss. To address this imbalance, we propose GRKV (Global Regression for KV Cache), a training-free KV-cache merging method that directly minimizes the discrepancy between compressed-cache and full-cache attention outputs. GRKV uses ridge-regression-based merge steps to distribute information from evicted tokens across retained tokens, while regularizing the updates to prevent over-smoothing. Across the LongBench and RULER long-context benchmarks, GRKV is the only merging method that improves overall performance with minimal overhead.

# GRKV 论文详细总结

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：长上下文大语言模型（LLM）依赖 KV 缓存来加速自回归推理，但缓存占用大量 GPU 内存。为降低内存开销，研究者提出了基于驱逐（eviction）和合并（merging）的 KV 缓存压缩方法。
- **问题**：现代驱逐方法（如 SnapKV）普遍采用 **span-based 保留**（保留连续跨度），这比逐 token 保留能更好地保持语义连贯性。然而，当与现有的局部合并方法（如 CaM、D2O、AsymKV）结合时，**合并操作会严重集中到少数跨度边界 token 上**，导致这些载体 token 吸收过多被驱逐 token 的信息，引发“过度合并”（over-merging），造成信息丢失和模型性能下降。
- **本文贡献**：提出 **GRKV（Global Regression for KV Cache）**，一种**免训练**的全局合并方法，通过直接最小化压缩缓存与全缓存注意力输出的差异，并利用岭回归正则化，将信息均匀分布到所有保留 token 上，从而缓解合并不平衡问题。

## 2. 方法论

### 核心思想
- GRKV 将 KV 缓存合并视为一个**全局回归问题**，目标是使压缩后的 KV 缓存产生的注意力输出尽可能接近全缓存产生的输出。
- 采用“代理查询窗口”（surrogate query window）——即 prompt 末尾的一段查询——来近似未来未知的查询，因为实验表明不同窗口的注意力输出高度一致。

### 关键技术细节
- **代理窗口**：使用大小为 m=32 的最后一个 prompt 查询窗口作为优化目标。
- **优化目标**：最小化窗口级差异 \( \| A_{\text{full}} V_{\text{full}} - A_{\text{ret}} V_{\text{ret}} \|_F^2 \)，其中 \( A \) 为注意力权重矩阵。
- **岭正则化**：对键（K）和值（V）的更新施加正则项 \( \lambda_k \|K - K_0\|_F^2 + \lambda_v \|V - V_0\|_F^2 \)，防止保留 token 偏离初始状态太远，避免过平滑。
- **交替优化**：固定 K 时通过封闭解（或 Woodbury 恒等式的对偶形式）更新 V；固定 V 时通过线性化岭回归更新 K（利用 Jacobian 矩阵和对偶系统）。
- **保留固定令牌**：将 sink token、代理窗口查询 token 以及注意力分数最高的前 10% 保留 token 固定不变，以保证生成稳定性。

### 算法流程（文字说明）
1. 输入：全缓存、初始保留缓存、代理查询窗口。
2. 计算全缓存注意力输出目标 Y。
3. 交替执行以下步骤（通常只需 1 次迭代）：
   - **值步**：用当前 K 计算注意力权重 X，解岭回归得到新 V（使用对偶形式加速）。
   - **键步**：用当前 V 线性化注意力输出，解线性化岭回归得到新 K。
   - 每一步只更新自由 token（非固定 token）。
4. 输出合并后的 KV 缓存。

## 6. 主要结论与发现

- GRKV **是唯一一种在两种缓存预算（10% 和 20%）下均能提升 LongBench 和 RULER 平均得分的合并方法**，而 CaM、D2O、AsymKV 经常导致性能下降。
- GRKV 通过全局分布合并负载和岭正则化，有效缓解了 span-based 保留引起的过度合并，在检索类任务（如 NIAH）上提升尤为明显。
- 消融实验表明：适当的正则化（λ=0.01）、固定少量高注意力 token、单步交替更新、窗口大小 m=32 为最优配置。
- GRKV 可以与多种驱逐方法（span-based 和 token-based）兼容，并适用于不同规模的 LLM。

## 7. 优点

- **免训练且即插即用**：无需额外训练，可直接插件到现有驱逐方法中。
- **理论动机清晰**：从数学上分析了 span-based 保留如何导致载体瓶颈（附录 A），GRKV 扩大了有效载体集，提升了重建能力。
- **计算开销低**：值步通过 Woodbury 恒等式转为小矩阵求解（m×m），键步通过矩阵自由共轭梯度求解，预填充开销远小于其他合并方法（如 AsymKV）。
- **鲁棒性**：在多个模型、多种驱逐方法和不同缓存预算下均一致改善。
- **正则化设计**：有效防止了保留 token 的过度更新，平衡了重建与稳定性。

## 8. 不足与局限

- **实验覆盖有限**：仅评估了三个开源模型（最大 14B），未测试更大模型（如 70B/100B+）或专有系统（GPT-4 等）。
- **语言与模态局限**：仅使用英语长上下文基准，未覆盖多语言或多模态场景。
- **部分任务未见提升**：虽然平均分提升，但在少数任务（如部分摘要/少数 QA）上 GRKV 可能持平或微降（见表 1 中的 Better 列显示 14/16 或 12/16 任务提升，即约 2~4 个任务未提升）。
- **依赖代理窗口**：方法假设 prompt 末尾窗口可代表未来查询，尽管实验支持这一假设，但在某些极端动态分布下可能不够鲁棒。
- **对 FlashAttention 的依赖**：AsymKV 等基线无法使用 FlashAttention 导致速度慢，而 GRKV 需依赖 FlashAttention 保持效率；若模型不支持 FA，GRKV 的键步线性化计算可能相对更慢。
