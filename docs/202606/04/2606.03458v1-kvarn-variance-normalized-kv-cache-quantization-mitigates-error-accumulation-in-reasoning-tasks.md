## 摘要
测试时扩展是一种在大语言模型中获得更好推理能力的有效方法，但在长时间解码过程中，由于KV缓存增长，它会成为内存瓶颈。KV缓存量化有助于改善这一点，但当前方法在类似预填充的设置下评估，而错误在自回归解码下的表现不同。我们表明，在后一种情况下，量化误差随时间步长累积，主要由不正确的词元尺度驱动。我们引入了KVarN，一种无需校准的KV缓存量化器，它应用哈达玛旋转，随后在K和V矩阵的两个轴上进行双尺度方差归一化。我们发现这种组合修复了离群词元尺度错误，并显著减少了相对于现有基线的误差累积。KVarN在生成基准测试（包括MATH500、AIME24和HumanEval）上建立了KV缓存量化的最新技术水平，精度为2比特。KVarN方法的vLLM实现可在https://github.com/huawei-csl/KVarN获取。

## Abstract
Test-time scaling is a powerful approach to obtain better reasoning in large language models, but it becomes memory-bottlenecked during long-horizon decoding, as the KV-cache grows. KV-cache quantization can help improve this, but current methods are evaluated under prefill-like settings and errors behave differently under autoregressive decoding. We show that in the latter regime, quantization errors accumulate across timesteps, driven primarily by incorrect token scales. We introduce KVarN, a calibration-free KV-cache quantizer that applies a Hadamard rotation followed by a dual-scaling variance normalization across both axes of the K and V matrices. We find that this combination fixes outlying token-scale errors and substantially reduces error accumulation over existing baselines. KVarN establishes a new state-of-theart for KV-cache quantization on generative benchmarks, including MATH500, AIME24 and HumanEval, at 2-bit precision. A vLLM implementation of the KVarN method is available at https://github.com/huawei-csl/KVarN

# 详细中文总结

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：大语言模型在推理任务中常采用“测试时扩展”（test-time scaling），即通过生成更长的思维链（chain-of-thought）来提升性能。然而，长序列解码会导致 KV 缓存急剧膨胀，成为内存瓶颈。
- **问题**：现有 KV 缓存量化方法主要在“预填充”场景下评估（即一次性量化整个上下文），但真实推理场景是**自回归解码**——每生成一个 token 就需更新缓存，量化误差会随时间步逐层累积。本文发现，这种累积误差主要由**错误的 token 尺度（token magnitude）** 引起，而非方向偏差。
- **动机**：需要一种面向自回归解码的量化方法，能有效抑制 token 尺度误差，从而减少误差累积，实现近无损的低比特 KV 缓存。

## 2. 方法论

### 核心思想
- 采用**双轴方差归一化**（dual-scaling variance normalization）配合**哈达玛旋转**，同时消除通道维和 token 维的离群值，保持 token 尺度的精度。

### 关键技术细节
1. **哈达玛旋转**：对 K 和 V 矩阵的通道维度施加块对角哈达玛变换（类似 QuaRot），消除通道维的离群值并使分布更接近高斯。该变换可吸收到相邻权重矩阵中，在线计算开销小（O(N log N)）。
2. **方差归一化（VarN）**：对每个分块（如 128×128 的 tile），沿 token 维和通道维迭代地调整对数域尺度，使两个维度的方差均匀。具体算法采用 Sinkhorn-Knopp 风格的迭代（附录 Algorithm 1），直到两轴方差平衡。
3. **量化与存储**：归一化后的矩阵使用 round-to-nearest (RTN) 量化为 2-bit，同时存储两个高精度尺度向量（一个来自 RTN 的 scale/zero-point，另一个来自方差归一化的行列尺度），以及一个 FP16 零点。总平均比特数为 2.3 bit/元素（含辅助参数）。
4. **伪解码评估（Pseudo-Decode）**：将序列分块，每生成一个块后立即量化并写回 KV 缓存，后续块访问已量化的缓存，从而模拟真实解码中的误差累积——这是论文用于评估累积误差的关键方法。

## 6. 主要结论与发现

1. **KV 量化误差在自回归解码中显著累积**，而预填充式的静态评估会低估实际损失。
2. **误差累积的主要原因是 token 尺度错误**，而非方向偏差——离群误差对端到端性能影响更大（图 3）。
3. **KVarN 有效抑制 token 尺度误差**，在 2.3 bit/元素的平均精度下，在 AIME24、MATH500、HumanEval、IFEval 上均达到或超过现有最优（甚至接近 FP16）。例如 Qwen3-4B 在 AIME24 上 60.0%（FP16 61.1%），仅下降 1.1%，而 KIVI 仅为 55.5%。
4. **KVarN 在长上下文检索（Line Retrieval）上表现最佳**，在 600 行场景下仍保持 85-95% 准确率，远优于 KIVI（74%）。
5. **开销极低**：量化时延仅增加 0.18%，解量化时延约 1%，适合实际部署。

## 7. 优点

- **创新性地识别了 token 尺度误差的主导作用**，并针对性地设计双轴归一化，思路新颖。
- **无需校准数据**：VarN 完全在线计算，无需额外数据集或训练，实用性强。
- **提出伪解码评估方法**，更真实地反映长序列解码中的长期误差累积，可推广至其他量化方法评估。
- **实现轻量高效**：将额外尺度融合到解量化核中，避免显存带宽浪费，实际开销可控。
- **实验全面**：覆盖多种模型家族和任务类型，且消融实验充分验证了各组件的贡献。

- **架构限制**：仅适用于需要 KV 缓存的 Transformer 类模型，无法应用于状态空间模型（如 Mamba）或使用了多头潜在注意力（MLA）的模型（如 DeepSeek-V2），因为其量化特性尚未研究。

- **未讨论超参数敏感性**：方差归一化迭代次数（默认 8 次）及 clamp 阈值的选择是否有鲁棒性影响未深入探讨。
