## 摘要
大语言模型（LLM）推理依赖于键值（KV）缓存来避免冗余的注意力计算。虽然近似的KV缓存保留技术通过牺牲模型精度来减少内存使用，但无损方法则从GPU内存中逐出KV缓存块，并在需要时重新构建以保留精确输出。现有的无损KV缓存管理系统主要基于访问频率或位置启发式做出逐出决策，而未考虑不同KV缓存块如何影响GPU注意力内核的执行效率。在本文中，我们提出AsymCache，一个面向LLM推理的计算延迟感知KV缓存管理系统，它明确地将缓存驻留决策与GPU注意力内核性能对齐，包括三个关键组件：用于高效非连续KV上下文处理的多段注意力（MSA）、联合优化命中率和位置感知重计算成本的缓存逐出策略，以及用于高硬件利用率的自适应分块调度器。实验表明，与最新基线相比，AsymCache将TTFT降低了最多1.90-2.03倍，将每个输出token的时间（TPOT）降低了1.62-1.71倍，证实了该方法在常见工作负载下的有效性，并验证了其在计算效率与缓存命中率之间平衡的设计目标。此外，AsymCache的低级设计允许无缝集成到诸如Continuum之类的代理服务系统中，进一步将平均作业延迟降低最多18.1%。

## Abstract
Large Language Model (LLM) inference relies on key-value (KV) caches to avoid redundant attention computation. While approximate KV cache retention techniques reduce memory usage by sacrificing model accuracy, lossless approaches instead evict KV cache blocks from GPU memory and reconstruct them on demand to preserve exact outputs. Existing lossless KV cache management systems primarily base eviction decisions on access frequency or positional heuristics, without considering how different KV cache blocks affect the execution efficiency of GPU attention kernels.   In this paper, we propose AsymCache, a computation-latency-aware KV cache management system for LLM inference that explicitly aligns cache residency decisions with GPU attention kernel performance, including three key components: Multi-Segment Attention (MSA) for efficient non-contiguous KV context processing, a cache eviction policy that jointly optimizes hit rate and position-aware recomputation cost, and an adaptive chunking scheduler for high hardware utilization. Experiments show that AsymCache reduces TTFT by up to 1.90-2.03x and time-per-output-token (TPOT) by 1.62-1.71x over latest baselines, confirming the effectiveness of the method in common workloads and validating its design goal of balancing computational efficiency with cache hit rate. Moreover, the low-level design of AsymCache allows seamless integration into agent serving systems such as Continuum, where it further reduces average job latency by up to 18.1%.

## 1. 核心问题与整体含义（研究动机与背景）

大语言模型（LLM）推理依赖键值（KV）缓存来避免冗余的注意力计算，但KV缓存的内存开销极大——例如一个70B模型、32K token会话需要超过40GB内存，超出单张A100 GPU容量。

- **现有问题**：无损KV缓存管理系统（如vLLM、SGLang、Pensieve）主要基于访问频率（LRU、LFU）或位置启发式（如优先保留后缀）来做出逐出决策，但**没有考虑不同KV缓存块对GPU注意力核执行效率的影响**，导致GPU资源利用不充分。
- **核心挑战**：如何在有限GPU内存下，通过合理的缓存块逐出策略，平衡缓存命中率与重计算成本，同时保持输出精确（lossless）。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
**AsymCache** 是一个**计算延迟感知**的KV缓存管理系统，它将缓存驻留决策显式地与GPU注意力核性能对齐。核心三组件：

1. **多段注意力（Multi-Segment Attention, MSA）**：一种融合的GPU注意力核，允许在单次内核调用中处理非连续的KV上下文段（例如同时包含前缀和后缀），避免多次核调用开销。
2. **计算感知块逐出器（Computational-Aware Block Evictor）**：基于每个块的**预期注意力延迟贡献**进行逐出选择，联合优化访问频率（复用概率）和位置感知的重计算成本。
3. **自适应分块调度器（Chunking Scheduler）**：支持分块跨越多个缓存/计算段，动态调整分块大小以应对不同负载，避免头部阻塞。

### 关键技术细节

- **逐出准则公式**：选择具有最低期望计算延迟的块逐出：
  \[
  \text{block\_id} \leftarrow \arg\min_B E(B,t) = \arg\min_B f_B(t) \cdot \Delta T_B
  \]
  其中 \(f_B(t)\) 是历史访问频率（经过指数权重衰减），\(\Delta T_B\) 是若该块被逐出需要重计算的成本（正向相关于其在序列中的位置）。

- **频率函数设计**：为了保证对数时间复杂度（O(log n)），\(f_B(t)\) 必须是满足**序保持规则**的指数函数。AsymCache 使用分段指数函数：
  \[
  f_B(t) := \min\Big(\exp(-\tau_B(t)/\alpha), \exp(-(\tau_B(t)-\tau_0)/\beta)\Big)
  \]
  其中 \(\tau_B(t)\) 是距离上次访问的时间，\(\alpha,\beta,\tau_0\) 为超参数。该函数在每个分段上满足序保持，从而可使用两个平衡树维护，实现 O(log n) 的插入、删除、取最小操作，远优于线性复杂度。

- **成本模型**：采用线性近似模型估算重计算时间 \(\Delta T_B\)，基于块的位置（前方块数）即可常数时间计算。模型在第4.3节给出，拟合R²>0.999。

- **MSA核**：将不同请求分配给不同的CTA（Cooperative Thread Array）组并行计算，通过预计算数组获取每个token块对应的实际序列长度，单核处理多段非连续KV。

## 7. 优点

- **首次引入计算不对称性**：识别并量化不同KV块对GPU注意力核执行效率的差异，将延迟感知引入缓存管理。
- **高效算法设计**：通过数学分析证明只有指数函数满足序保持规则，从而设计分段指数函数实现O(log n)逐出算法，避免线性开销。
- **MSA核融合**：单个GPU核处理非连续KV段，减少内核启动开销，尤其在短序列场景优势明显。
- **自适应分块调度器**：与MSA结合，使预填充块可跨越缓存/计算段，在高负载下进一步降低延迟。
- **广泛适用性**：不仅适用于多轮对话，还能集成到agent系统，且超参数鲁棒性高。
