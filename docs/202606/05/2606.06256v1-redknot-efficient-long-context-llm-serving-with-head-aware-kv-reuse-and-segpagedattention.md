## 摘要
随着大语言模型（LLM）服务输入长度的持续增长，KV缓存已成为人工智能基础设施中的主要瓶颈。它限制了GPU内存容量、服务并发性、缓存复用以及分布式可扩展性。多个重要问题（包括位置无关的KV缓存、前缀KV缓存压缩、热/冷KV缓存分离以及分布式KV缓存管理）都依赖于KV缓存的表示和管理方式。然而，现有服务系统在很大程度上依赖于单一的KV缓存抽象，将KV缓存视为由token级内存块构成的同质序列，并在所有注意力头和服务场景中采用相似策略进行管理。我们观察到KV缓存的效用在不同KV头之间具有高度结构性：不同头展现出不同的功能角色、注意力距离和运行时重要性。因此，并非每个头、token范围或服务场景都需要完整的KV缓存。本文提出RedKnot，一种面向LLM服务的头感知KV缓存管理系统。RedKnot通过沿KV头分解KV缓存来打破传统的单一KV缓存抽象，而不同KV头的重要性和有效注意力范围在不同服务场景中差异显著。这种头级分解将KV缓存从单一的张量抽象转变为结构化的内存对象，使RedKnot能够统一支持位置无关的KV复用、前缀KV压缩、热/冷KV分离以及分布式KV放置，同时保持输出保真度并提高资源效率，无需模型重训练或微调。RedKnot通过将KV缓存从被动的单一运行时工件转变为动态的、模型感知的可扩展LLM服务运行时基础，为人工智能基础设施建立了新的基础。

## Abstract
As the input length of large language model (LLM) serving continues to grow, the KV cache has become a dominant bottleneck in AI infrastructure. It limits GPU memory capacity, serving concurrency, cache reuse, and distributed scalability. Several important problems, including position-independent KV cache, prefix KV cache compression, hot/cold KV cache separation, and distributed KV cache management, all depend on how the KV cache is represented and managed. However, existing serving systems largely rely on a monolithic KV cache abstraction, where the KV cache is treated as a homogeneous sequence of token-level memory blocks and managed with similar policies across attention heads and serving scenarios. We observe that KV cache utility is highly structured across KV heads: different heads exhibit different functional roles, attention distances, and runtime importance. Therefore, a full KV cache is not always necessary for every head, token range, or serving scenario.   We present RedKnot, a head-aware KV cache management system for LLM serving. RedKnot breaks the conventional monolithic KV cache abstraction by decomposing the KV cache along KV heads, whose importance and effective attention ranges vary significantly across serving scenarios. This head-level decomposition turns the KV cache from a monolithic tensor abstraction into a structured memory object, enabling RedKnot to uniformly support position-independent KV reuse, prefix KV compression, hot/cold KV separation, and distributed KV placement while preserving output fidelity and improving resource efficiency, without requiring model retraining or fine-tuning. RedKnot establishes a new foundation for AI infrastructure by transforming the KV cache from a monolithic, passive runtime artifact into a dynamic, model-aware runtime substrate for scalable LLM serving.

*   **背景与动机**：
    *   **现有系统的局限**：当前的LLM服务系统（如SGLang， vLLM）大多将KV Cache视为一个**单一的、同质的**张量，所有注意力头都被同等对待。这意味着系统对所有头的KV Cache采用相同的存储、管理和访问策略。

    *   **现有方法的失效**：
        *   **位置无关缓存（PIC）**：在Token级别进行KV Cache恢复时，不同头需要恢复的重要Token集不同，导致恢复集是各头重要Token的并集，这会迅速膨胀，抵消缓存复用带来的收益。
        *   **多头稀疏化**：虽然算法上可以只保留部分头所需的KV，但KV仍以稠密张量存储。在运行时通过`attn_mask`实现稀疏性会禁用FlashAttention的快速路径，导致4.9-7.6倍的性能惩罚，算法上的字节节省无法转化为实际的加速。
        *   **短上下文瓶颈**：在代理由工作负载等短上下文场景中，**前馈神经网络（FFN）** 计算主导了首token时延（TTFT），而PIC和注意力稀疏化技术无法影响FFN的计算开销。
*   **整体含义**：论文的核心贡献在于打破传统的单一KV Cache抽象，通过将KV Cache沿注意力头维度分解，实现了在**头层级**上的精细化管理、复用和计算。这使得PIC恢复、KV压缩、热/冷分离和分布式放置能在一个统一的框架下高效协同，将KV Cache从被动的运行时产物转变为主动的、模型感知的运行时基础，为可扩展的长上下文LLM服务奠定了基础。

## 2. 论文提出的方法论：核心思想、关键技术细节

RedKnot的核心方法论由紧密结合的两部分组成：**弹性稀疏（Elastic Sparsity）** 和 **SegPagedAttention**。

### 核心思想：头感知（Head-Aware）分解与协同优化

将KV Cache的管理和计算从“Token-所有头”的粗粒度，转变为“头（Head）”级别的细粒度，使不同的头可以根据其特性应用不同的策略。

### 关键技术细节

1.  **弹性稀疏（Elastic Sparsity）**：一个高保真、低成本的PIC恢复方案。
    *   **离线头分类**：通过离线profile，将每个（layer, head）对分类为**全局头**（约12-15%）或**局部头**（约85-88%）。这种分类是模型固有的，可在不同请求间复用。
        *   全局头：对前缀变化敏感，需要全范围KV访问，计算其完整KV。
        *   局部头：对前缀变化不敏感，主要关注局部窗口和注意力汇，直接复用缓存。
    *   **RoPE位置对齐**：在复用KV前，利用RoPE的旋转结构，将缓存的Key旋转到新的在线位置，消除因位置偏移导致的确定性误差。
    *   **层式恢复策略**：
        *   **浅层**：采用保守策略，使用局部注意力恢复，并执行**完整FFN计算**，以保护对误差敏感的早期残差流。
        *   **深层**：启用头部感知的注意力恢复和**部分稀疏FFN计算**。全局头被全面恢复，局部头仅在其局部可见窗口内进行修复。
    *   **部分稀疏FFN恢复**：利用恢复后的注意力信号估计Token重要性，仅对重要的Token执行完整的FFN计算，不重要的Token跳过FFN，仅保留残差连接。这直接解决了短上下文场景中FFN的瓶颈问题。

2.  **SegPagedAttention**：一种支持头感知并发的存储和计算引擎。
    *   **头分段KV布局**：打破传统的PagedAttention（以Token块为单位）布局，将KV Cache以**头段**进行组织。每个头段对应一个特定（layer, head）的KV状态，并映射到物理页。
    *   **独立执行策略**：在运行时，为每个头段查询其执行策略。
        *   全局头：访问该头段映射的所有页，执行全范围注意力。
        *   局部头：仅访问该头段中与“注意力汇”和“最新窗口”重叠的页，执行局部注意力。
    *   **融合变长注意力内核**：通过融合的`flash_attn_varlen_func`内核，直接利用每个头的变长序列长度，无需构建`attn_mask`，从而始终保持在FlashAttention的快速路径上，消除掩码惩罚。

## 6. 论文的主要结论与发现

1.  **RedKnot实现了更优的质量-延迟权衡**：相比Token级PIC基线（CacheBlend, ProphetKV），RedKnot在保持更高答案质量（EM，F1）的同时，实现了更低的TTFT，尤其是在长上下文场景下，其速度优势更明显。
2.  **Token级PIC存在根本性局限**：由于不同注意力头对重要Token的敏感度不同，Token级的恢复策略要么恢复集膨胀抵消加速，要么因恢复不完整导致精度下降。RedKnot的头级恢复从根本上解决了这一问题。
3.  **SegPagedAttention是关键架构创新**：它不仅将算法的稀疏性转化为实际的性能收益，而且通过消除`attn_mask`的开销（4.9-7.6x性能惩罚），使得稀疏性在GPU上运行时反而是加速的，其加速比随上下文长度单调递增，且支持更高的并发。
4.  **协同优化的收益是乘性的**：头部注意力稀疏化、SegPagedAttention和稀疏FFN在正交维度上工作（头、存储、通道），它们的结合能产生远高于单一技术的收益。最终，RedKnot在8×H800上实现了**1.6-3.54倍TTFT加速**和**4.7-7.8倍并发会话数**，同时削减**67-79.5%的Prefill FLOPs**。
