## 摘要
动态稀疏注意力（DSA）通过仅关注与每个查询最相关的top-K KV块来加速长上下文LLM解码，但它引入了一种串行的选择到注意力的依赖关系，这成为了新的延迟瓶颈。我们提出了PRR，一种推测-重用-修复运行时，利用DSA选择中的时间局部性来预测可能的块，在选择进行时推测对这些块的注意力，并在已知真实选择集后增量式修复遗漏的块。PRR使用轻量级的基于EMA的预测器、一个基于性能分析的推测预算来使推测工作远离关键路径，以及一个基于FlashAttention的修复内核，利用在线softmax统计将遗漏的块折叠到部分注意力状态中。在长上下文基准测试和代表性DSA方法上，PRR将每token解码延迟最多降低40%，同时保持下游任务准确性。Github: https://github.com/Tianyu9748/Incremental_FlashAttention

## Abstract
Dynamic sparse attention (DSA) accelerates long-context LLM decoding by attending to only the top-K KV blocks relevant to each query, but it introduces a serialized selection-to-attention dependency that emerges as a new latency bottleneck. We present PRR, a speculate-reuse-repair runtime that exploits temporal locality in DSA selections to predict likely blocks, speculate the attention over them while selection is in flight, and incrementally repair missed blocks once the true selected set is known. PRR uses a lightweight EMA-based predictor, a profiling-guided speculation budget that keeps speculative work off the critical path, and a FlashAttention-based repair kernel that folds missed blocks into the partial attention state using online-softmax statistics. Across long-context benchmarks and representative DSA methods, PRR reduces per-token decoding latency by up to 40% while preserving downstream task accuracy. Github: https://github.com/Tianyu9748/Incremental_FlashAttention

# 论文总结：PRR: Predict, Reuse, and Repair — 加速长上下文LLM解码的动态稀疏注意力

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：大语言模型（LLM）越来越多地部署于长上下文任务（深度研究、多步推理、工具使用等），但解码时需访问并计算不断增长的KV缓存，开销巨大。
- **现有方案**：动态稀疏注意力（DSA）通过在每个解码步仅选择与当前查询最相关的 top-K 个 KV 块来降低注意力计算量，保持了上下文相关性并显著减少计算。
- **核心问题**：DSA 引入了新的关键路径瓶颈——**选择到注意力的串行依赖**（selection-to-attention dependency）。在选择阶段（压缩注意力 + top-K选定）完成之前，注意力无法执行。随着上下文长度增加，选块阶段可占总解码延迟的 41%。
- **本文目标**：提出一种名为 **PRR**（Predict, Reuse, Repair）的推测-复用-修复运行时系统，通过利用 DSA 选择的时间局部性，将注意力计算与选块重叠，从而打破该串行依赖，降低解码延迟并保持准确率。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
- **时间局部性**：相邻解码步中，约 68% 的 top-K 块被重复选择。PRR 预测当前步可能被选中的块，在选块阶段并行执行对这些块的注意力（推测），待真实 top-K 集合已知后，仅增量修复遗漏的块，避免完全重算。

### 关键技术细节

1. **轻量级 EMA 预测器**（Predict）
   - 基于指数移动平均（EMA）对每个块的重要性分数进行预测，无需额外训练。
   - 维护每个块的水平 ℓᵢ 和趋势 vᵢ，预测公式：`cISᵗᵢ = ℓᵗ⁻¹ᵢ + γ vᵗ⁻¹ᵢ`
   - 更新公式：`vᵗᵢ = β(ISᵗᵢ - ℓᵗ⁻¹ᵢ) + (1-β)vᵗ⁻¹ᵢ`，`ℓᵗᵢ = α ISᵗᵢ + (1-α)ℓᵗ⁻¹ᵢ`
   - 超参数 (α, β, γ) 在预填充阶段通过网格搜索（80个候选）基于重要性分数轨迹校准，新增开销极小（约 0.06 ms）。

2. **推测注意力与动态预算**（Reuse）
   - 在选块阶段并行执行对预测块集 P 的注意力（KV gather + FlashAttention）。
   - 通过**关键路径感知的推测预算**控制预测集大小：离线分析选块和推测注意力的延迟曲线，在线选择最大安全 δ 使推测注意力不延迟 FFN。

3. **增量修复内核**（Repair）
   - 基于 FlashAttention 的在线 softmax 递归，实现增量注意力修复。
   - 已计算的推测输出带有部分 log-sum-exp 统计量（ℓ, m）；对遗漏块 M = A\P 应用以下递归：
     - `mᵗ⁺¹ = max(mᵗ, m̃)`
     - `ℓᵗ⁺¹ = e^(mᵗ-mᵗ⁺¹)ℓᵗ + e^(m̃-mᵗ⁺¹)ℓ̃`
     - `Oᵗ⁺¹ = e^(mᵗ-mᵗ⁺¹)(ℓᵗ/ℓᵗ⁺¹)Oᵗ + e^(m̃-mᵗ⁺¹)/ℓᵗ⁺¹ Σ e^(Sᵗ⁺¹ⱼ-m̃)Vᵗ⁺¹ⱼ`
   - 仅需流式读取遗漏块，无需重新计算所有已处理块，修复代价与 |M| 成正比。

### 整体执行流程（每层解码）
- 并行执行：选块（压缩注意力 + top-K） + 对预测集 P 的推测注意力
- 选块完毕得到真实集 A
- 增量修复 M = A\P（若 M 非空）
- 继续执行 FFN

## 7. 优点

- **方法新颖**：将推测执行与增量修复引入 DSA 解码，从根本上打破选块-注意力串行瓶颈，是首个系统性地解决该问题的方案。

- **通用性强**：适用于多种 LLM（6种）和即插即用 DSA 方法（Quest、InfLLM-v2），性能提升稳定。
- **工程贡献**：开源了增量 FlashAttention CUDA 内核，可直接用于其他研究。
