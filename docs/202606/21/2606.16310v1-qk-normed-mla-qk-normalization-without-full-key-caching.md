## 摘要
查询-键（QK）归一化通过在点积前控制查询和键的尺度来稳定注意力，但它与多头潜在注意力（MLA）并不直接兼容。MLA通过缓存低维潜在状态而非完整键来实现高效解码，而投影后的QK RMSNorm似乎需要为每个缓存的令牌使用完全投影后的键。我们证明这种表面上的不兼容性是一个实现细节造成的假象，而非架构限制。RMSNorm分解为一个静态仿射权重和一个动态标量RMS统计量。静态的键侧权重可以被吸收到MLA的查询侧投影中；动态键统计量简化为每个令牌和KV组的一个逆RMS标量。得到的公式在精确算术中与显式的投影后QK RMSNorm完全等价，并保留了MLA的潜在解码路径。在我们的400M参数模型训练高达100B令牌的实验中，与QK剪裁相比，QK归一化的MLA实现了更低的训练损失和更好的下游准确率，同时在H800解码基准测试中，对于高达256k的上下文，延迟开销小于2%。这些结果使得QK归一化成为MLA模型无需完整键缓存的实用稳定化选项。

## Abstract
Query-key (QK) normalization stabilizes attention by controlling the scale of queries and keys before the dot product, but is not immediately compatible with Multi-head Latent Attention (MLA). MLA achieves efficient decoding by caching low-dimensional latent states instead of full keys, whereas post-projection QK RMSNorm appears to require the fully projected key for every cached token. We show this apparent incompatibility is an implementation artifact, not an architectural constraint. RMSNorm decomposes into a static affine weight and a dynamic scalar RMS statistic. The static key-side weight can be absorbed into the MLA query-side projection; the dynamic key statistic reduces to one inverse-RMS scalar per token and KV group. The resulting formulation is exactly equivalent to explicit post-projection QK RMSNorm in exact arithmetic and preserves MLA's latent decode path. In our 400M runs trained for up to 100B tokens, QK-Normed MLA achieves lower training loss and better downstream accuracy than QK clipping, while H800 decode benchmarks show less than 2% latency overhead up to 256k context. These results make QK normalization a practical stabilization option for MLA models without requiring full-key caching.

## 1. 论文的核心问题与整体含义

**研究动机**：现代语言模型注意力机制面临两个实用压力：一是KV缓存效率（MLA通过缓存低维潜在状态而非完整键来减少内存），二是训练稳定性（QK归一化通过控制查询和键的尺度避免注意力饱和）。然而，标准QK RMSNorm（投影后归一化）似乎需要为每个缓存的令牌显式存储完整投影后的键，这与MLA的高效解码路径不兼容。现有方案（如QK剪裁）是在点积后裁剪logit，而非从根本上控制尺度。

**整体意义**：论文证明这种不兼容是**实现假象**而非架构限制。通过将RMSNorm分解为静态仿射权重和动态标量RMS统计量，可以设计出等价于投影后QK RMSNorm的公式，同时保留MLA的潜在缓存机制。这使得无需缓存完整键就能使用QK归一化稳定训练，为MLA模型提供了实用的稳定性选项。

## 2. 论文提出的方法论

### 核心思想
RMSNorm可分解为：`RMSNorm(z) = (z ⊙ γ) / S(z)`，其中γ是静态仿射权重（可被吸收），`S(z)`是动态RMS统计量（标量）。对于键侧，静态权重γ_k可吸收到查询投影矩阵中；动态标量`α_{k,t,g} = 1/S_{k,t,g}`只需额外缓存一个标量（每个令牌每个KV组），而不需缓存完整键。

### 关键技术细节
- **内容键归一化的吸收公式**：
  - 原始：`ℓ_{i,t,h} = q_{i,h}^c · [ (c_t W_{K_g}) ⊙ γ_k / S_{k,t,g} ]`
  - 吸收后：定义`˜q_{i,h}^c = (q_{i,h}^c ⊙ γ_k) (W_{K_g})^T`，`α_{k,t,g} = 1/S_{k,t,g}`，则`ℓ_{i,t,h} = (˜q_{i,h}^c · c_t) · α_{k,t,g}`。
- **查询侧也归一化**：类似地处理查询RMSNorm，得到标量`α_{q,i,h}`。
- **分块归一化**：将内容分量和RoPE分量分别归一化（而非联合归一化），以保持内容路径的潜在计算和RoPE路径的小型化，避免同步开销。
- **解码数据流**：缓存仍以潜在状态`c_t`为主，额外增加键侧标量缓存`A_{cache}^k ∈ R^{T×G}`。在注意力计算中，用当前查询标量`α_{q,i,h}`和缓存键标量`α_{k,t,g}`缩放原始潜在点积结果，然后送入softmax。

### 公式流程（文字说明）
1. 训练/缓存写入阶段：对每个令牌t，计算潜在状态c_t并存储；计算临时内容键矩阵`c_t W_K`，按组求RMS得到`α_{k,t,g}`并缓存；丢弃临时矩阵。
2. 解码阶段：对于当前查询位置i，计算查询RMS标量`α_{q,i,h}`，并构造吸收了键权重γ_k的潜在空间查询向量`˜q_{i,h}^c`。
3. 注意力计算：对每个历史令牌t，计算原始点积`˜q_{i,h}^c · c_t`，再乘以`α_{q,i,h}·α_{k,t,g}`得到归一化logit。
