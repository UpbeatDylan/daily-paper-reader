## 摘要
大型语言模型（LLM）越来越需要处理长上下文，但标准的softmax注意力机制会产生随序列长度线性增长的KV缓存，迅速成为长上下文推理的瓶颈。一种实用的缓解方法是淘汰不重要的KV条目；然而，现有的淘汰策略大多是启发式的，难以捕捉令牌重要性的丰富且依赖于输入的分布。在这项工作中，我们引入了一个可学习的索引器来预测KV重要性，从而更准确地保留关键令牌。同时，简单地永久淘汰令牌会丢弃其信息，导致不可逆的遗忘和长距离检索性能下降。为了解决这个问题，我们提出了一个轻量级的潜在记忆模块，将淘汰的令牌压缩成一个紧凑的、在线更新的状态，并提供残差读取来补偿因KV淘汰而损失的注意力贡献。总的来说，我们的方法能够在有界的KV预算下实现准确的长上下文推理，在RULER（4K/16K）上对Qwen、Mistral和Llama模型持续改进（在激进淘汰下最多提升25分），在“大海捞针”检索中显著更稳定，并且与现有淘汰策略相比，获得了更优的LongBench分数和压缩曲线。

## Abstract
Large Language Models (LLMs) are increasingly expected to operate over long contexts, yet standard softmax attention incurs a KV cache that grows linearly with sequence length, quickly becoming the bottleneck for long context inference. A practical remedy is to evict less important KV entries; however, existing eviction policies are largely heuristic and struggle to capture the rich, input-dependent distribution of token importance. In this work, we introduce a learnable indexer that predicts KV importance, enabling more accurate retention of critical tokens. Meanwhile, naively evicting tokens permanently discards their information, leading to irreversible forgetting and degraded retrieval over long ranges. To address this, we propose a lightweight latent memory module that compresses evicted tokens into a compact, online-updated state and provides residual readouts to compensate for the attention contributions lost through KV eviction. Collectively, our method enables accurate long-context inference under a bounded KV budget, delivering consistent improvements on RULER (4K/16K) across Qwen, Mistral, and Llama models (up to 25 points under aggressive eviction), markedly more stable Needle-in-a-Haystack retrieval, and superior LongBench scores and compression curves compared to existing eviction policies.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题本质**：大型语言模型（LLM）在处理长上下文时，标准 softmax 注意力机制产生的 KV 缓存随序列长度线性增长，成为推理的显存瓶颈。例如，70B 模型处理 100 万 token 的上下文需要约 320 GB GPU 显存，远超当前主流加速器容量。
- **现有方法局限**：
  - 启发式驱逐策略（如 SnapKV、KeyDiff、Expected Attention）依赖手工设计的评分函数，无法捕捉输入相关的 token 重要性分布，容易产生位置偏差（如局部偏好）。
  - 永久丢弃 evicted token 会导致不可逆的信息丢失，尤其在需要综合多段信息的长上下文推理任务中（如 QA、摘要），仅保留少数“关键”token 是不够的。
- **整体意义**：本文旨在提出一种**可学习**的 KV 缓存压缩框架，提高 token 重要性预测的准确性，并通过**记忆补偿**机制减少信息丢失，从而在有限的 KV 预算下维持长上下文推理性能。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **可学习索引器**：训练一个轻量级索引器，预测每个 token 对后续查询的重要性，以决定保留哪些 KV 对。
- **潜记忆模块**：将被驱逐的 token 信息压缩为固定大小的在线更新状态，并在注意力输出中加入残差补偿，弥补因驱逐损失的注意力贡献。

### 关键技术细节

#### 2.1 可学习索引器（Learnable Indexer）
- **架构**：采用 MQA（Multi-Query Attention）风格，用单头 key 与多头 query 计算相似度。query 通过下投影从多头 query 状态得到，key 从隐藏状态投影得到。应用 QK-Norm（RMSNorm）稳定训练。
- **门控机制**：对每个索引器头计算门控向量 α，加权聚合各头的相似度分数，提高表达能力。
- **重要性聚合**：对给定的查询集合（prefill 阶段用所有 prompt query，decoding 阶段用压缩窗口内的 query），取每个 key 在所有相关查询上的最大分数作为 token 重要性。
- **训练**：冻结骨干 LLM，仅优化索引器。使用 KL 散度蒸馏，让索引器输出的重要性分布匹配骨干模型真实注意力 logits 的池化分布（max 聚合）。采用流式计算避免全注意力矩阵内存爆炸，并排除 sink token 防止梯度主导。

#### 2.2 潜记忆模块（Latent Memory Module）
- **设计动机**：直接将被驱逐 token 作为额外 KV token 放入 softmax 中会导致注意力坍塌或位置冲突。因此采用**残差补偿**形式：\( o = o_{attn} + g(q) \cdot m(q) \)，其中门控 \( g(q) \in [0,1] \) 控制记忆读取量。
- **读写机制**：
  - **慢权重**（slow weights）：训练时学习的投影矩阵 \( \text{Linear}_\theta \) 和门控 \( g \)。
  - **快权重**（fast weights）：在线更新的状态矩阵 \( M \)（\( d_{mem} \times d_{model} \)）和稳定向量 \( b \)（\( d_{mem} \)）。读取公式使用二次归一化避免尺度爆炸。
  - **在线写**：当 token 被驱逐时，用外积累积更新 \( M \) 和 \( b \)，并加入衰减因子 λ。
- **训练**：以 MSE 损失优化慢权重，使记忆读出的残差尽量接近全注意力输出与压缩后注意力输出的差值。

#### 2.3 训练流程
- **两阶段训练**：
  1. 单独训练索引器（蒸馏骨干注意力）。
  2. 联合训练索引器与记忆模块（MSE 损失）。
- **训练数据**：LongAlpaca（SFT 数据），使用 chunk-wise KL 目标。
- **学习率**：WSD 调度，100 warmup -> 2000 稳定 @1e-3 -> 2000 decay 到 7.5e-6。

1. **可学习索引器显著优于启发式方法**：在 RULER 4K/16K 上，25%–90% 压缩率下 IndexMem 持续领先，尤其在 90% 压缩时比第二好方法高 25 个点以上。

## 7. 优点

- **方法创新性**：将 token 重要性预测从启发式升级为可学习，并独创性地引入潜记忆残差补偿机制，突破了传统“全有或全无”的驱逐思路。
- **轻量化设计**：索引器采用 MQA 风格、低维度，记忆模块仅增加少量参数，实验显示训练成本可控（单卡 H800）。
- **训练友好**：支持流式 KL 蒸馏避免 O(L²) 内存，两阶段训练稳定；快权重更新规则简单，无需额外梯度。
- **实验完备性**：覆盖多种模型、任务、压缩率，消融和迁移实验设计清晰，结果一致性高。
- **实用导向**：能实现 pre-eviction（解码前即可决定保留哪些 KV），减少无效计算；记忆补偿可随时关闭（g=0）保证安全。
