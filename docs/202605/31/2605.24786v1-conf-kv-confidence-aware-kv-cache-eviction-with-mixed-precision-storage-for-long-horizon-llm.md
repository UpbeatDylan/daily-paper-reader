## 摘要
长跨度大语言模型推理将键值（KV）缓存变成主要的GPU内存消耗者，并使每个令牌的注意力计算越来越昂贵。许多常见的逐出策略使用静态的新近窗口或历史注意力，忽略了在每个解码步骤上计算的一个信号：模型的当前不确定性。我们提出了CONF-KV，一种KV缓存管理器，它将来令牌分布转换为标量置信度分数，并用它来选择每步缓存预算，在模型不确定时保留更多上下文，在置信时激进地剪枝。在每个预算内，令牌根据累积注意力质量和新鲜度的复合指标排序，同时一个受保护的新近窗口保持局部连贯性。我们将该策略与分块在线softmax注意力、混合FP16/INT8存储以及金字塔式逐层预算变体相结合。在四个模型家族和生成长度高达4K的情况下，CONF-KV接近固定512令牌滑动窗口的内存占用，同时保持在全KV的1.5-2.1困惑度点内。在长达32K令牌的大海捞针任务中，CONF-KV达到91.4%的检索准确率，而滑动窗口为53.8%，H2O为80.6%；在75个VisualWebArena任务中，它保留了全KV成功率的95.3%，峰值内存降低2.8倍。

## Abstract
Long-horizon LLM inference turns the key--value (KV) cache into the dominant GPU memory consumer and makes per-token attention increasingly expensive. Many common eviction policies use static recency windows or historical attention, leaving unused a signal computed on every decoding step: the model's current uncertainty. We introduce CONF-KV, a KV-cache manager that converts the next-token distribution into a scalar confidence score and uses it to choose the per-step cache budget, retaining more context when the model is uncertain and pruning aggressively when it is confident. Within each budget, tokens are ranked by a composite of accumulated attention mass and recency, while a protected recent window preserves local coherence. We combine the policy with blockwise online-softmax attention, mixed FP16/INT8 storage, and a pyramidal per-layer budget variant. Across four model families and generated lengths up to 4K, CONF-KV stays near the footprint of a fixed 512-token sliding window while remaining within 1.5--2.1 perplexity points of full KV. On Needle-in-a-Haystack up to 32K tokens, CONF-KV reaches 91.4% retrieval accuracy versus 53.8% for sliding windows and 80.6% for H2O; on 75 VisualWebArena tasks it retains 95.3% of full-KV success at 2.8 times lower peak memory.

- **核心问题**：长序列LLM推理中，KV缓存随序列长度线性增长，成为GPU显存的主要消耗者，并导致每个token的注意力计算日益昂贵。现有常见的缓存逐出策略（如滑动窗口、H2O、Scissorhands等）依赖于静态的新近窗口或历史注意力累积，忽略了在每个解码步骤中已经计算出的一个宝贵信号——模型当前的不确定性（即下一个token分布的置信度）。
- **整体含义**：如果能利用这一免费的、前向的置信度信号来动态调整每步的缓存预算（不确定时保留更多上下文，自信时积极剪枝），则可以在保持模型质量的同时大幅降低显存占用，改善长序列推理的效率-质量权衡。论文提出CONF-KV，证明这种置信度感知的缓存管理策略显著优于静态或仅基于历史的方法。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- 在每个解码步骤，将下一个token的logits通过softmax得到分布，然后将其转换为一个标量置信度分数 \(c\)。
- 根据置信度与阈值 \(\tau\) 的比较，动态选择该步的缓存预算（tight budget \(N_{\text{high}}\) 或 loose budget \(N_{\text{low}}\)）。模型自信时使用更小的预算，不确定时使用更大的预算。
- 在选定预算内，按累积注意力质量（EMA）和新近度的加权组合对缓存中的token排序，并逐出排名最低的token，同时保留一个受保护的新近窗口（最近P个token）以保证局部连贯性。
- 支持混合精度存储（FP16保留最近W个token，其余量化到INT8）和金字塔式逐层预算分配（深层使用更小预算）。

### 关键技术细节
- **置信度计算公式**：结合归一化熵 \( \hat{H} \)、log概率边际 \(m\) 和top-token概率 \(p_{(1)}\)，加权求和后经sigmoid映射：
  \[
  c = w_H (1-\hat{H}) + w_m \sigma(m) + w_p p_{(1)},\quad (w_H,w_m,w_p)=(0.4,0.3,0.3)
  \]
  权重通过小范围网格搜索确定，在各种模型上鲁棒。
- **预算选择**：\( N = \begin{cases} N_{\text{high}} & \text{if } c \ge \tau \\ N_{\text{low}} & \text{if } c < \tau \end{cases} \)，其中 \(N_{\text{high}} \le N_{\text{low}}\)。
- **排序分数**：\( s_i = \alpha \hat{a}_i + (1-\alpha)\hat{r}_i \)，其中 \(\hat{a}_i\) 是归一化的EMA注意力质量（衰减因子 \(\lambda=0.9\)），\(\hat{r}_i\) 是归一化的新近度（按生成步数，越新越大）。对非保护候选进行min-max归一化。默认 \(\alpha=0.65\)。
- **混合精度存储**：最近W个（默认128或256）保留的token以FP16存储，更早的token对称量化到INT8（每头每通道），量化/反量化融合在分块注意力的读取中。这提供了大部分INT8的显存收益，同时避免NF4/INT4的较大质量损失。
- **金字塔逐层预算**：CONF-KV-L变体将预算按层指数衰减：\( N^{(\ell)}_{\text{high}} = \max(N_{\min}, N^{(0)}_{\text{high}} \beta^{\ell/L}) \)，其中 \(\beta=0.5\)，\(N_{\min}=96\)。深层运用更小的预算，进一步压缩显存。

### 算法流程（单步生成）
1. 用模型和当前缓存计算logits和注意力矩阵。
2. 从logits计算置信度c。
3. 根据c选择预算N。
4. 对每层：更新EMA注意力质量；若缓存长度>N，则对非保护token排序并逐出低分token，然后紧凑压缩幸存者。
5. 将新生成的K/V以FP16追加到缓存，同时将超出FP16窗口的旧token量化到INT8。
6. 从分布中采样下一个token。

## 6. 论文的主要结论与发现

- **主要结论**：CONF-KV在匹配显存占用条件下，大幅优于固定滑动窗口和历史注意力基线。在GPT-2上，CONF-KV+INT8将PPL从滑动窗口的34.37降至31.26（全KV为29.14），CONF-KV-L进一步降至30.48（仅比全KV高1.34点，关闭74%的差距）。Qwen-32B上绝对显存节省13.2 GB。
- **长上下文检索**：在32K NIAH中，CONF-KV达到91.4%平均准确率（滑动窗口53.8%，H2O 80.6%）。失败主要发生在模型对罕见实体过于自信时，可通过调高\(\tau\)恢复。
- **Web agent任务**：CONF-KV保留全KV成功率的95.3%（40.2%→38.3%），峰值显存降低2.8×；而滑动窗口损失11.1个绝对点。
- **延迟与吞吐量**：在2048 tokens生成时，CONF-KV相比全KV减少p50延迟约1.8×；batch=8时吞吐量提升2.06×，匹配滑动窗口。
- **置信度信号的有效性**：隔离实验显示，相同逐出率下随机逐出PPL为36.54（比滑动窗口还差），而CONF-KV为30.92，且置信度与KL损失（上下文依赖性）呈显著负相关（Pearson \(r=-0.77\)）。

## 7. 优点

1. **无需训练**：CONF-KV不改变模型权重，不进行额外训练，可直接插入标准生成循环，实用性高。
2. **利用免费信号**：下一个token的logits是每个解码步骤已计算的，无需额外计算开销。
3. **自适应预算**：动态每步调整，在困难步骤保留更多上下文，在简单步骤积极压缩，优于静态全局上限。
4. **可组合性**：可与头级分配、层预算、量化、分页注意力等技术正交组合，具有很好的扩展性。
5. **系统设计清晰**：包含紧凑压缩、混合精度存储、金字塔层预算，整体实现简洁高效。
6. **实证全面**：在多个模型、多个任务上验证了有效性，消融实验充分说明了各组件贡献。

## 8. 不足与局限

1. **短上下文无优势**：当上下文长度从未超过逐出阈值时，CONF-KV不会触发，无法带来收益。
2. **高温采样下置信度信号不灵敏**：高温导致熵饱和，置信度区分度下降，可能需要重新调整阈值\(\tau\)。
3. **压缩开销**：连续紧凑压缩增加0.22ms/步（GPT-2上），虽然小于注意力节省，但仍是一个开销来源。
4. **与PagedAttention布局的兼容性**：token级逐出会导致部分空页，生产系统中可能需要粗粒度逐出或稀疏掩码。
5. **剩余失败模式**：在罕见实体检索或Web任务中，模型在自信状态下可能错误逐出关键历史信息，虽然可通过调高\(\tau\)缓解，但会牺牲部分内存节省。
6. **实验覆盖**：仅测试了最大32K上下文（虽然NIAH中已包含），未在更长序列（如128K以上）上验证；吞吐量实验受显存限制未能完成batch>8的全KV对比。
7. **超参数选择**：置信度权重（0.4,0.3,0.3）基于GPT-2的网格搜索，虽在其他模型上表现稳定，但可能不是最优，需用户微调。
