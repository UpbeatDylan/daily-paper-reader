## 摘要
长上下文语言模型受限于键值（KV）缓存的内存占用。现有的免训练KV压缩方法通常根据单一重要性信号——注意力、时效性、逐层分配或键独特性——对词元进行排序，这在有用上下文具有全局独特性、局部片段性或即时相关性时会变得脆弱。我们提出NestedKV，一种受嵌套学习中连续记忆系统启发的仅键KV缓存压缩方法。NestedKV维护全局、块级和滑动窗口键锚点，通过多时间尺度余弦异常对词元进行评分，并将所得排序与使用头部自适应混合和惊喜门控词元路由的免训练外部学习器相结合。该分数与每个头部的自适应预算配对，无需训练或修改LLM。在Qwen3和Llama-3.2模型上的RULER（4k--32k）、LooGLE、LongBench、LongBench-E、InfiniteBench和MMLU-Pro测试中，当保留的缓存较小时，NestedKV表现最强。在Qwen3-4B上，当$r=0.75$时，它在RULER上比KeyDiff提升高达19.10分，在LongBench上提升19.29分；当$r=0.95$时，它在LongBench上保留37.32分，而KeyDiff为17.55分。

## Abstract
Long-context language models are limited by the memory footprint of the key-value (KV) cache. Existing training-free KV compression methods usually rank tokens by one importance signal -- attention, recency, layer-wise allocation, or key distinctiveness -- which becomes brittle when useful context is globally distinctive, locally episodic, or immediately relevant. We introduce NestedKV, a key-only KV cache compression method inspired by the Continuum Memory System in Nested Learning. NestedKV maintains global, block-level, and sliding-window key anchors, scores tokens by multi-time-scale cosine anomaly, and combines the resulting rankings with a training-free outer learner using head-adaptive mixing and surprise-gated token routing. The score is paired with adaptive per-head budgets and requires no training or LLM modification. Across RULER (4k--32k), LooGLE, LongBench, LongBench-E, InfiniteBench, and MMLU-Pro on Qwen3 and Llama-3.2 models, NestedKV is strongest when the retained cache is small. On Qwen3-4B, it improves over KeyDiff by up to 19.10 points on RULER and 19.29 on LongBench at $r=0.75$; at $r=0.95$, it retains 37.32 on LongBench versus 17.55 for KeyDiff.

## 1. 核心问题与整体含义（研究动机和背景）
- **核心问题**：长上下文大语言模型（LLM）的键值（KV）缓存随着上下文长度和批处理规模线性增长，在推理时成为内存瓶颈。现有免训练（training-free）KV压缩方法通常仅依赖单一重要性信号（如注意力分布、最近性、逐层预算分配或键的几何独特性）来筛选缓存中的词元，这在复杂上下文（同时包含全局独特信息、局部片段性信息以及即时相关流）中表现脆弱，易将关键证据过早剔除。
- **研究动机**：上述单一信号在强压缩比或长上下文场景下会错误丢弃重要词元（例如注意力集中于前缀与尾部，而答案词元位于低注意力区域）。作者受**嵌套学习（Nested Learning）**中的**连续记忆系统（Continuum Memory System）**启发，提出一种多时间尺度、自适应的无训练压缩策略。
- **整体含义**：将KV压缩重新定义为对键流（key stream）进行**连续记忆异常检测**的维护过程，通过保留在不同时间尺度上“异常”（信息量高）的词元，以更鲁棒地支持长上下文推理，且无需微调模型或修改注意力实现。

## 2. 方法论：核心思想、关键技术细节、公式或算法流程
### 2.1 核心思想
- **三层连续记忆状态**：为每个缓存词元维护三个时间尺度的键均值锚点：
  - **稳定记忆（Stable Memory）**：整个前缀上下文的全局均值 \(\mu_s = \frac{1}{N}\sum_{j=1}^N \hat{k}_j\)，捕捉文档级常规模式。
  - **片段记忆（Episodic Memory）**：词元所在局部块的均值 \(\mu_e(i)\)，块大小由 \(b = \text{clip}(\lfloor N/32\rfloor, 128, 256)\) 决定，捕捉段落或轮次级结构。
  - **当前记忆（Current Memory）**：以词元结尾的滑动窗口均值 \(\mu_c(i)\)，窗口大小固定为 \(W=64\)，捕捉短期连续性。
- **异常评分**：对每个词元计算它与三个锚点的负余弦相似度（即余弦异常），得分越高表示该词元在该尺度上越“异常”（信息量大），越应保留。
- **外部学习器（Outer Learner）**：免训练地融合三个尺度的评分：
  - **头部自适应混合（Head-adaptive Blend）**：对每个注意力头，根据其三个尺度得分的区分度（top 10% 与 bottom 10% 的差值）动态调整混合权重，通过带固定对数先验 \((0.4,0.4,0.2)\) 的 softmax 实现。
  - **压缩诱导惊喜（Compression-induced Surprise）**：计算三个尺度打分标准差作为“惊喜”，当惊喜高（三个尺度分歧大）时，用 sigmoid 门控转向最强个体记忆（取最大值）而非混合值。
- **头部级别自适应预算**：将所有头-词元的得分统一排序，按层级总预算 \(B_\ell\) 选择全局最高的头-词元对，再分配每头的实际保留数，同时每头保留至少 20% 的预算作为安全界限。

### 2.2 关键技术细节
- 仅使用键（key）进行评分，不依赖查询（query），值与评分无关（仅跟随键保留）。
- 前 4 个词元（attention sink）强制固定保留。
- 所有评分在预填充后、自回归解码前一次性完成，不随生成更新。
- 无需训练参数，不修改 LLM 权重或注意力函数。
